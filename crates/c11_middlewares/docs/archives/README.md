## 📝 归档规则

文档在以下情况下会被归档：

1. **版本更新**: 被新版本文档完全替代
2. **功能废弃**: 对应功能已从项目中移除
3. **重构完成**: 文档经过大规模重构，旧版本归档
4. **内容过时**: 信息严重过时且不再适用

**归档流程**:

1. 将文档移动到 `archives/` 目录
2. 在原位置添加指向新文档的链接（如有）
3. 更新本 README 的归档列表
4. 在文档顶部添加归档标记

**归档标记示例**:

```markdown
> ⚠️ **归档文档**: 本文档已归档，请查看 [新文档链接](../path/to/new/doc.md)
> 
> - 归档日期: 2025-XX-XX
> - 归档原因: 文档已被新版本替代
```
