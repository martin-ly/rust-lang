## 🔍 详细解析步骤

### 1. 依赖图构建

```text
┌─────────────────────────────────────────────────────┐
│                   根包 (Root)                        │
│                  my-project 1.0.0                    │
└───────────┬─────────────────────────┬────────────────┘
            │                         │
            ├─────────────┐           ├─────────────┐
            ▼             ▼           ▼             ▼
       ┌────────┐    ┌────────┐  ┌────────┐   ┌────────┐
       │ tokio  │    │ serde  │  │ reqwest│   │ clap   │
       │ 1.48   │    │ 1.0    │  │ 0.12   │   │ 4.5    │
       └────┬───┘    └────┬───┘  └────┬───┘   └────────┘
            │             │           │
            ├──────┐      │           ├──────┐
            ▼      ▼      ▼           ▼      ▼
        ┌──────┐┌──────┐┌────┐    ┌─────┐┌─────┐
        │ mio  ││libc  ││serde│    │tokio││http │
        │ 1.0  ││0.2   ││ 1.0 │    │ 1.48││ 1.1 │
        └──────┘└──────┘└─────┘    └─────┘└─────┘
                        ▲                   │
                        └───────────────────┘
                         (版本统一)
```

**关键点**:

- 相同依赖不同版本会被检测
- Resolver 3 会尝试统一版本
- 避免不必要的重复编译

### 2. 版本冲突解析 (Resolver 3)

```text
冲突场景:
┌──────────────────────────────────────────┐
│  Package A 需要 tokio ^1.40              │
│  Package B 需要 tokio ^1.48              │
│  如何解决？                               │
└──────────────────────────────────────────┘

Resolver 3 策略:
┌──────────────────────────────────────────┐
│ 1. 计算版本兼容性                         │
│    ^1.40 ∩ ^1.48 = 1.48.0                │
│                                          │
│ 2. 选择最新兼容版本                       │
│    ✓ 选择 tokio 1.48.0                   │
│                                          │
│ 3. 验证所有依赖满足                       │
│    ✓ A: 1.48.0 ≥ 1.40.0 ✅              │
│    ✓ B: 1.48.0 ≥ 1.48.0 ✅              │
└──────────────────────────────────────────┘

结果: 统一使用 tokio 1.48.0
```

### 3. 特性传播流程

```mermaid
graph LR
    A[根包启用 tokio/full] --> B[tokio 1.48]
    B --> C{检查特性依赖}

    C -->|需要 tokio-macros| E[启用 tokio-macros]

    E --> G
    F --> G
    G --> H[生成最终特性集]

    style A fill:#e1f5ff
    style H fill:#c8e6c9
```

**Resolver 3 改进**:

- 更精确的特性传播
- 避免不必要的特性激活
- 减少编译时间和二进制大小

## 🐛 常见问题诊断

### 问题 1: 版本冲突

```bash
# 查看依赖树
cargo tree --duplicates

# 输出示例:
tokio v1.40.0
└── package-a v1.0.0

tokio v1.48.0
└── package-b v2.0.0

# 解决: 更新 Cargo.toml
[dependencies]
tokio = "1.48"  # 统一版本
```

### 问题 2: 特性冲突

```bash
# 查看特性传播
cargo tree --format "{p} {f}"

# 解决策略
[dependencies]
tokio = { version = "1.48", features = ["full"], default-features = false }
```
