认证说明

所有API请求都需要在请求头中包含有效的认证令牌：

Authorization: Bearer your_token_here

除了健康检查端点外，所有其他端点都需要认证。

错误处理

API使用标准的HTTP状态码，错误响应格式如下： 

{
  "success": false,
  "error": "错误描述",
  "message": "详细错误信息"
}

常见状态码

  • 200 - 成功
  • 400 - 请求错误
  • 401 - 未授权
  • 404 - 资源未找到
  • 429 - 请求过多
  • 500 - 服务器错误

错误类型

  • ModuleNotFound - 模块未找到
  • ConfigNotFound - 配置未找到
  • InvalidConfig - 无效配置
  • ResourceLimitExceeded - 资源限制超出
  • InternalError - 内部错误
