## 🚀 快速开始

### 1. Fork和克隆项目

```bash
# Fork项目到您的GitHub账户
# 然后克隆您的fork
git clone https://github.com/YOUR_USERNAME/c10_networks.git
cd c10_networks

# 添加上游仓库
git remote add upstream https://github.com/ORIGINAL_OWNER/c10_networks.git
```

### 2. 设置开发环境

```bash
# 确保使用Rust 1.90+
rustup update
rustup default 1.90

# 安装开发工具
cargo install cargo-fmt cargo-clippy cargo-test

# 运行测试确保环境正常
cargo test
```

### 3. 创建功能分支

```bash
# 创建新的功能分支
git checkout -b feature/your-feature-name

# 或者创建修复分支
git checkout -b fix/issue-number
```

## 📋 开发流程

### 🔄 开发循环

1. **创建Issue**: 描述您要解决的问题或添加的功能
2. **创建分支**: 基于main分支创建功能分支
3. **编写代码**: 实现功能并添加测试
4. **运行测试**: 确保所有测试通过
5. **代码审查**: 提交Pull Request
6. **合并代码**: 经过审查后合并到main分支

### ✅ 代码质量标准

- **编译通过**: 代码必须能够成功编译
- **测试通过**: 所有现有测试必须通过
- **新测试**: 新功能必须包含相应的测试
- **文档完整**: 公共API必须有完整的文档
- **格式规范**: 代码必须符合rustfmt格式
- **Clippy检查**: 通过所有Clippy警告检查

## 🧪 测试指南

### 运行测试

```bash
# 运行所有测试
cargo test

# 运行特定测试
cargo test test_name

# 运行文档测试
cargo test --doc

# 运行示例测试
cargo test --examples
```

### 添加新测试

```rust
#[cfg(test)]
mod tests {
    use super::*;
    
    #[tokio::test]
    async fn test_new_feature() -> NetworkResult<()> {
        // 测试实现
        Ok(())
    }
}
```

### 性能测试

```bash
# 运行性能基准测试
cargo bench

# 运行特定基准测试
cargo bench --bench benchmark_name
```

## 📝 提交规范

### 提交信息格式

```text
类型(范围): 简短描述

详细描述（可选）

相关Issue: #123
```

### 提交类型

- **feat**: 新功能
- **fix**: Bug修复
- **docs**: 文档更新
- **style**: 代码格式调整
- **refactor**: 代码重构
- **test**: 测试相关
- **chore**: 构建过程或辅助工具的变动

### 示例

```text
feat(protocol): 添加HTTP/2支持

- 实现HTTP/2协议解析
- 添加流控制和多路复用
- 包含完整的测试覆盖

相关Issue: #45
```

## 🔍 代码审查

### 审查要点

- **功能正确性**: 代码是否实现了预期功能
- **性能影响**: 是否对性能产生负面影响
- **代码质量**: 代码是否清晰、可维护
- **测试覆盖**: 是否有足够的测试覆盖
- **文档完整**: 是否有完整的文档和注释

### 审查流程

1. **自动检查**: CI/CD流水线自动运行
2. **人工审查**: 维护者进行代码审查
3. **讨论修改**: 根据反馈进行修改
4. **最终批准**: 维护者批准合并
