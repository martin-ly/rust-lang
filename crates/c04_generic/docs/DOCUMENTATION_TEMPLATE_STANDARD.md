## 📝 格式规范

### 标题层级

- `#` H1: 文档标题（每个文档只有一个）
- `##` H2: 主要章节
- `###` H3: 次要章节  
- `####` H4: 子章节（谨慎使用）

### 编号规范

**主题文档系列** (01-05):

- 使用 `## 1.`, `## 2.` 等数字编号
- 子章节使用 `### 1.1`, `### 1.2`

**概念文档** (fundamentals, traits, etc):

- 使用 `##` 不带编号
- 保持简洁的标题

### 代码块规范

```rust
// ✅ 好的示例：清晰注释，完整可运行
/// 计算两个数的和
fn add(a: i32, b: i32) -> i32 {
    a + b
}

#[cfg(test)]
mod tests {
    use super::*;
    
    #[test]
    fn test_add() {
        assert_eq!(add(2, 3), 5);
    }
}
```

**要求**:

- 所有代码必须添加语言标识（`rust`）
- 添加清晰的注释
- 复杂示例应包含测试用例
- 避免不完整的代码片段（除非明确标注）

### 提示框规范

```markdown
> **💡 提示**: 有用的额外信息
> 
> **⚠️ 注意**: 需要注意的事项
> 
> **❌ 错误**: 常见错误
> 
> **✅ 正确**: 推荐做法
```

### 链接规范

**内部链接**:

```markdown
- [相对路径](./other_doc.md)
- [带锚点](./other_doc.md#section)
```

**外部链接**:

```markdown
- [Rust官方文档](https://doc.rust-lang.org/)
```

## 🎨 样式规范

### 强调文本

- **粗体**: 重要概念、术语定义
- *斜体*: 轻微强调、引用
- `代码`: 代码元素、文件名、命令

### 列表

**无序列表**: 使用 `-` 开头

```markdown
- 项目1
- 项目2
  - 子项目
```

**有序列表**: 使用数字

```markdown
1. 第一步
2. 第二步
   1. 子步骤
```

### 表格

```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 值1 | 值2 | 值3 |
```

**要求**:

- 表头使用清晰的列名
- 保持列对齐
- 适度使用（不超过5列）

## 📊 文档类型分类

### A. 教程类文档

**目标**: 引导学习  
**结构**: 循序渐进  
**特点**: 完整示例 + 练习

**模板**:

```markdown

## 学习目标
- 目标1
- 目标2

## 前置知识
- 知识点1

### 步骤1: ...
### 步骤2: ...

## 实践练习
### 练习1: ...

## 总结
```

### B. 参考类文档

**目标**: 快速查找  
**结构**: 主题分类  
**特点**: 完整性 + 准确性

**模板**:

```markdown

## 概述
简短说明

## 主题索引
- [主题1](#主题1)
- [主题2](#主题2)

## 主题2
...
```

**目标**: 深入理解  
**结构**: 问题导向  
**特点**: 原理 + 应用

**模板**:

