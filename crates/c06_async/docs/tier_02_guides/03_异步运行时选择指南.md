## 📐 知识结构

### 概念定义

**异步运行时 (Async Runtime)**:

- **定义**: 执行和管理异步任务的执行环境，提供任务调度、I/O 事件循环等功能
- **类型**: 库/框架
- **范畴**: 异步编程
- **版本**: Rust 1.39+
- **相关概念**: Future、async/await、任务调度、事件循环

**Tokio**:

- **定义**: 生产级异步运行时，提供完整的异步 I/O 和任务调度
- **类型**: 异步运行时
- **属性**: 工作窃取调度器、高性能、完整生态
- **关系**: 与异步编程、网络编程相关

### 属性特征

**核心属性**:

- **任务调度**: 高效的任务调度机制
- **I/O 事件循环**: 非阻塞 I/O 事件处理
- **生态系统**: 丰富的异步库支持
- **性能**: 高性能的并发执行

**性能特征**:

- **Tokio**: 最高性能，生态最完善，**新项目首选**
- **async-std [已归档]**: 已停止维护（2024），仅作为历史参考
- **Smol**: 轻量级，适合小型项目和嵌入式
- **适用场景**: 网络编程、I/O 密集型、高并发服务器

### 关系连接

**继承关系**:

- Tokio --[is-a]--> 异步运行时
- async-std [已归档] --[is-a]--> 异步运行时

**组合关系**:

- 异步应用 --[uses]--> 异步运行时
- 异步运行时 --[has-a]--> 任务调度器

**依赖关系**:

- 异步运行时 --[depends-on]--> Future Trait
- 异步运行时 --[depends-on]--> 操作系统异步 I/O

## 1. Tokio - 生产级异步运行时

### 1.1 核心特性

**优势**:

- ✅ **生态最完善**: 90%+ 的异步库支持
- ✅ **性能最优**: 工作窃取调度器，高效多核利用
- ✅ **功能最全**: Runtime、I/O、Timer、Sync 等完整支持
- ✅ **企业级**: 被 Discord、AWS、Cloudflare 等使用

**适用场景**:

- 🚀 生产环境的 Web 服务器
- 🚀 高并发微服务
- 🚀 分布式系统
- 🚀 需要极致性能的场景

### 1.2 快速开始

**添加依赖** (`Cargo.toml`):

```toml
[dependencies]
tokio = { version = "1", features = ["full"] }
```

**基本使用**:

```rust
#[tokio::main]
async fn main() {
    println!("Hello, Tokio!");
}
```

### 1.3 运行时配置

**多线程运行时（默认）**:

```rust
#[tokio::main]
async fn main() {
    // 自动使用所有 CPU 核心
}

// 手动配置
#[tokio::main(worker_threads = 4)]
async fn main() {
    // 使用 4 个工作线程
}
```

**单线程运行时**:

```rust
#[tokio::main(flavor = "current_thread")]
async fn main() {
    // 只在当前线程运行
}
```

### 1.4 核心组件

| 组件          | 功能              |
| :--- | :--- || `tokio::net`  | 异步 TCP/UDP 网络 |
| `tokio::fs`   | 异步文件系统      |
| `tokio::time` | 定时器和超时      |
| `tokio::sync` | 异步同步原语      |
| `tokio::task` | 任务管理          |

## 2. async-std [已归档] — 历史参考（已停止维护）

> ⚠️ **维护状态**: async-std [已归档] 项目已于 **2024 年停止维护**，不再接收更新或安全补丁。
> **建议**: 新项目请直接使用 **Tokio**；旧项目建议迁移至 Tokio。

### 2.1 历史背景

async-std [已归档] 曾是 Rust 异步生态的重要运行时，以"标准库风格 API"著称：

- **设计目标**: 让异步 API 尽可能接近 `std`
- **历史贡献**: 推动了 Rust 异步标准的发展
- **停止维护原因**: 社区资源向 Tokio 集中，async-std [已归档] 核心维护者退出

### 2.2 迁移路径

从 async-std [已归档] 迁移到 Tokio 的常见替换：

| async-std [已归档] | Tokio 等价方案 |
|:---|:---|
| `async_std::fs::read_to_string().await` | `tokio::fs::read_to_string().await` |
| `async_std::net::TcpListener` | `tokio::net::TcpListener` |
| `async_std::task::sleep().await` | `tokio::time::sleep().await` |
| `async_std::task::spawn` | `tokio::task::spawn` |
| `#[async_std::main]` | `#[tokio::main]` |

迁移工具: [`tokio-async-std [已归档]-compat`](<https://docs.rs/tokio-async-std> [已归档]-compat) 或逐步替换。

### 2.3 历史 API 对照（仅供参考）

> 以下代码仅作为理解 async-std [已归档] 设计的参考，**请勿在新项目中使用**。

| 标准库                    | async-std [已归档]（历史）                       | Tokio（推荐替代）                        |
| :--- | :--- | :--- |
| `std::fs::read_to_string` | `async_std::fs::read_to_string().await` | `tokio::fs::read_to_string().await`      |
| `std::net::TcpListener`   | `async_std::net::TcpListener`           | `tokio::net::TcpListener`                |
| `std::thread::sleep`      | `async_std::task::sleep().await`        | `tokio::time::sleep().await`             |

## 6. 实战示例

### 6.1 HTTP 服务器对比

**Tokio (with Axum)**:

```rust
use axum::{routing::get, Router};

#[tokio::main]
async fn main() {
    let app = Router::new().route("/", get(|| async { "Hello, Tokio!" }));

    let listener = tokio::net::TcpListener::bind("0.0.0.0:3000")
        .await
        .unwrap();
    axum::serve(listener, app).await.unwrap();
}
```

**async-std [已归档] (历史示例，不推荐)**:

```rust
// ⚠️ async-std [已归档] 和 Tide 均已停止维护，以下代码仅作历史参考
use tide::Request;

#[async_std::main]
async fn main() -> tide::Result<()> {
    let mut app = tide::new();
    app.at("/").get(|_: Request<()>| async { Ok("Hello, async-std [已归档]!") });
    app.listen("0.0.0.0:3000").await?;
    Ok(())
}
```

**Tokio (with Axum) — 推荐方案**:

```rust
use axum::{routing::get, Router};

#[tokio::main]
async fn main() {
    let app = Router::new().route("/", get(|| async { "Hello, Tokio!" }));
    let listener = tokio::net::TcpListener::bind("0.0.0.0:3000").await.unwrap();
    axum::serve(listener, app).await.unwrap();
}
```

**Smol (with Hyper)**:

```rust
use hyper::{Body, Request, Response, Server};
use hyper::service::{make_service_fn, service_fn};

fn main() {
    smol::block_on(async {
        let make_svc = make_service_fn(|_| async {
            Ok::<_, hyper::Error>(service_fn(|_: Request<Body>| async {
                Ok::<_, hyper::Error>(Response::new(Body::from("Hello, Smol!")))
            }))
        });

        let server = Server::bind(&([0, 0, 0, 0], 3000).into()).serve(make_svc);
        server.await.unwrap();
    });
}
```

## 7. 混用运行时（高级）

### 7.1 兼容层

**async-compat**: 桥接不同运行时

```toml
[dependencies]
async-compat = "0.2"
```

```rust
use async_compat::Compat;

#[tokio::main]
async fn main() {
    // 兼容层示例（历史遗留场景）
    let result = Compat::new(async {
        // 旧 async-std [已归档] 代码通过兼容层在 Tokio 上运行
        tokio::time::sleep(std::time::Duration::from_secs(1)).await;
        "Done"
    }).await;

    println!("{}", result);
}
```

⚠️ **警告**: 混用运行时会增加复杂性，仅在必要时使用。

## 8. 最佳实践
