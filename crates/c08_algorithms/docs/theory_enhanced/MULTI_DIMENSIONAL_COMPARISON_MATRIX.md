## 1. 排序算法全面对比

### 排序性能对比 (10万元素)

| 算法 | 时间 | 内存 | 稳定性 | 适用 |
|------|------|------|--------|------|
| 快速排序 | 8 ms | O(log n) | ❌ | 通用 |
| 归并排序 | 12 ms | O(n) | ✅ | 稳定 |
| 堆排序 | 15 ms | O(1) | ❌ | 原地 |
| Tim排序 | 10 ms | O(n) | ✅ | 部分有序 |

### 排序特性矩阵

| 算法 | 最优 | 平均 | 最坏 | 空间 |
|------|------|------|------|------|
| **快速排序** | O(n log n) | O(n log n) | O(n²) | O(log n) |
| **归并排序** | O(n log n) | O(n log n) | O(n log n) | O(n) |
| **堆排序** | O(n log n) | O(n log n) | O(n log n) | O(1) |

## 2. 搜索算法对比

### 搜索性能对比

| 算法 | 时间复杂度 | 前提条件 | 适用 |
|------|-----------|---------|------|
| **线性搜索** | O(n) | 无 | 小数据集 |
| **二分搜索** | O(log n) | 有序 | 大数据集 |
| **哈希查找** | O(1) | 哈希表 | 快速查找 |
| **DFS** | O(V+E) | 图 | 路径搜索 |
| **BFS** | O(V+E) | 图 | 最短路径 |

## 5. 技术选型决策

### 按场景选型

| 场景 | 推荐算法 | 原因 |
|------|---------|------|
| **通用排序** | 快速排序 | 平均最优 |
| **稳定排序** | 归并/Tim | 稳定性 |
| **大数据排序** | 外部归并 | 内存限制 |
| **快速查找** | HashMap | O(1)查找 |
| **有序遍历** | BTreeMap | 有序迭代 |
| **并行计算** | rayon | 多核加速 |

## 6. 总结与最佳实践

### 黄金法则

1. ✅ 优先标准库算法
2. ✅ 使用rayon并行化
3. ✅ 选择合适的数据结构
4. ❌ 避免过早优化
