## 三、改进计划（三阶段）

### Phase A — 版本对齐冲刺（2-3 周，最高优先级）

**目标**: 将 Rust 1.96 引用提升至与 1.94 同等水平，清理 1.94 过时内容

| 任务 | 范围 | 工作量 | 验收标准 |
|------|------|--------|----------|
| A1. `rust_194_*` 文件重命名与内容更新 | ~20 个文件名含 `194` 的 concept 文件 | 中等 | 文件名更新为 `rust_196_*` 或通用名称，内容确认 1.96 兼容性 |
| A2. `array_windows` 等 1.94→1.96 特性勘误 | concept/ + knowledge/ 中提及版本号的代码示例 | 大 | 所有 `Rust 1.94` 引用中，已稳定的特性改为 `Rust 1.96`，并验证代码块编译 |
| A3. `version_fact_check.py` 批量修复 | 全项目 3,819 处 1.94 引用 | 大 | 1.96 引用数 ≥ 1,000，1.94 引用数降至 < 500（仅保留历史说明） |
| A4. `knowledge/INDEX.md` 版本矩阵更新 | 1 个文件 | 小 | 1.96 特性完整列出，1.97 预览正确标注 |

### Phase B — 生态清理与国际化（2-3 周）

**目标**: 清理 async-std 残留，建立最低限度双语支持

| 任务 | 范围 | 工作量 | 验收标准 |
|------|------|--------|----------|
| B1. async-std 全局标记 | 66 个文件 | 中等 | 所有 async-std 引用均标注 `[已归档 2025-03]` 或 `[历史]` |
| B2. crates 英文 doc 注释 — 零英文优先 | `c02`, `c05`, `common` + examples/ | 中等 | 三个 crate 的 lib.rs 和所有 pub 项均有英文 `///` |
| B3. crates 英文 doc 注释 — 批量补充 | `c03`, `c08`, `c10`, `c13`, `exercises` | 大 | 80% 的 pub 项有双语 doc |
| B4. L6 代码示例补充 | 21 个标记文件 | 大 | 每个文件至少 1 个可编译 ` ```rust ` 代码块 |

### Phase C — 结构优化与深度对齐（3-4 周）

**目标**: 减少三轨重复，提升权威来源质量

| 任务 | 范围 | 工作量 | 验收标准 |

## 四、执行顺序建议

```
Week 1-2: Phase A (版本对齐) — 影响面最大，学习者受益最直接
    └── A1 + A2 同步推进，A3 脚本自动化，A4 收尾

Week 3-4: Phase B (生态清理 + i18n) — 质量提升
    └── B1 与 A 并行，B2 优先零英文 crate，B3 批量处理，B4 逐步补充

Week 5-8: Phase C (结构优化) — 长期维护收益
    └── C1 需人工决策，C2 可脚本辅助，C3 可用链接检查器，C4 等待上游
```
