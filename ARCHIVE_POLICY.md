# Archive 政策

> **版本**: 1.0
> **生效日期**: 2026-06-28
> **适用范围**: 全仓库 `archive/`、`docs/archive/`、`docs/rust-ownership-decidability/` 等历史内容

---

## 1. 目的

本政策明确 `archive/` 目录的用途与管理规则，解决历史内容、重复内容、审计备份与活跃内容混放导致的一致性和可发现性问题。

## 2. Archive 定义

`archive/` 用于存放**只读历史参考内容**，包括但不限于：

- 已被替代或合并的旧版本文档
- 历史审计、检查清单、周报、计划
- 研究笔记、实验性内容、未完成的草稿
- 从 `docs/`、`concept/`、`knowledge/` 迁移出来的重复或过期内容

## 3. 归档标准

满足以下任一条件的内容应归档：

1. **时间标准**: 超过 6 个月未更新，且不再作为活跃学习/参考路径
2. **替代标准**: 内容已被新文件替代，活跃路径已指向新文件
3. **审计标准**: 历史审计、检查清单、周报等一次性或周期性产物
4. **重复标准**: 与 `concept/` 中权威内容重复，且 `concept/` 版本更完整
5. **用户决策**: 经维护者/用户明确决策后迁移

## 4. 目录结构

```text
archive/
├── docs/                          # 从 docs/archive/ 迁移的历史文档
│   └── ...
├── rust-ownership-decidability/   # 从 docs/rust-ownership-decidability/ 迁移的历史研究
│   └── ...
├── kimi/                          # 从 .kimi/archive/ 迁移的历史计划
│   └── ...
├── scripts/                       # 从 scripts/archive/ 整理的历史脚本
│   └── ...
└── README.md                      # 本政策摘要与 archive 目录说明
```

## 5. 迁移规则

- 迁移时使用 `git mv` 保留文件历史
- 迁移后必须修复活跃目录中的内部链接
- 不再维护 archive 内文件的内容，但允许添加 `README.md` 说明或目录索引
- 禁止将新产生的活跃内容放入 archive

## 6. 链接策略

- 活跃文件应尽量避免直接链接到 archive 内容
- 如必须引用历史资料，链接文本应明确标注“（历史归档）”
- 迁移后，原路径不应再出现在新的活跃内容中

## 7. 禁止行为

- 禁止 CI、脚本、构建流程向 archive 写入新文件
- 禁止在 archive 中继续编辑旧内容（除非修正明显的 broken link 或添加归档说明）
- 禁止将 archive 文件作为学习路径的默认入口

## 8. 例外

以下情况可暂不归档：

- 仍有活跃学习路径引用的过渡性内容（应先迁移学习路径再归档）
- 用户明确保留在活跃目录中的参考内容
- 正在进行的审计或迁移过程中的临时副本
