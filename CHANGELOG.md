## 版本说明

### 版本号格式

我们使用语义化版本号：`主版本号.次版本号.修订号`

- **主版本号**：不兼容的 API 修改
- **次版本号**：向下兼容的功能性新增
- **修订号**：向下兼容的问题修正

### 变更类型

- **新增 (Added)**: 新功能
- **变更 (Changed)**: 对现有功能的变更
- **弃用 (Deprecated)**: 即将删除的功能
- **移除 (Removed)**: 已删除的功能
- **修复 (Fixed)**: Bug 修复
- **安全 (Security)**: 安全相关的修复

## 如何查看变更

### 查看所有版本

```bash
git tag -l
```

### 查看特定版本

```bash
git show v1.0.0
```

### 对比版本差异

```bash
git diff v0.9.0..v1.0.0
```

## 贡献更新日志

当你为项目做出贡献时，请在 PR 中说明变更内容，维护者会将其添加到更新日志中。

### 更新日志规范

1. **保持格式一致**：遵循现有格式
2. **清晰描述变更**：说明变更内容和原因
3. **引用相关 Issue**：添加 Issue/PR 链接
4. **按类型分组**：使用标准的变更类型
