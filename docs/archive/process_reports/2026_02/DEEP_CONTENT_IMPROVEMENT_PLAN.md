## 📋 对齐标准规范

### 1. 层次对齐标准

#### 1.1 概念层次

```markdown
## 概念层次分析

### L1: 元概念
- **定义**: 最高层抽象概念
- **示例**: 资源管理、类型安全

### L2: 核心概念族
- **定义**: 由元概念派生的概念族
- **示例**: 所有权系统、借用系统

### L3: 具体概念
- **定义**: 可直接应用的概念
- **示例**: 移动语义、可变借用

### L4: 实现/应用
- **定义**: 代码层面的实现
- **示例**: `let s = String::from("x");`
```

#### 1.2 文档层次

```markdown
## 文档层次

| 层次 | 职责 | 内容深度 | 示例 |
| :--- | :--- | :--- | :--- |
| **索引层** | 导航入口 | 极简 | README.md |
| **概览层** | 全景概览 | 概述 | 00_MASTER_INDEX.md |
| **核心层** | 核心概念 | 中等 | ownership_model.md |
| **详细层** | 深入细节 | 深入 | CORE_THEOREMS_FULL_PROOFS.md |
| **示例层** | 代码示例 | 实用 | quick_reference/*.md |
```

```markdown
## 领域: [领域名称]

### 边界定义
- **包含**: 哪些概念
- **排除**: 哪些概念

### 核心抽象
- **代数模型**: ...

- **操作语义**: ...

### 关键定理
| 定理 | 说明 | 证明 |
| :--- | :--- | :--- |
| T1 | ... | [链接] |
```

### 3. 权威对齐标准

#### 3.1 对齐标记

```markdown
> **权威来源**: [The Rust Book - 章节](链接)
> **对齐状态**: ✅ 一致 / ⚠️ 差异 / ❌ 冲突
> **差异说明**: 如有差异，说明原因
```

#### 3.2 差异处理

- **一致** - 直接引用
- **差异** - 说明差异原因（补充/简化/项目特定）
- **冲突** - 详细论证为何采用本项目定义

### 4. 思维表征标准

#### 4.1 表征选择指南

| 表征类型 | 适用场景 | 工具 |
| :--- | :--- | :--- |
| 思维导图 | 概念层级、知识结构 | Mermaid mindmap |
| 矩阵 | 对比分析、技术选型 | Markdown table |
| 决策树 | 选择逻辑、故障排查 | Mermaid graph TD |
| 证明树 | 逻辑推导、定理证明 | Mermaid graph BT |
| 知识图谱 | 概念关联、文档网络 | Mermaid graph LR |
| 时序图 | 流程、生命周期 | Mermaid sequenceDiagram |

#### 4.2 表征完整性要求

每个核心概念至少应有：

- 1 个思维导图（层级结构）
- 1 个对比矩阵（与其他概念）
- 1 个决策树（何时使用）
- 1 个知识图谱（关联网络）
