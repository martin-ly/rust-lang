## 🌳 决策树：任务推进策略

```mermaid
flowchart TD
    Start([开始任务推进]) --> CheckPriority{检查优先级}

    CheckPriority -->|高优先级| HighPriority[高优先级任务]
    CheckPriority -->|中优先级| MidPriority[中优先级任务]
    CheckPriority -->|低优先级| LowPriority[低优先级任务]

    HighPriority --> CheckDependency{检查依赖}
    CheckDependency -->|无依赖| StartImmediate[立即开始]
    CheckDependency -->|有依赖| WaitDependency[等待依赖完成]

    StartImmediate --> ParallelExecution[并行执行]
    WaitDependency --> CheckReady{依赖就绪?}
    CheckReady -->|是| ParallelExecution
    CheckReady -->|否| WaitDependency

    MidPriority --> CheckResources{检查资源}
    CheckResources -->|资源充足| StartMid[开始执行]
    CheckResources -->|资源不足| QueueMid[加入队列]

    LowPriority --> CheckTime{检查时间}
    CheckTime -->|有时间| StartLow[开始执行]
    CheckTime -->|无时间| QueueLow[加入队列]

    ParallelExecution --> TaskExecution[任务执行]
    StartMid --> TaskExecution
    StartLow --> TaskExecution

    TaskExecution --> CheckProgress{检查进度}
    CheckProgress -->|完成| Verify[验证质量]
    CheckProgress -->|未完成| Continue[继续执行]
    Continue --> TaskExecution

    Verify --> QualityCheck{质量检查}
    QualityCheck -->|通过| MarkComplete[标记完成]
    QualityCheck -->|未通过| FixIssues[修复问题]
    FixIssues --> TaskExecution

    MarkComplete --> UpdateStatus[更新状态]
    UpdateStatus --> NextTask[下一个任务]
    NextTask --> CheckPriority

    QueueMid --> NextTask
    QueueLow --> NextTask

    style HighPriority fill:#ff6b6b
    style MidPriority fill:#ffd93d
    style LowPriority fill:#6bcf7f
    style MarkComplete fill:#51cf66
```
