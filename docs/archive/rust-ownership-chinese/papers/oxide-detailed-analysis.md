## 1. 论文概述

### 核心贡献

Oxide提出了**首个接近源代码级别的Rust形式化模型**，将Rust的借用检查器重新表述为**声明式子结构类型系统**。

**关键创新：**

- 使用`shrd`/`uniq`而非`&`/`&mut`，强调别名语义而非可变性
- 将生命周期视为**来源集（provenance）**而非代码行抽象
- 声明式借用检查（而非算法描述）

---

## 2. 核心语言定义

### 2.1 语法

```rust
// Oxide语法（接近Rust源码）

// 类型
τ ::= bool | u32 | () | τ → τ | Box<τ> | &ω π τ

// 权限 ω
ω ::= shrd | uniq

// 位置 π
π ::= x | π.f | π[i]

// 表达式
e ::= x | n | () | true | false
  | let x : τ = e in e
  | &ω π
  | *e
  | e.f
  | e(e)
  | Box::new(e)
```

### 2.2 权限（Permissions）

| 权限 | 含义 | 别名规则 |
|------|------|----------|
| `shrd` | 共享 | 多个共存，只读 |
| `uniq` | 唯一 | 独占访问，读写 |

**与Rust的对比：**

```rust
// Rust
let r1 = &x;       // &T
let r2 = &mut x;   // &mut T

// Oxide
let r1 = &shrd x;  // shrd强调别名语义
let r2 = &uniq x;  // uniq强调别名语义
```

## 3. 来源集（Provenance）

### 3.1 概念

Oxide将生命周期视为**来源集**——引用所依赖的内存位置的集合。

```text
传统视角：
  'a = "从第5行到第10行"

Oxide视角：
  'a = { 位置ℓ₁, 位置ℓ₂, ... }
```

### 3.2 来源集近似

```text
精确来源集 = 引用实际指向的所有内存位置
来源集近似 = 程序分析推断的位置集合（保守超集）

Rust借用检查器使用近似来源集
```

## 4. 类型系统

### 4.1 贷款（Loans）

```text
贷款 ℓ ::= ωp

其中：
  ω = shrd | uniq   (权限)
  p = 位置           (内存位置)
```

### 4.2 所有权安全性

```text
判断规则：

[OWN-SAFE]
对于位置p以ω方式访问：
  - 如果ω = uniq：没有活跃贷款覆盖p
  - 如果ω = shrd：没有uniq贷款覆盖p
```

### 4.3 类型判断

```text
环境：
  Σ : 存储类型
  Δ : 区域环境
  Γ : 类型环境（带贷款集合）
  Θ : 贷款环境

判断形式：
  Σ; Δ; Γ; Θ ⊢ e : τ ⇒ Γ'; Θ'

输出环境Γ'表示表达式后的状态
```

## 5. 与Rust的比较

### 5.1 抽象级别

| 特性 | Oxide | Rust | RustBelt |
|------|-------|------|----------|
| 抽象级别 | 接近源码 | 源码 | CPS核心语言 |
| 借用检查 | 声明式 | 算法式 | 分离逻辑 |
| 生命周期 | 来源集 | 区域 | 逻辑关系 |
| 验证器 | 参考实现 | rustc | Coq |

### 5.2 表达能力

Oxide覆盖：

- 基础借用
- 生命周期参数
- 函数类型
- Box
- 元组

Oxide不覆盖：

- 内部可变性
- 并发原语
- Unsafe代码

## 7. 理论意义

### 7.1 声明式vs算法式

```text
算法式（Rustc）：
  "按以下步骤检查借用..."

声明式（Oxide）：
  "借用有效当且仅当满足这些条件..."
```

声明式的优势：

- 更清晰的安全条件
- 更容易推理
- 更易于形式化证明

### 7.2 来源集视角

来源集视角揭示了：

- 生命周期的本质是内存依赖
- NLL的优化空间
- 借用检查的核心是别名分析

## 9. 总结

Oxide的贡献：

1. **声明式借用检查**：清晰的数学定义
2. **来源集生命周期**：新的理解视角
3. **接近源码**：更容易理解和使用
4. **验证基础**：为后续研究提供基础

---
