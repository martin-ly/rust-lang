## 设计规则
>
> **[来源: Rust Official Docs]**

### 规则 1: 维度必须正交或有意耦合

> **[来源: Wikipedia - Asynchronous I/O]**

好的维度选择：

- 内存安全、数据竞争防护、空指针安全（**有意耦合**：都是安全属性维度）
- 编译时检查、运行时开销、学习曲线（**有意耦合**：都是 trade-off 维度）

坏的维度选择：

- "是否好用"、"社区活跃"、"文档质量"（过于主观，缺乏可验证标准）

### 规则 2: 每个单元格必须有判定标准

> **[来源: Wikipedia - Rust (programming language)]**

| 不合格 | 合格 |
|--------|------|
| "好" / "差" | "编译器静态保证" / "运行时 GC" / "手动管理" |
| "快" / "慢" | "零成本抽象" / "单次间接调用" / "全局锁" |

## 常用矩阵类型

### 类型 1: 语言特性对比矩阵

| 维度 | Rust | C++ | Go | Java |
|------|------|-----|-----|------|
| **内存安全机制** | 所有权+借用 | RAII+智能指针 | GC | GC |
| **空指针安全** | Option<T> | std::optional | nil+panic | NullPointerException |
| **数据竞争防护** | 编译期(Send/Sync) | 无 | 无 | happens-before |
| **运行时开销** | 零成本 | 零成本 | GC暂停 | GC暂停 |

### 类型 2: 工具/库选择矩阵

| 维度 | Tokio | async-std | smol | embassy |
|------|-------|-----------|------|---------|
| **运行时模型** | 多线程 work-stealing | 多线程 | 小型模块化 | 嵌入式单线程 |
| **生态成熟度** | ⭐⭐⭐ | ⭐⭐ | ⭐ | ⭐⭐ |
| **适用场景** | 服务端 | 通用 | 轻量/定制 | no_std 嵌入式 |

### 类型 3: 概念正交性矩阵

| | 泛型 | 生命周期 | Trait | 宏 |
|---|------|----------|-------|-----|
| **泛型** | - | 正交 | 正交 | 正交 |
| **生命周期** | 耦合(GAT) | - | 耦合(trait bound) | 正交 |
| **Trait** | 耦合(关联类型) | 耦合 | - | 正交 |
| **宏** | 正交 | 正交 | 正交 | - |

## 进阶：条件矩阵（Conditional Matrix）

当某个属性的取值依赖于条件时，使用条件矩阵：

| 条件 \ 实体 | Rust | C++ |
|-------------|------|-----|
| **默认情况** | 编译期保证 | 无保证 |
| **使用 unsafe** | 程序员负责 | 程序员负责 |
| **使用 std only** | 保证 | 依赖实现 |
