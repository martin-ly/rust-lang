# Rust Crates 文档统一标准规范

**版本**: v2.0
**制定日期**: 2025-11-15
**适用版本**: Rust 1.91.1+
**适用范围**: 所有 crates 文档
**维护者**: 文档团队

---

## 📋 文档定位

本文档定义了所有 Rust crates 文档的统一标准，确保文档结构、格式、术语和版本信息的一致性。

---

## 🎯 核心原则

### 1. 结构一致性原则

**所有 crates 的 README.md 必须遵循统一结构**:

1. **文档头部** - 元数据信息
2. **模块概述** - 简要介绍
3. **快速开始** - 入门指南
4. **核心内容** - 主要功能
5. **文档导航** - 文档索引
6. **学习路径** - 推荐路径
7. **项目状态** - 完成度信息

### 2. 版本信息一致性

**所有文档必须明确标注**:

- **Rust 版本**: Rust 1.91.1+
- **更新日期**: 2025-11-15
- **Edition**: 2024

### 3. 术语一致性

**统一术语使用**:

- **Tier 1-4**: 使用 "Tier 1: 基础层"、"Tier 2: 实践层"、"Tier 3: 参考层"、"Tier 4: 高级层"
- **文档类型**: 使用统一的图标和名称
- **状态标识**: 使用统一的 emoji 标识

## 🔢 序号格式标准

### 目录序号格式

**统一使用中文序号**:

- 一级标题: `## 1. [标题]` 或 `## 📋 [标题]` (使用图标时)
- 二级标题: `### 1.1 [标题]` 或 `### 1.1.1 [标题]`
- 三级标题: `#### 1.1.1 [标题]`

**示例**:

```markdown
## 1. 基础概念
### 1.1 所有权系统
#### 1.1.1 所有权规则
```

### Tier 文档序号格式

**统一使用两位数字序号**:

- Tier 1: `01_项目概览.md`, `02_主索引导航.md`, `03_术语表.md`, `04_常见问题.md`
- Tier 2: `01_[主题]指南.md`, `02_[主题]指南.md`, ...
- Tier 3: `01_[主题]参考.md`, `02_[主题]参考.md`, ...
- Tier 4: `01_[主题]深入.md`, `02_[主题]深入.md`, ...

## 🔧 更新流程

### 版本更新流程

1. **更新版本信息**
   - 更新所有 README.md 中的 Rust 版本
   - 更新所有文档中的版本标注
   - 更新更新日期为当前日期

2. **检查术语一致性**
   - 检查所有文档的术语使用
   - 统一 Tier 命名
   - 统一文档类型命名

3. **检查格式一致性**
   - 检查所有文档的序号格式
   - 检查所有文档的目录结构
   - 检查所有文档的链接格式

4. **验证链接有效性**
   - 检查所有内部链接
   - 检查所有外部链接
   - 修复无效链接
