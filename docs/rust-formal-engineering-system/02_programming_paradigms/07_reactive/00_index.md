## 目的

- 明确响应式编程范式在 Rust 中的定位与适用场景。
- 统一响应式编程相关术语，建立与其他编程范式的映射关系。
- 汇总仓库中与响应式编程相关的示例与深度文章。

---

## 术语

- **响应式编程（Reactive Programming）**: 基于数据流和变化传播的编程范式。
- **流（Stream）**: 随时间变化的数据序列。
- **观察者模式（Observer Pattern）**: 对象间一对多的依赖关系。
- **背压（Backpressure）**: 处理数据流速度不匹配的机制。

---

## 核心概念

- **数据流**: 通过流处理异步数据。
- **组合操作**: 通过组合操作符处理流。
- **错误处理**: 流中的错误传播和处理。
- **背压管理**: 控制数据流速度。

## 设计建议

- 使用 Stream 处理异步数据流。
- 合理使用背压机制。
- 组合操作符简化流处理。
- 正确处理流中的错误。

---

## 常见陷阱

- 忽略背压导致内存问题。
- 错误处理不当。
- 流组合过于复杂。
