## 目的

- 明确并发编程范式在 Rust 中的定位与适用场景。
- 统一并发编程相关术语，建立与其他编程范式的映射关系。
- 汇总仓库中与并发编程相关的示例与深度文章。

---

## 术语

- **并发（Concurrency）**: 多个任务交替执行，看起来同时进行。
- **并行（Parallelism）**: 多个任务真正同时执行。
- **数据竞争（Data Race）**: 多个线程同时访问同一数据，至少一个写入。
- **同步原语（Synchronization Primitive）**: 用于协调并发访问的机制。

---

## 核心概念

- **线程安全**: 通过类型系统保证线程安全。
- **消息传递**: 通过 channel 进行线程间通信。
- **共享状态**: 通过 `Arc`、`Mutex` 等管理共享状态。
- **无锁编程**: 使用原子操作和无锁数据结构。

## 设计建议

- 优先使用消息传递而非共享状态。
- 使用类型系统保证线程安全。
- 避免数据竞争。
- 合理使用同步原语。

---

## 常见陷阱

- 死锁：多个锁的获取顺序不一致。
- 数据竞争：未正确同步的共享访问。
- 线程爆炸：创建过多线程。
