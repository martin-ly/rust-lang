## 术语

- **面向对象编程（Object-Oriented Programming）**: 以对象为中心，通过封装、继承、多态组织代码。
- **Trait**: Rust 中实现多态的主要机制，类似于接口。
- **结构体（Struct）**: 数据容器，可以包含方法和关联函数。
- **枚举（Enum）**: 可以包含数据的类型，支持模式匹配。
- **实现（Impl）**: 为类型实现方法和关联函数。

---

## 核心概念

- **封装**: 通过 `pub` 关键字控制可见性。
- **多态**: 通过 Trait 实现多态。
- **组合优于继承**: Rust 不支持传统继承，鼓励使用组合。
- **类型安全**: 编译时保证类型正确性。

## 设计建议

- 使用 Trait 实现多态。
- 优先使用组合而非继承。
- 利用枚举实现状态机。
- 通过结构体组织相关数据。

---

## 常见陷阱

- 试图在 Rust 中实现传统继承。
- 过度使用 `dyn Trait` 导致性能损失。
- 忽略 Trait 对象的大小限制。
