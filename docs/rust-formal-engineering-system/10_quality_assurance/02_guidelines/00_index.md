## 目的

- 提供 Rust 项目质量保障的实践指南与最佳实践。
- 建立代码质量、测试质量、文档质量的统一标准。

## 代码质量指南

- 代码风格：遵循 `rustfmt` 标准，使用 `clippy` 检查
- 命名规范：使用有意义的变量名和函数名
- 注释规范：为公共 API 提供文档注释
- 错误处理：使用 `Result` 和 `Option` 进行错误处理
- 性能考虑：避免不必要的分配和拷贝

## 测试质量指南

- 测试覆盖：核心功能必须有测试覆盖
- 测试分类：单元测试、集成测试、端到端测试
- 测试数据：使用工厂模式生成测试数据
- 测试隔离：每个测试独立运行，不依赖外部状态
- 属性测试：使用 `proptest` 进行基于属性的测试

## 文档质量指南

- API 文档：为所有公共 API 提供文档注释
- 示例代码：提供可运行的示例代码
- 变更日志：维护详细的 CHANGELOG
- 贡献指南：提供清晰的贡献指南
