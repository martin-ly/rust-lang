## 核心理念

本模块旨在阐明 Rust 如何通过其独特的所有权系统和 `Send`/`Sync` Trait，将传统的、充满危险的并发编程，转变为一种编译器保驾护航的、"无畏"的工程实践。

**核心保证**：

1. **内存安全**：通过所有权系统防止数据竞争
2. **类型安全**：通过类型系统保证线程安全
3. **零成本抽象**：并发抽象不带来运行时开销

**形式化表示**：

$$
\text{ConcurrencySafe}(\text{program}) \iff \text{OwnershipSystem} \land \text{SendSync} \implies \neg \exists \text{DataRace}
$$
