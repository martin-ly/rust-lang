## 🔬 核心概念

### 所有权三定律

```rust
1. 唯一性：每个值有且只有一个所有者
2. 排他性：值只能有一个可变借用或多个不可变借用
3. 生命周期绑定：引用不能比被引用的值活得更久
```

### 可判定性四层次

| 层次 | 判定能力 | 示例 | Rust策略 |
|------|----------|------|----------|
| **完全可判定** | 编译期精确判定 | 类型匹配、所有权转移 | 编译期强制 |
| **近似可判定** | 编译期保守近似 | NLL借用检查 | 宁可误报 |
| **运行时判定** | 运行时检查 | RefCell、Rc | panic安全失败 |
| **理论不可判定** | 无法判定 | 程序终止性 | 无支持 |

## 📋 核心发现

1. **判定边界清晰分层**：完全可判定 → 近似可判定 → 运行时判定 → 理论不可判定

2. **静态与动态无冲突**：静态层提供基础保证，动态层提供灵活性，动态检查失败时安全panic

3. **理论不可判定性无处不在**：Halting Problem和Rice's Theorem设定了理论上限
