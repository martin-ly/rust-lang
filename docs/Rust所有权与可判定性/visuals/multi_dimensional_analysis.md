## 1. 形式化公理系统

### 1.1 Rust所有权公理

```text
┌─────────────────────────────────────────────────────────────────────┐
│                    所有权系统公理 (Axioms)                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  Axiom 1 (唯一性):                                                  │
│  ∀v ∈ Value. owns(v) → ¬∃v'. owns(v') ∧ v ≠ v' ∧ mem(v) = mem(v')  │
│                                                                     │
│  Axiom 2 (可转移性):                                                 │
│  ∀v. owns(x, v) ∧ move(x, y) → owns(y, v) ∧ ¬owns(x, v)            │
│                                                                     │
│  Axiom 3 (借用不变性):                                               │
│  ∀v. borrow(v) → ¬mutate(v)                                        │
│                                                                     │
│  Axiom 4 (生命周期包含):                                             │
│  ∀'a, 'b. 'a: 'b → scope('a) ⊃ scope('b)                           │
│                                                                     │
│  Axiom 5 (Drop语义):                                                │
│  ∀v. drop(v) → resource_free(mem(v))                               │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## 2. 定理与证明树

### 2.1 内存安全性定理

```text
定理 1 (内存安全):
─────────────────────────────────────────────────────────
对于所有良类型程序 P,
如果 P 在 Rust 类型系统中类型良好,
那么 P 的执行不会出现:
  1. 悬垂指针解引用
  2. 缓冲区溢出
  3. 数据竞争
─────────────────────────────────────────────────────────

证明结构 (Proof Tree):
══════════════════════════════════════════════════════════

                    Γ ⊢ P : unit
                         │
         ┌───────────────┼───────────────┐
         │               │               │
    所有权保持      借用保持        生命周期保持
         │               │               │
         ▼               ▼               ▼
    ┌─────────┐     ┌─────────┐     ┌─────────┐
    │ 唯一性  │     │ 无冲突  │     │ 有效性  │
    │ 保持    │     │ 保证    │     │ 验证    │
    └────┬────┘     └────┬────┘     └────┬────┘
         │               │               │
         └───────────────┴───────────────┘
                         │
                         ▼
                ┌─────────────────┐
                │   内存安全性    │
                │   得证 ✓        │
                └─────────────────┘
```

### 2.2 类型安全性定理

```text
定理 2 (类型安全 - Progress + Preservation):
══════════════════════════════════════════════════════════

Progress (进度定理):
─────────────────────────────────────────────────────────
如果 Γ ⊢ e : T, 那么要么:
  1. e 是一个值 v
  2. 存在 e' 使得 e → e'

Preservation (保持定理):
─────────────────────────────────────────────────────────
如果 Γ ⊢ e : T 且 e → e', 那么 Γ ⊢ e' : T

证明树:
         Γ ⊢ e : T
              │
      ┌───────┴───────┐
      │               │
   e = v (值)      e → e'
      │               │
      ▼               ▼
  终止          Γ ⊢ e' : T
                      │
                      ▼
                 继续推导
```

### 3.2 所有权转移推理

```text
所有权转移推理树:
═══════════════════════════════════════════════════════════

场景: let s2 = s1;  // String 所有权转移

初始状态:
Γ = { s1: String }

推理步骤:

Step 1: 检查 s1 的有效性
─────────────────────────────────────
Γ ⊢ s1 : String        String 不是 Copy
─────────────────────────────────────
Γ ⊢ s1 可以被移动

Step 2: 执行转移
─────────────────────────────────────
Γ ⊢ s1 : String        s1 有效
─────────────────────────────────────
Γ' = Γ \ {s1} ∪ {s2: String}

Step 3: 验证 s1 失效
─────────────────────────────────────
s1 ∉ dom(Γ')
─────────────────────────────────────
后续使用 s1 会导致编译错误

最终状态:
Γ' = { s2: String }    // s1 已从环境中移除
```

## 6. 实例推理树

### 6.1 Vec::push 操作推理

```text
Vec::push 操作的形式化推理:
═══════════════════════════════════════════════════════════════════════

代码: let mut v = vec![1, 2, 3]; v.push(4);

Step 1: 创建 Vec
─────────────────────────────────────────
Γ ⊢ vec![1, 2, 3] : Vec<i32>
Γ' = Γ, v: Vec<i32>
v 拥有堆内存 [1, 2, 3]

Step 2: 可变借用 v
─────────────────────────────────────────
v: Vec<i32> ∈ Γ'
v ∉ borrowed(Γ')
─────────────────────────────────────────
Γ' ⊢ &mut v : &mut Vec<i32>

Step 3: 调用 push
─────────────────────────────────────────
push(&mut self, elem: T) 需要 &mut Vec<T>
Γ' ⊢ &mut v : &mut Vec<i32>
Γ' ⊢ 4 : i32
─────────────────────────────────────────
Γ' ⊢ v.push(4) : ()

Step 4: 借用结束
─────────────────────────────────────────
push 返回后，&mut v 生命周期结束
v 的独占借用释放

Step 5: 验证安全性
─────────────────────────────────────────
• 无悬垂指针: 原内存可能被重新分配，但所有引用已失效
• 无溢出: Vec 自动扩容
• 无数据竞争: push 期间独占访问
─────────────────────────────────────────
✓ 操作安全
```

### 6.2 多线程数据共享推理

```text
多线程数据共享推理:
═══════════════════════════════════════════════════════════════════════

代码:
  let data = Arc::new(Mutex::new(0));
  thread::spawn(move || { *data.lock().unwrap() += 1; });

Step 1: 创建共享数据
─────────────────────────────────────────
Γ ⊢ 0 : i32
Γ ⊢ Mutex::new(0) : Mutex<i32>
Γ ⊢ Arc::new(Mutex::new(0)) : Arc<Mutex<i32>>

Step 2: 检查 Send/Sync
─────────────────────────────────────────
i32: Send + Sync
Mutex<T>: Send + Sync if T: Send
Arc<T>: Send + Sync if T: Send + Sync
─────────────────────────────────────────
Arc<Mutex<i32>>: Send + Sync ✓

Step 3: 跨线程移动
─────────────────────────────────────────
move || { ... } 捕获 Arc
Arc 实现 Send
─────────────────────────────────────────
可以安全跨线程移动

Step 4: 运行时借用检查
─────────────────────────────────────────
data.lock() 尝试获取 Mutex
• 如果可用: 返回 MutexGuard
• 如果已被持有: 阻塞等待

Step 5: 修改数据
─────────────────────────────────────────
MutexGuard 实现 DerefMut
*guard += 1 解引用后修改

Step 6: 释放锁
─────────────────────────────────────────
MutexGuard 在作用域结束时 Drop
自动释放 Mutex

安全保证:
• 编译期: Send/Sync 确保类型安全
• 运行时: Mutex 确保互斥访问
• 无数据竞争 ✓
```
