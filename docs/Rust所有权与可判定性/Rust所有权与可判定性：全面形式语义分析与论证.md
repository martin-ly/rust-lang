## 第一部分：概念定义与形式化基础

### 1.1 核心概念精确定义

#### 定义 1.1.1：所有权（Ownership）

**直观定义**：所有权是一种资源管理策略，确保每个值在任意时刻有且只有一个所有者。

**形式化定义**：设 $\mathcal{V}$ 为值域，$\mathcal{L}$ 为位置（内存地址）域，$\mathcal{T}$ 为时间（程序点）域。

$$\text{Own}: \mathcal{V} \times \mathcal{L} \times \mathcal{T} \to \{\top, \bot\}$$

$\text{Own}(v, \ell, t) = \top$ 当且仅当在时刻 $t$，值 $v$ 存储在位置 $\ell$ 且被唯一所有者持有。

**所有权三定律**：

1. **唯一性**：$\forall t, v: \exists! \ell. \text{Own}(v, \ell, t) = \top$
2. **排他性**：$\forall t, \ell: \exists! v. \text{Own}(v, \ell, t) = \top$
3. **生命周期绑定**：$\text{Own}(v, \ell, t) = \top \land t' > t \land \neg\text{Own}(v, \ell, t') \Rightarrow \text{Drop}(v, t')$

#### 定义 1.1.2：借用（Borrowing）

**直观定义**：借用是临时访问值的引用，不改变所有权，但受严格规则约束。

**形式化定义**：设 $\mathcal{R}$ 为引用类型集合，$\omega \in \{\text{shrd}, \text{uniq}\}$ 为借用权限（共享/唯一）：

$$\text{Borrow}: \mathcal{V} \times \mathcal{R} \times \mathcal{T} \times \mathcal{T} \to \{\top, \bot\}$$

$\text{Borrow}(v, r, t_s, t_e) = \top$ 表示值 $v$ 在时间区间 $[t_s, t_e]$ 被引用 $r$ 借用。

**借用约束条件**：

1. **共享借用（&T）**：
   $$\forall r_1, r_2: \text{Borrow}(v, r_1, t_s, t_e) \land \text{Borrow}(v, r_2, t_s, t_e) \Rightarrow \text{perm}(r_1) = \text{shrd} \land \text{perm}(r_2) = \text{shrd}$$

2. **可变借用（&mut T）**：
   $$\text{Borrow}(v, r, t_s, t_e) \land \text{perm}(r) = \text{uniq} \Rightarrow \neg\exists r'. \text{Borrow}(v, r', t_s, t_e) \land r' \neq r$$

#### 定义 1.1.3：生命周期（Lifetime）

**直观定义**：生命周期是引用有效的程序执行区间，Rust使用区域（region）来近似表示。

**形式化定义**（基于Oxide）：生命周期是程序点（控制流图节点）的集合：

$$\text{Lifetime} = \mathcal{P}(\mathcal{CFG})$$

其中 $\mathcal{CFG}$ 是控制流图节点集合。

**子类型关系（Outlives）**：
$$'a: 'b \Leftrightarrow 'b \subseteq 'a$$

即 $'a$ outlives $'b$ 当且仅当 $'a$ 包含的程序点集合是 $'b$ 的超集。

#### 定义 1.1.4：仿射类型（Affine Types）

**直观定义**：仿射类型要求值最多使用一次（可以使用零次或一次）。

**形式化定义**：类型 $T$ 是仿射的，如果对于任意值 $v: T$：

$$\text{use}(v, t_1) \land \text{use}(v, t_2) \Rightarrow t_1 = t_2$$

**Rust中的仿射类型**：

- `String`、`Vec<T>`、自定义非`Copy`类型都是仿射的
- `Copy`类型（如`i32`、`bool`）不是仿射的（可多次使用）

#### 定义 1.1.5：可判定性（Decidability）

**形式化定义**（基于计算理论）：

语言 $L$ 是**可判定的**，如果存在图灵机 $M$ 使得：
$$\forall w: M(w) \text{ 停机并接受} \Leftrightarrow w \in L$$
$$M(w) \text{ 停机并拒绝} \Leftrightarrow w \notin L$$

**半可判定（识别）**：$M$ 在 $w \in L$ 时停机接受，在 $w \notin L$ 时可能不停机。

**不可判定**：不存在这样的图灵机。

## 第二部分：形式语义与公理系统

### 2.1 操作语义完整定义

#### 2.1.1 配置（Configuration）

Rust程序的执行状态由以下配置表示：

$$\langle e, \sigma, \pi \rangle$$

其中：

- $e$：待求值的表达式
- $\sigma: \mathcal{L} \rightharpoonup \mathcal{V}$：存储（部分函数，将位置映射到值）
- $\pi: \mathcal{P}(\mathcal{L} \times \mathcal{L})$：权限关系（记录借用关系）

#### 2.1.2 所有权转移语义

**[MOVE]**

```text
────────────────────────────────────────────────────────
⟨let x = v; e, σ, π⟩ → ⟨e[x/v], σ, π⟩    (v: !Copy)
```

**[COPY]**

```text
────────────────────────────────────────────────────────
⟨let x = v; e, σ, π⟩ → ⟨e[x/v'], σ, π⟩   (v: Copy, v' = copy(v))
```

**[DROP]**

```text
σ(ℓ) = v    v: !Copy    v's owner goes out of scope
────────────────────────────────────────────────────────
⟨drop(ℓ), σ, π⟩ → ⟨(), σ[ℓ ↦ ⊥], π⟩
```

#### 2.1.3 借用语义

**[BORROW-SHARED]**

```text
σ(x) = ℓ    ℓ is valid    no &mut borrow active on ℓ
────────────────────────────────────────────────────────
⟨&x, σ, π⟩ → ⟨ℓ^{shrd}, σ, π ∪ {(ℓ, ℓ^{shrd})}⟩
```

**[BORROW-MUT]**

```text
σ(x) = ℓ    ℓ is valid    no borrow active on ℓ
────────────────────────────────────────────────────────
⟨&mut x, σ, π⟩ → ⟨ℓ^{uniq}, σ[x ↦ ⊥], π ∪ {(ℓ, ℓ^{uniq})}⟩
```

#### 2.1.4 分离逻辑视角

基于RustBelt，我们使用分离逻辑（Separation Logic）来形式化所有权：

**所有权谓词**（Ownership Predicate）：

$$[T].\text{own}(t, \vec{v})$$

表示在线程 $t$ 中，值 $\vec{v}$ 构成一个合法的 $T$ 类型值。

**基本类型的所有权**：

- 整数：$[\texttt{i32}].\text{own}(t, n) \triangleq n \in \mathbb{Z}_{32}$
- 元组：$[\tau_1 \times \tau_2].\text{own}(t, \vec{v}_1 + \vec{v}_2) \triangleq [\tau_1].\text{own}(t, \vec{v}_1) * [\tau_2].\text{own}(t, \vec{v}_2)$

**可变引用的所有权**（RustBelt）：

$$[\&_{\text{mut}}^{\kappa} \tau].\text{own}(t, \ell) \triangleq \exists \vec{v}. \ell \mapsto \vec{v} * [\tau].\text{own}(t, \vec{v}) * \text{borrowed}^{\kappa}(\ell)$$

### 2.2 公理系统：Rust类型系统的推理规则

#### 2.2.1 环境

**类型环境**：$\Gamma ::= \emptyset \mid \Gamma, x: T$

**生命周期环境**：$\Delta ::= \emptyset \mid \Delta, 'a$

**借用环境**：$\Theta ::= \emptyset \mid \Theta, \omega p$

#### 2.2.2 所有权推理规则

**[OWN-MOVE]**

```text
Γ ⊢ e : T    T: !Copy    Γ' = Γ ⊘ e
────────────────────────────────────────
Γ ⊢ let x = e; P ⊣ Γ', x: T
```

其中 $\Gamma \oslash e$ 表示从环境中移除 $e$ 使用的资源。

**[OWN-COPY]**

```text
Γ ⊢ e : T    T: Copy
────────────────────────────────────────
Γ ⊢ let x = e; P ⊣ Γ, x: T
```

#### 2.2.3 借用推理规则

**[BORROW-SHARED]**

```text
Γ ⊢ p : T    p is valid    ¬(p mutably borrowed in Γ)
────────────────────────────────────────────────────────
Γ ⊢ &p : &T ⊣ Γ with p immutably borrowed
```

**[BORROW-MUT]**

```text
Γ ⊢ p : T    p is valid    ¬(p borrowed in Γ)
────────────────────────────────────────────────────────
Γ ⊢ &mut p : &mut T ⊣ Γ with p mutably borrowed (frozen)
```

#### 2.2.4 生命周期推理规则

**[LIFETIME-OUTLIVES]**

```text
'a : 'b    Γ ⊢ e : &'a T
────────────────────────────────────────
Γ ⊢ e : &'b T
```

**[LIFETIME-STATIC]**

```text
────────────────────────────────────────
'static : 'a    (for all 'a)
```

### 2.3 元定理与形式证明

#### 定理 2.3.1：类型安全性（Type Safety）

**定理陈述**：如果 $\Gamma \vdash e : T$ 且 $\langle e, \sigma_0, \pi_0 \rangle \to^* \langle v, \sigma, \pi \rangle$，则：

1. **进度（Progress）**：$v$ 是值，或者存在 $e'$ 使得 $\langle v, \sigma, \pi \rangle \to \langle e', \sigma', \pi' \rangle$

2. **保持（Preservation）**：$\sigma \models \Gamma$ 且 $\Gamma \vdash v : T$

**证明概要**（基于Oxide）：

**引理 1（替换引理）**：如果 $\Gamma, x: T_1 \vdash e : T_2$ 且 $\Gamma \vdash v : T_1$，则 $\Gamma \vdash e[x/v] : T_2$。

*证明*：通过对推导树的高度进行归纳。□

**引理 2（规范形式）**：如果 $\Gamma \vdash v : T$ 且 $v$ 是值，则：

- 若 $T = \texttt{i32}$，则 $v = n$（整数）
- 若 $T = \&T'$，则 $v = \ell$（位置）
- 若 $T = T_1 \times T_2$，则 $v = (v_1, v_2)$

*证明*：通过对类型 $T$ 的结构归纳。□

**主定理证明**：通过对求值步数进行归纳，结合引理1和引理2。□

#### 定理 2.3.2：借用安全性（Borrow Safety）

**定理陈述**：在程序执行过程中，以下不变式始终成立：

$$\text{BS}(\sigma, \pi) \triangleq \forall \ell, \ell'. (\ell, \ell') \in \pi \Rightarrow \sigma(\ell) \neq \bot$$

即：活跃借用所指向的内存始终有效。

**证明概要**：

通过对操作语义规则的案例分析：

1. **[MOVE]**：原变量失效，但借用关系 $\pi$ 不变，借用仍指向有效内存。

2. **[DROP]**：仅当没有活跃借用时才能执行（由借用检查器保证）。

3. **[BORROW-SHARED]**：检查无活跃可变借用，确保共享借用安全。

4. **[BORROW-MUT]**：检查无任何活跃借用，确保可变借用独占访问。

□

#### 定理 2.3.3：内存安全性（Memory Safety）

**定理陈述**：对于所有通过Rust编译器检查的Safe Rust程序 $P$，$P$ 不会出现：

1. **悬垂指针**（Dangling Pointers）
2. **双重释放**（Double Free）
3. **数据竞争**（Data Races）
4. **使用未初始化内存**（Use of Uninitialized Memory）

**证明概要**：

由定理2.3.1（类型安全）和定理2.3.2（借用安全）直接推出：

- 类型安全确保值被正确使用
- 借用安全确保内存有效性
- 所有权系统确保正确的释放时机
- `Send`/`Sync` trait确保线程安全

□

#### 定理 2.3.4：静态分析的保守性（Conservatism）

**定理陈述**：Rust借用检查器是**可靠但不完备**的（Sound but Incomplete）：

$$\forall P: \text{Compiles}(P) \Rightarrow \text{Safe}(P)$$
$$\exists P: \text{Safe}(P) \land \neg\text{Compiles}(P)$$

**证明概要**：

**可靠性**：由定理2.3.3直接得出。

**不完备性**：构造反例：

```rust
fn conservative_reject(flag: bool) {
    let mut x = 0;
    let mut y = 1;
    let p = if flag { &mut x } else { &mut y };
    // p只指向x或y之一，但编译器拒绝以下代码：
    // x = 2; // 编译错误：即使flag为false时安全
}
```

在合并点，编译器保守地假设 $p$ 可能指向 $x$ 或 $y$，因此拒绝后续对 $x$ 和 $y$ 的访问。□

    match self.borrow.get() {
        UNUSED => {
            self.borrow.set(1);  // 第一个共享借用
        }
        n if n > 0 => {
            self.borrow.set(n + 1);  // 增加计数
        }
        _ => panic!("already mutably borrowed"),  // 运行时panic
    }
    // ...
}

### 3.4 理论不可判定边界

#### 3.4.1 Halting Problem

**定理（Turing, 1936）**：不存在算法可以判定任意程序 $P$ 在给定输入 $I$ 下是否会终止。

**Rust中的体现**：

```rust
fn collatz(n: u64) {
    let mut n = n;
    while n != 1 {
        if n % 2 == 0 {
            n /= 2;
        } else {
            n = 3 * n + 1;  // Collatz猜想未解决
        }
    }
}
```

编译器无法确定此函数是否对所有输入都会终止。

#### 3.4.2 Rice's Theorem

**定理（Rice, 1953）**：对于任何非平凡的语义属性 $P$，判定"程序 $Q$ 是否具有属性 $P$"是不可判定的。

**非平凡属性**：既不是所有程序都有，也不是所有程序都没有的属性。

**应用**：以下属性均不可判定：

- 程序是否存在运行时panic
- 程序是否存在内存泄漏
- 程序功能是否符合规范

#### 3.4.3 指针别名分析的复杂性

**定理（Landi-Ryder, 1992）**：精确的指针别名分析是 **PSPACE-hard** 的，在一般情况下是不可判定的。

**Rust的策略**：

1. Safe Rust：通过借用检查器完全禁止别名
2. Unsafe Rust：将责任交给程序员
3. Miri：运行时检测别名违规

## 第五部分：应用实例与反例

### 5.1 正例：可静态验证的代码模式

#### 正例 1：所有权转移的正确使用

```rust
// ✅ 可静态验证：所有权唯一性
fn ownership_transfer() {
    let s = String::from("hello");
    let t = s;  // 所有权从s转移到t
    // s不再有效，编译器保证不会误用
    println!("{}", t);  // ✅ OK
    // println!("{}", s);  // ❌ 编译错误：borrow of moved value
}
```

**验证依据**：

- [OWN-MOVE] 规则保证转移后原变量失效
- 线性分析跟踪每个值的使用路径

#### 正例 2：借用的正确模式

```rust
// ✅ 可静态验证：共享借用与可变借用的互斥
fn borrowing_patterns() {
    let mut data = vec![1, 2, 3];

    // 多个不可变借用
    let r1 = &data;
    let r2 = &data;
    println!("{} {}", r1[0], r2[0]);  // ✅ OK

    // r1, r2 在此之后不再使用
    // NLL允许此处开始新的借用
    let r3 = &mut data;  // ✅ OK: 不可变借用已结束
    r3.push(4);
}
```

**验证依据**：

- [BORROW-SHARED] 允许多个共享借用
- NLL分析确定借用的最后使用点
- [BORROW-MUT] 确保可变借用独占

#### 正例 3：类型状态模式

```rust
// ✅ 可静态验证：编译期状态机
struct Uninitialized;
struct Initialized;
struct Running;

struct Process<State> {
    name: String,
    _state: std::marker::PhantomData<State>,
}

impl Process<Uninitialized> {
    fn new(name: &str) -> Self {
        Self {
            name: name.to_string(),
            _state: std::marker::PhantomData,
        }
    }

    fn initialize(self) -> Process<Initialized> {
        Process {
            name: self.name,
            _state: std::marker::PhantomData,
        }
    }
}

impl Process<Initialized> {
    fn start(self) -> Process<Running> {
        Process {
            name: self.name,
            _state: std::marker::PhantomData,
        }
    }
}

fn typestate_demo() {
    let p = Process::new("worker");
    // p.start();  // ❌ 编译错误：Uninitialized状态没有start方法
    let p = p.initialize();
    let p = p.start();  // ✅ OK
}
```

**验证依据**：

- 泛型参数编码状态
- 编译期方法可用性检查
- 状态转换通过所有权转移实现

### 5.2 反例：编译器拒绝的安全代码

#### 反例 1：条件借用保守拒绝

```rust
// ❌ 编译器拒绝，但实际上安全
fn conditional_borrow_rejected(flag: bool) {
    let mut x = 0;
    let mut y = 1;

    let p = if flag {
        &mut x
    } else {
        &mut y
    };

    // 编译器保守地假设p可能指向x或y
    // x = 2;  // ❌ 编译错误：即使flag为false时安全
}
```

**分析**：

- 路径敏感性限制：编译器在控制流合并点保守合并类型信息
- $p$ 的类型变为 `&mut {x, y}`，导致对 $x$ 和 $y$ 的访问被拒绝
- 这是 **可靠性（Soundness）** 与 **完备性（Completeness）** 权衡的结果

#### 反例 2：循环中的借用

```rust
// ❌ 编译器拒绝，需要更复杂的模式
fn loop_borrow_rejected() {
    let mut data = vec![1, 2, 3];
    let mut iter = data.iter_mut();

    // while let Some(item) = iter.next() {
    //     data.push(*item);  // ❌ 编译错误：data已被可变借用
    // }
}
```

**分析**：

- 借用检查器无法确定 `iter` 和 `data` 的生命周期关系
- 虽然某些迭代可能安全，但编译器保守拒绝
- 解决方案：使用 `std::mem::take` 或重新设计数据结构

### 5.3 边界案例：不可判定场景

#### 边界案例 1：程序终止性

```rust
// 不可判定：是否对所有输入都会终止
fn collatz_conjecture(n: u64) {
    let mut n = n;
    while n != 1 {
        if n % 2 == 0 {
            n /= 2;
        } else {
            n = 3 * n + 1;
        }
    }
}
```

**理论依据**：

- Collatz猜想尚未被证明或证伪
- 由 **Halting Problem** 不可判定性，不存在通用算法可判定任意程序的终止性

#### 边界案例 2：功能正确性

```rust
// 不可判定：是否正确实现了排序
fn my_sort<T: Ord>(data: &mut [T]) {
    // 某种排序实现
    unimplemented!()
}

// 正确性要求：
// 1. 输出是有序的
// 2. 输出是输入的排列
```

**理论依据**：

- 判定程序是否满足规范是语义属性
- 由 **Rice's Theorem**，非平凡语义属性不可判定
- 需要形式化验证（Prusti/Creusot）或测试
