## 1. 正例库：可静态验证的代码模式

### 1.1 所有权基础模式

#### 模式 1.1.1：RAII资源管理

```rust
use std::fs::File;
use std::io::{Write, Result};

/// ✅ 完全静态可验证的RAII模式
/// 文件在作用域结束时自动关闭，无资源泄漏风险
fn write_to_file(data: &str) -> Result<()> {
    // File::create 返回文件的所有权
    let mut file = File::create("output.txt")?;

    // file被可变借用用于写入
    file.write_all(data.as_bytes())?;

    // file在此处自动drop，资源被释放
    Ok(())
}

/// ✅ 早期返回仍然安全
fn conditional_write(data: &str, condition: bool) -> Result<()> {
    let mut file = File::create("output.txt")?;

    if condition {
        file.write_all(data.as_bytes())?;
        // 此处返回，file仍然正确drop
        return Ok(());
    }

    // 正常结束，file正确drop
    Ok(())
}
```

**可判定性分析**：

- 所有权跟踪：✅ 完全可判定（线性分析）
- 资源释放点：✅ 编译期确定（作用域结束）
- 异常安全：✅ 析构函数保证

## 2. 反例库：编译器拒绝的安全代码

### 2.1 路径敏感性限制

#### 反例 2.1.1：条件借用保守拒绝

```rust
/// ❌ 编译器拒绝，但实际上安全
fn conditional_borrow_conservative(flag: bool) {
    let mut x = 0;
    let mut y = 1;

    let p = if flag {
        &mut x
    } else {
        &mut y
    };

    // p只指向x或y之一，但编译器在合并点保守假设p可能指向两者
    // x = 2;  // ❌ 编译错误: cannot assign to `x` because it is borrowed
    // y = 3;  // ❌ 编译错误: cannot assign to `y` because it is borrowed

    println!("{}", p);  // 使用p
}

/// ✅  workaround: 使用Polonius（实验性）或重构代码
fn conditional_borrow_workaround(flag: bool) {
    let mut x = 0;
    let mut y = 1;

    if flag {
        let p = &mut x;
        *p = 2;
        // p在此处结束
    } else {
        let p = &mut y;
        *p = 3;
        // p在此处结束
    }

    // 现在x和y都可以访问
    println!("x = {}, y = {}", x, y);
}
```

**理论分析**：

- 路径敏感分析等价于符号执行
- 精确分析需要指数级空间（路径爆炸）
- Rust采用保守合并策略，牺牲完备性换取可靠性

#### 反例 2.1.2：循环内的借用

```rust
/// ❌ 编译器无法验证的循环借用模式
fn loop_borrow_rejected() {
    let mut data = vec![1, 2, 3, 4, 5];

    // 编译器无法确定在每次迭代中data和item的关系
    // for item in &mut data {
    //     data.push(*item);  // ❌ 编译错误: cannot borrow `data` as mutable more than once
    // }
}

/// ✅ workaround: 使用临时存储
fn loop_borrow_workaround() {
    let mut data = vec![1, 2, 3, 4, 5];
    let to_add: Vec<i32> = data.iter().copied().collect();

    for item in to_add {
        data.push(item);  // ✅ OK
    }
}
```

### 2.2 自引用结构

#### 反例 2.2.1：经典自引用问题

```rust
/// ❌ 无法直接创建的自引用结构
struct SelfReferential {
    data: String,
    // ptr: &str,  // 指向data内部的引用
}

/// ❌ 以下代码不安全
fn self_referential_problem() {
    let mut s = SelfReferential {
        data: String::from("hello"),
        // ptr: &s.data[..],  // 无法创建：s尚未完全初始化
    };

    // 如果允许，移动s会导致ptr悬垂
    // let t = s;  // 移动后ptr指向无效内存
}

/// ✅ workaround: 使用Pin和unsafe
use std::pin::Pin;
use std::marker::PhantomPinned;

struct PinnedSelfRef {
    data: String,
    ptr: *const str,  // 原始指针，unsafe
    _pin: PhantomPinned,  // 禁止Unpin
}

impl PinnedSelfRef {
    fn new(data: String) -> Pin<Box<Self>> {
        let mut boxed = Box::new(Self {
            data,
            ptr: std::ptr::null(),
            _pin: PhantomPinned,
        });

        let ptr = &boxed.data[..] as *const str;
        boxed.ptr = ptr;

        Pin::from(boxed)
    }

    fn get_ref(&self) -> &str {
        unsafe { &*self.ptr }  // unsafe解引用
    }
}
```

## 3. 边界案例：理论不可判定场景

### 3.1 程序终止性

#### 案例 3.1.1：Collatz猜想

```rust
/// ❓ 是否对所有n都会终止？（未解决的数学问题）
fn collatz(n: u64) {
    let mut n = n;
    while n != 1 {
        if n % 2 == 0 {
            n /= 2;
        } else {
            n = 3 * n + 1;
        }
    }
}

/// ❓ 包含Collatz的函数
fn process_with_collatz(data: &mut [u64]) {
    for item in data.iter_mut() {
        collatz(*item);
        *item += 1;
    }
}
```

**理论限制**：

- 由 **Halting Problem** 不可判定性
- 不存在通用算法可判定任意程序是否终止
- Rust编译器无法在编译期确定此函数的行为

### 3.2 语义属性

#### 案例 3.2.1：功能正确性

```rust
/// ❓ 是否正确实现了排序？（不可静态判定）
fn my_sort<T: Ord>(data: &mut [T]) {
    // 某种排序实现
    // 即使编译通过，也不能保证正确性
}

/// ❓ 是否满足以下性质？
/// 1. 输出有序: ∀i. output[i] ≤ output[i+1]
/// 2. 排列关系: output是input的排列
fn verify_sort_properties() {
    let mut data = vec![3, 1, 4, 1, 5];
    my_sort(&mut data);

    // 需要运行时验证或形式化证明
    assert!(is_sorted(&data));
    assert!(is_permutation(&data, &[3, 1, 4, 1, 5]));
}

fn is_sorted<T: Ord>(data: &[T]) -> bool {
    data.windows(2).all(|w| w[0] <= w[1])
}

fn is_permutation<T: Ord>(a: &[T], b: &[T]) -> bool {
    let mut a_sorted = a.to_vec();
    let mut b_sorted = b.to_vec();
    a_sorted.sort();
    b_sorted.sort();
    a_sorted == b_sorted
}
```

**理论限制**：

- 由 **Rice's Theorem**，非平凡语义属性不可判定
- 功能正确性是典型的语义属性

## 4. 运行时判定示例

### 4.1 RefCell动态借用检查

```rust
use std::cell::RefCell;

/// ✅ 运行时借用检查
fn refcell_runtime_check() {
    let data = RefCell::new(vec![1, 2, 3]);

    // 获取不可变借用
    let borrow1 = data.borrow();
    let borrow2 = data.borrow();  // ✅ OK: 多个不可变借用
    println!("{:?}, {:?}", borrow1, borrow2);
    drop(borrow1);
    drop(borrow2);

    // 获取可变借用
    let mut borrow3 = data.borrow_mut();
    borrow3.push(4);  // ✅ OK

    // let borrow4 = data.borrow();  // ❌ 运行时panic: already mutably borrowed
}

/// ✅ 动态借用状态机
describe_refcell_state_machine! {
    States: Unused → Shared(n) → Exclusive

    borrow():
        Unused → Shared(1)
        Shared(n) → Shared(n+1)
        Exclusive → panic!("already mutably borrowed")

    borrow_mut():
        Unused → Exclusive
        Shared(n) → panic!("already immutably borrowed")
        Exclusive → panic!("already mutably borrowed")

    drop(Ref):
        Shared(1) → Unused
        Shared(n) → Shared(n-1)
        Exclusive → Unused
}
```

### 4.2 Rc引用计数

```rust
use std::rc::Rc;

/// ✅ 运行时引用计数
fn rc_runtime_counting() {
    let data = Rc::new(vec![1, 2, 3]);
    println!("初始引用计数: {}", Rc::strong_count(&data));  // 1

    {
        let clone1 = Rc::clone(&data);
        println!("clone后计数: {}", Rc::strong_count(&data));  // 2

        {
            let clone2 = Rc::clone(&data);
            println!("再次clone后: {}", Rc::strong_count(&data));  // 3

            // clone2在此处drop，计数减1
        }
        println!("clone2 drop后: {}", Rc::strong_count(&data));  // 2

        // clone1在此处drop，计数减1
    }
    println!("clone1 drop后: {}", Rc::strong_count(&data));  // 1

    // data在此处drop，计数归零，内存释放
}

/// ⚠️ 循环引用导致内存泄漏
fn rc_cycle_leak() {
    use std::cell::RefCell;

    struct Node {
        value: i32,
        next: RefCell<Option<Rc<Node>>>,
    }

    let node1 = Rc::new(Node {
        value: 1,
        next: RefCell::new(None),
    });

    let node2 = Rc::new(Node {
        value: 2,
        next: RefCell::new(Some(Rc::clone(&node1))),
    });

    // 创建循环引用
    *node1.next.borrow_mut() = Some(Rc::clone(&node2));

    // node1和node2的引用计数都是2
    // 即使它们离开作用域，计数也不会归零
    // 内存泄漏发生
}
```
