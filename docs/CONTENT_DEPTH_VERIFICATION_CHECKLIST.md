# 内容深度验证检查清单

> **创建日期**: 2026-02-28
> **用途**: 系统化检查所有文档的实质内容质量
> **适用范围**: 全项目文档 (crates/, docs/, guides/)

---

## 📋 使用说明

1. **逐文档检查**: 对每个文档应用此清单
2. **标记状态**: ✅ 通过, ⚠️ 需改进, ❌ 缺失
3. **记录问题**: 在问题追踪中记录具体缺失
4. **迭代改进**: 根据检查结果完善内容

## 自动化检查工具

### 命令行检查

```bash
# 检查 Markdown 链接
./scripts/check_links.ps1

# 统计代码示例数量
grep -r "^\`\`\`rust" crates/ docs/ | wc -l

# 检查 TODO/FIXME
grep -r "TODO\|FIXME\|XXX" crates/*/docs/ docs/

# 检查文档大小（小于 100 行可能内容不足）
find crates -name "*.md" -exec wc -l {} + | sort -n
```

### 质量评分脚本

```bash
#!/bin/bash
# quality_check.sh

doc=$1
echo "检查文档: $doc"

echo "1. 检查形式化定义:"
grep -c "^Def [0-9]" "$doc" || echo "  无形式化定义"

echo "2. 检查定理:"
grep -c "^定理\|^Thm" "$doc" || echo "  无定理"

echo "3. 检查代码示例:"
grep -c "^\`\`\`rust" "$doc" || echo "  无代码示例"

echo "4. 检查外部链接:"
grep -c "https://" "$doc" || echo "  无外部链接"

echo "5. 文档总行数:"
wc -l < "$doc"
```
