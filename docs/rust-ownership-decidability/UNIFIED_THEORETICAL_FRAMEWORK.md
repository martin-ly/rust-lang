### 1.3 核心数学构造

#### 归纳定义 (Inductive Definitions)

**在我们的工作中的应用**:

```coq
(* 类型: 归纳定义 *)
Inductive ty : Type :=
  | TBase : base_ty -> ty
  | TRef : region -> mutability -> ty -> ty
  | ...

(* 表达式: 归纳定义 *)
Inductive expr : Type :=
  | EValue : value -> expr
  | EVar : var -> expr
  | ...

(* 类型判断: 归纳关系 *)
Inductive has_type :
  region_env -> type_env -> loan_env -> expr -> ty -> Prop :=
  | T_Value : ...
  | T_Var : ...
  | ...
```

**为什么使用归纳定义**:

1. **有限表示**: 无限集合的有限描述
2. **结构递归**: 自然支持结构归纳证明
3. **构造性**: 与 Coq 的计算模型匹配

#### 偏序关系 (Partial Orders)

**生命周期包含**:

```text
'static ⊇ 'a ⊇ 'b  (如果 'a 比 'b 长)

形式化:
  subregion : region -> region -> Prop

性质:
  - 自反性: ∀ρ, subregion ρ ρ
  - 传递性: ρ₁ ⊆ ρ₂ ∧ ρ₂ ⊆ ρ₃ → ρ₁ ⊆ ρ₃
  - 反对称性: ρ₁ ⊆ ρ₂ ∧ ρ₂ ⊆ ρ₁ → ρ₁ = ρ₂
```

**良基关系 (Well-founded Relations)**:

```text
ty_rank : ty -> nat

用于终止性证明:
  ∀τ₁ τ₂, τ₁ 包含在 τ₂ 中 → ty_rank τ₁ < ty_rank τ₂
```

### 2.3 判断体系统一描述

#### 判断的层次结构

```text
Level 0: 基础判断
  ├─ τ wf                 (类型良构)
  ├─ Γ wf                 (环境良构)
  └─ σ wf                 (状态良构)

Level 1: 语法判断
  ├─ Γ ⊢ e : τ            (类型检查)
  ├─ ⊢ω p ok              (所有权安全)
  └─ Δ ⊢ ρ wf             (区域良构)

Level 2: 语义判断
  ├─ σ ⊢ e ⇓ v            (大步求值)
  ├─ ⟨σ, e⟩ → ⟨σ', e'⟩    (小步求值)
  └─ σ ⊢ p ↝ ℓ            (位置求值)

Level 3: 元判断
  ├─ safe(Γ, e)           (程序安全)
  ├─ terminates(e)        (终止性)
  └─ decidable(P)         (可判定性)
```

#### 判断间的逻辑关系

```coq
(* 语法正确性 → 类型正确性 → 语义正确性 *)
Theorem syntax_to_type_to_semantics :
  forall e,
    well_formed_expr e ->
    (exists Γ τ, has_type Γ e τ) ->
    (forall σ, exists v, eval σ e v).

(* 类型正确 + 所有权安全 → 内存安全 *)
Theorem type_and_ownership_implies_memory_safety :
  forall Γ e τ,
    has_type Γ e τ ->
    ownership_safe_program Γ e ->
    memory_safe e.
```

## 第五部分: 统一证明方法论

### 5.1 证明模式库

#### 模式 1: 结构归纳法

**适用**: 语法定义的命题

```coq
(* 模板 *)
Theorem property_by_structural_induction :
  forall e, P(e).
Proof.
  induction e; intros.
  - (* 基本情况 1: EValue *)
    simpl. auto.
  - (* 基本情况 2: EVar *)
    simpl. auto.
  - (* 归纳情况: ESeq *)
    simpl. apply IHe1. apply IHe2.
  - (* 其他情况... *)
Qed.
```

**在我们的工作中使用**:

- `preservation` - 按表达式结构归纳
- `progress` - 按类型判断归纳

#### 模式 2: 良基归纳法

**适用**: 带有度量函数的终止性证明

```coq
(* 模板 *)
Theorem property_by_well_founded_induction :
  forall x, P(x).
Proof.
  remember (measure x) as n.
  generalize dependent x.
  induction n using lt_wf_ind.
  intros x Hmeasure.
  (* 证明 P(x) *)
  (* 使用 H: forall y, measure y < measure x -> P(y) *)
Qed.
```

**在我们的工作中使用**:

- `borrow_checking_termination` - 按类型秩归纳

#### 模式 3: 反演 (Inversion)

**适用**: 从已知判断推导信息

```coq
(* 模板 *)
Theorem property_by_inversion :
  forall Γ e τ, has_type Γ e τ -> P(Γ, e, τ).
Proof.
  intros Γ e τ Htype.
  inversion Htype; subst; clear Htype.
  - (* 情况 T_Value *)
    ...
  - (* 情况 T_Var *)
    ...
Qed.
```

**在我们的工作中使用**:

- 所有证明中都大量使用

#### 模式 4: 矛盾法

**适用**: 证明否定命题

```coq
(* 模板 *)
Theorem property_by_contradiction :
  forall x, ~P(x).
Proof.
  intros x H.
  (* 从 H 推导出矛盾 *)
  contradiction.
Qed.
```

**在我们的工作中使用**:

- `linearizable_acyclic` - 证明无环性

### 5.2 自动化策略

#### 层次化自动化

```coq
(* Level 1: 基本简化 *)
Ltac basic_simpl :=
  simpl; intros; try subst; auto.

(* Level 2: 上下文处理 *)
Ltac ctx_simpl :=
  basic_simpl;
  repeat match goal with
  | H: context[te_lookup _ _] |- _ => simpl in H
  end.

(* Level 3: 证明搜索 *)
Ltac auto_proof :=
  ctx_simpl;
  eauto using ty_rank_nonneg, linearizable_rank_decreasing.

(* Level 4: 完整自动化 *)
Ltac full_auto :=
  auto_proof;
  try solve [induction e; auto_proof].
```
