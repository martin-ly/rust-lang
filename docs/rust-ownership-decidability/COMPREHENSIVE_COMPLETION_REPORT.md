## 二、关联性论证的核心内容

### 2.1 横向关联论证（HORIZONTAL_CONNECTIONS.md）

**核心贡献**：

**概念关联网络**：

```text
所有权 ←→ 借用 ←→ 生命周期
   ↑______________↑

三角关系论证：
- 所有权是独占访问权（空间）
- 借用是临时访问权（权限）
- 生命周期是访问有效期（时间）
- 三者组合产生涌现性质（零成本内存安全）
```

**定理关联网络**：

```text
数学基础
    ↓
Linearizability（枢纽）
    ↓
终止性 ──→ 可判定性
    ↓
保持性 + 进展 ──→ 类型安全
    ↓
借用检查等价性 ──→ 内存安全
```

**证明技术关联**：

```text
结构归纳 + 良基归纳 + 反证法 + 构造性证明
         ↓
    协同证明保持性、进展等定理
```

### 2.2 历史演化论证（HISTORICAL_EVOLUTION.md）

**核心贡献**：

**时间线**：

```text
1970s-1990s: C语言手动管理 → GC兴起 → 线性逻辑
2006-2015: Rust诞生 → 所有权系统演化 → 1.0发布
2015-2020: RustBelt → Featherweight Rust → Oxide
2024-2026: 本工作（整合与发展）
```

**概念演化**：

```text
所有权：C++ RAII → C++11 move → Rust所有权 → 线性类型形式化
借用：C指针 → C++引用 → Rust借用 → 仿射类型形式化
生命周期：Region inference → Cyclone → Rust生命周期 → 区域类型形式化
```

**历史定位**：

```text
本工作在演化中的位置：
- 承前：整合 Featherweight Rust、Oxide、RustBelt
- 启后：为unsafe形式化、并发形式化奠定基础
- 传播：让更多人理解Rust的理论基础
```

### 2.3 比较分析论证（COMPARATIVE_ANALYSIS.md）

**核心贡献**：

**与函数式语言对比**：

```text
ML：多态类型，GC
Haskell：纯函数，Monad副作用控制
Rust：所有权系统，编译时副作用控制

联系：Rust的Trait借鉴Haskell的Type Class
创新：用所有权替代Monad进行副作用控制
```

**与命令式语言对比**：

```text
C/C++：手动管理，约定式所有权
Java：GC，共享引用
Rust：编译器强制所有权，零成本

演化：从约定到编译器强制
```

**与学术语言对比**：

```text
Cyclone：区域类型（直接影响Rust）
ATS：依赖类型+线性类型（太强，不实用）
Mezzo：能力类型（与Rust借用相似）

Rust的位置：表达能力高 + 实用性高
```

### 2.4 设计决策原理论证（DESIGN_RATIONALE.md）

**核心贡献**：

**整体设计哲学**：

```text
可理解性 > 完备性
教学性 > 前沿性
实用性 > 理论性
```

**语法设计原理**：

```text
EValue vs EVar：区分常量和变量
EBorrow包含lifetime：生命周期是引用核心
ELet包含mutability：可变性是绑定的一部分
```

**语义设计原理**：

```text
大步+小步：互补，证明等价性
三层环境：关注点分离（Δ/Γ/Θ）
操作语义：直观，适合结构归纳
```

**定理设计原理**：

```text
五个核心定理：Safety + Liveness + Decidability
层次化依赖：避免循环，保证坚固
不追求完备：Rust故意不完备（保守）
```
