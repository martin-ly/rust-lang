## 引言

Rust 的类型系统是建立在 Hindley-Milner (HM) 类型推断基础上的高级扩展系统。
它通过引入所有权、生命周期和 Trait 系统，在保持类型推断能力的同时，提供了强大的内存安全保证。

### 形式化目标

我们旨在形式化 Rust 类型系统的以下方面：

- **类型推断**：完整的 HM 算法及其扩展
- **多态性**：参数多态、限定多态、高阶多态
- **约束系统**：类型约束的生成与求解
- **Rust 特有扩展**：生命周期参数、Trait 约束

### 数学符号约定

```
τ, σ    ::= 类型变量 | 函数类型 | 构造器类型
α, β, γ ::= 类型变量（小写希腊字母）
Γ       ::= 类型环境（typing context）
⊢       ::= 推导关系（turnstile）
≻       ::= 实例化关系
∀       ::= 全称量词（universal quantification）
```

## HM 类型推断系统

### 2.1 语法定义

#### 2.1.1 类型语法

```
τ ::= α              (类型变量)
   |  ι              (基本类型: Int, Bool, Unit, ...)
   |  τ₁ → τ₂        (函数类型)
   |  (τ₁, ..., τₙ)  (元组类型)
   |  [τ]            (数组类型)
   |  τ⟨α⟩           (参数化类型)
```

#### 2.1.2 表达式语法

```
e ::= x              (变量)
   |  λx.e           (抽象)
   |  e₁ e₂          (应用)
   |  let x = e₁ in e₂  (let 绑定)
   |  c              (常量)
```

### 2.2 类型规则

#### 2.2.1 变量规则 (T-Var)

```
─────────────  (x : τ) ∈ Γ
Γ ⊢ x : τ
```

变量规则表明：如果变量 x 在类型环境 Γ 中具有类型 τ，则表达式 x 的类型为 τ。

#### 2.2.2 常量规则 (T-Const)

```
─────────────  typeof(c) = τ
Γ ⊢ c : τ
```

常量规则使用内置的类型函数 typeof 确定常量的类型。

#### 2.2.3 抽象规则 (T-Abs)

```
Γ, x : τ₁ ⊢ e : τ₂
────────────────────
Γ ⊢ λx.e : τ₁ → τ₂
```

抽象规则处理 λ 抽象：如果假设参数 x 具有类型 τ₁ 时，函数体 e 具有类型 τ₂，则整个 λ 抽象具有函数类型 τ₁ → τ₂。

#### 2.2.4 应用规则 (T-App)

```
Γ ⊢ e₁ : τ₁ → τ₂    Γ ⊢ e₂ : τ₁
────────────────────────────────
      Γ ⊢ e₁ e₂ : τ₂
```

应用规则是函数调用的核心：被调用函数必须具有函数类型，且实参类型必须与形参类型一致。

#### 2.2.5 Let 绑定规则 (T-Let)

```
Γ ⊢ e₁ : τ₁    Γ, x : Gen(Γ, τ₁) ⊢ e₂ : τ₂
────────────────────────────────────────────
     Γ ⊢ let x = e₁ in e₂ : τ₂
```

Let 绑定规则引入了多态类型，通过泛化操作 Gen 将类型变量泛化为多态类型。

### 2.3 泛化与实例化

#### 2.3.1 自由类型变量

```
FTV(α)          = {α}
FTV(ι)          = ∅
FTV(τ₁ → τ₂)    = FTV(τ₁) ∪ FTV(τ₂)
FTV((τ₁,...,τₙ))= ⋃ᵢ FTV(τᵢ)
FTV(∀α.τ)       = FTV(τ) \ {α}
```

#### 2.3.2 泛化操作 (Generalization)

```
Gen(Γ, τ) = ∀α₁...∀αₙ.τ
其中 {α₁, ..., αₙ} = FTV(τ) \ FTV(Γ)
```

泛化操作找出类型 τ 中未在环境 Γ 中出现的自由类型变量，并将它们量化为全称量词。

#### 2.3.3 实例化关系 (Instantiation)

```
τ[α₁ ↦ τ₁, ..., αₙ ↦ τₙ] ≺ ∀α₁...∀αₙ.τ
```

实例化是多态类型的具体化，通过替换类型变量得到具体类型。

### 2.4 算法 W

算法 W 是 HM 类型推断的经典算法，它是一个完整的、多态的类型推断算法。

#### 2.4.1 统一算法 (Unification)

统一算法找出使两个类型相等的替换（substitution）。

```ocaml
unify(τ₁, τ₂) = S  where S(τ₁) = S(τ₂)
```

统一规则：

```
unify(α, τ) = [α ↦ τ]           if α ∉ FTV(τ)
unify(τ, α) = [α ↦ τ]           if α ∉ FTV(τ)
unify(ι, ι) = id
unify(τ₁₁ → τ₁₂, τ₂₁ → τ₂₂) =
    let S₁ = unify(τ₁₁, τ₂₁) in
    let S₂ = unify(S₁(τ₁₂), S₁(τ₂₂)) in
    S₂ ∘ S₁
unify((τ₁₁,...,τ₁ₙ), (τ₂₁,...,τ₂ₙ)) =
    unify_components([(τ₁₁,τ₂₁), ..., (τ₁ₙ,τ₂ₙ)])
```

#### 2.4.2 类型推断算法

```
W(Γ, x) = (id, τ)           where (x : ∀α₁...αₙ.τ₀) ∈ Γ
          and τ = τ₀[α₁↦β₁, ..., αₙ↦βₙ] (fresh βᵢ)

W(Γ, c) = (id, typeof(c))

W(Γ, λx.e) =
    let β = fresh() in
    let (S₁, τ₁) = W(Γ, x:β, e) in
    (S₁, S₁(β) → τ₁)

W(Γ, e₁ e₂) =
    let (S₁, τ₁) = W(Γ, e₁) in
    let (S₂, τ₂) = W(S₁(Γ), e₂) in
    let β = fresh() in
    let S₃ = unify(S₂(τ₁), τ₂ → β) in
    (S₃ ∘ S₂ ∘ S₁, S₃(β))

W(Γ, let x = e₁ in e₂) =
    let (S₁, τ₁) = W(Γ, e₁) in
    let τ₁' = Gen(S₁(Γ), τ₁) in
    let (S₂, τ₂) = W(S₁(Γ), x:τ₁', e₂) in
    (S₂ ∘ S₁, τ₂)
```

### 2.5 类型推断的正确性

#### 2.5.1 健全性 (Soundness)

**定理 2.1 (算法 W 的健全性)**：
如果 W(Γ, e) = (S, τ)，则 S(Γ) ⊢ e : τ。

*证明概要*：通过对 e 的结构归纳。

- 基本情况：变量和常量直接由类型规则得出
- 归纳步骤：每种表达式形式对应相应的类型规则

#### 2.5.2 完备性 (Completeness)

**定理 2.2 (算法 W 的完备性)**：
如果存在替换 S' 和类型 τ' 使得 S'(Γ) ⊢ e : τ'，
则 W(Γ, e) = (S, τ) 成功，且存在替换 R 使得 τ' = R(S(τ))。

*证明概要*：通过对推导深度的归纳，证明算法 W 计算最一般的统一子 (mgu)。

#### 2.5.3 主类型 (Principal Typing)

**定理 2.3 (主类型性质)**：
对于任何类型环境 Γ 和表达式 e，如果 e 在 Γ 下可类型化，
则存在主类型 τ，使得对于任何其他可派生类型 τ'，都有 τ' ≻ τ。

## 多态类型系统

### 3.1 System F (多态 λ 演算)

System F 是带有显式类型抽象和类型应用的 λ 演算。

#### 3.1.1 语法扩展

```
τ ::= ... | ∀α.τ        (全称类型)
e ::= ... | Λα.e        (类型抽象)
   |  e[τ]              (类型应用)
```

#### 3.1.2 类型规则扩展

**类型抽象规则 (T-TAbs)**：

```
Γ, α ⊢ e : τ
───────────────
Γ ⊢ Λα.e : ∀α.τ
```

**类型应用规则 (T-TApp)**：

```
Γ ⊢ e : ∀α.τ
───────────────
Γ ⊢ e[τ'] : τ[α ↦ τ']
```

### 3.2 限定多态 (Qualified Types)

Rust 的 Trait 系统对应于类型类（Type Classes）或限定多态。

#### 3.2.1 限定类型语法

```
π ::= τ₁ : Trait₁      (限定谓词)
Q ::= ε | π, Q         (限定集合)
ρ ::= τ | Q ⇒ ρ        (限定类型)
σ ::= ρ | ∀α.σ         (多态限定类型)
```

#### 3.2.2 限定类型规则

**限定引入规则 (⇒-Intro)**：

```
Γ, π ⊢ e : ρ
──────────────
Γ ⊢ e : π ⇒ ρ
```

**限定消去规则 (⇒-Elim)**：

```
Γ ⊢ e : π ⇒ ρ    Γ ⊢ π
────────────────────────
      Γ ⊢ e : ρ
```

### 3.3 Rust 的 Trait 形式化

#### 3.3.1 Trait 定义

```
TraitDef ::= trait Trait<ᾱ> where Q̄ {
               type Assoc;
               fn method(self, ...) -> τ;
             }
```

形式化表示为：

```
Trait ::= (Name, Params, SuperTraits, AssociatedTypes, Methods)
```

#### 3.3.2 Trait 实现

```
Impl ::= impl<ᾱ> Trait<τ̄> for τ₀ where Q̄ {
           type Assoc = τ;
           fn method(self, ...) -> τ { ... }
         }
```

形式化约束：

```
Γ ⊢ impl Trait for τ where Q
要求：
1. τ 满足 Trait 的所有超类约束
2. 关联类型的实现一致
3. 方法签名满足协变/逆变规则
```

### 3.4 高阶多态 (Higher-Ranked Polymorphism)

Rust 支持高阶 Trait 约束（Higher-Ranked Trait Bounds, HRTB）：

```rust
fn foo<F>() where F: for<'a> Fn(&'a T) -> &'a U
```

形式化：

```
∀F. (∀α. F(α)) → τ[F]
```

#### 3.4.1 HRTB 类型规则

```
Γ, ∀α ⊢ e : τ
───────────────────
Γ ⊢ e : ∀α.τ
```

## 约束求解

### 4.1 约束生成

类型推断过程中生成约束集合。

#### 4.1.1 约束类型

```
Constraint ::= τ₁ ≡ τ₂           (类型相等)
            |  τ : Trait         (Trait 约束)
            |  α : Region        (生命周期约束)
            |  Constraint ∧ Constraint
            |  ∃α.Constraint     (存在约束)
            |  ∀α.Constraint     (全称约束)
```

#### 4.1.2 约束生成算法

```
C(Γ, x) = ∅                          (返回值: (C, τ))
        返回 τ where (x:τ) ∈ Γ

C(Γ, λx.e) =
    let α = fresh() in
    let (C, τ) = C(Γ, x:α, e) in
    (C, α → τ)

C(Γ, e₁ e₂) =
    let (C₁, τ₁) = C(Γ, e₁) in
    let (C₂, τ₂) = C(Γ, e₂) in
    let α = fresh() in
    (C₁ ∧ C₂ ∧ τ₁ ≡ τ₂ → α, α)

C(Γ, let x = e₁ in e₂) =
    let (C₁, τ₁) = C(Γ, e₁) in
    let ᾱ = FTV(τ₁) \ FTV(C₁, Γ) in
    let (C₂, τ₂) = C(Γ, x:∀ᾱ.τ₁, e₂) in
    (C₁ ∧ C₂, τ₂)
```

### 4.2 约束求解算法

#### 4.2.1 统一约束求解

```
solve(τ₁ ≡ τ₂) = unify(τ₁, τ₂)

solve((C₁ ∧ C₂)) =
    let S₁ = solve(C₁) in
    let S₂ = solve(S₁(C₂)) in
    S₂ ∘ S₁

solve(∃α.C) =
    let S = solve(C) in
    S|_{FV(C)\{α}}
```

#### 4.2.2 Trait 约束求解

Rust 使用 Chalk 风格的基于逻辑的求解器：

```
solve(τ : Trait) =
    查找 impl Trait for τ' where Q
    如果 ∃σ. σ(τ') = τ
    则递归求解 σ(Q)
```

### 4.3 约束图与环检测

#### 4.3.1 约束图表示

```
G = (V, E)
V = {τ | τ 是类型或类型变量}
E = {(τ₁, τ₂) | τ₁ ≡ τ₂ ∈ C} ∪ {(τ, Trait) | τ : Trait ∈ C}
```

#### 4.3.2 环检测算法

使用 Tarjan 强连通分量算法检测循环约束：

```
SCC(G):
    1. 对 G 进行 DFS，记录访问顺序
    2. 在转置图 G^T 上按完成时间降序 DFS
    3. 每次 DFS 得到的树即为一个 SCC
```

## Rust 类型系统扩展

### 5.1 生命周期参数

#### 5.1.1 生命周期语法

```
ℓ ::= 'static | 'a | 'b | ...  (生命周期变量)
τ ::= ... | &'ℓ τ | &'ℓ mut τ   (引用类型)
```

#### 5.1.2 生命周期约束

```
Constraint ::= ... | ℓ₁ : ℓ₂    (生命周期包含)
```

**包含规则**：

```
Γ ⊢ ℓ₁ : ℓ₂   (读作 ℓ₁ 至少和 ℓ₂ 一样长)
```

#### 5.1.3 引用类型规则

**共享引用规则 (T-SharedRef)**：

```
Γ ⊢ e : τ    Γ ⊢ ℓ_valid
────────────────────────
Γ ⊢ &ℓ e : &ℓ τ
```

**可变引用规则 (T-MutRef)**：

```
Γ ⊢ e : τ    Γ ⊢ ℓ_valid    unique(e)
────────────────────────────────────
    Γ ⊢ &ℓ mut e : &ℓ mut τ
```

### 5.2 所有权类型

#### 5.2.1 所有权修饰

```
ownership ::= Own | Borrow(ℓ) | MutBorrow(ℓ)
```

#### 5.2.2 移动语义

**移动规则 (T-Move)**：

```
Γ ⊢ e : τ^Own
────────────────
Γ ⊢ move e : τ^Own
Γ' = Γ \ {e}  (从环境移除)
```

#### 5.2.3 复制语义

**复制规则 (T-Copy)**：

```
Γ ⊢ e : τ^Copy
────────────────
Γ ⊢ e : τ^Copy
Γ' = Γ  (环境不变)
```

### 5.3 关联类型

#### 5.3.1 关联类型语法

```
trait Iterator {
    type Item;
    fn next(&mut self) -> Option<Self::Item>;
}
```

形式化：

```
Trait Iterator = (Item: Type, next: &mut Self → Option<Item>)
```

#### 5.3.2 关联类型投影

```
<Self as Iterator>::Item ≡ τ
```

归一化规则：

```
如果 impl Iterator for T { type Item = U; }
则 <T as Iterator>::Item ~> U
```

## 证明概要

### 6.1 类型安全性定理

**定理 6.1 (类型安全性)**：
如果 ⊢ e : τ，则：

1. **进展性 (Progress)**：e 是值，或存在 e' 使得 e → e'
2. **保持性 (Preservation)**：如果 e → e'，则 ⊢ e' : τ

#### 6.1.1 进展性证明

*证明*：对推导 ⊢ e : τ 进行结构归纳。

**基本情况**：

- e 是常量 c：已是值
- e 是变量 x：在闭式程序中不可能

**归纳情况**：

- e = e₁ e₂：
  - 由归纳假设，e₁ 是值或可归约
  - 若 e₁ = λx.e'，则应用 β 归约
  - 否则先归约 e₁

#### 6.1.2 保持性证明

*证明*：对归约关系 → 进行情况分析。

**β 归约**：

```
(λx.e) v → e[x ↦ v]
```

需证：如果 Γ ⊢ (λx.e) v : τ，则 Γ ⊢ e[x ↦ v] : τ。

由抽象规则和应用规则可得。

### 6.2 类型推断的终止性

**定理 6.2 (算法 W 终止性)**：
对于任何有限的 Γ 和 e，W(Γ, e) 在有限步内终止。

*证明*：

1. 每次递归调用使表达式结构变小
2. 统一算法基于 Robinson 算法，保证终止
3. 递归深度受表达式大小限制

### 6.3 约束求解的完备性

**定理 6.3 (约束求解完备性)**：
如果约束集合 C 有解，则 solve(C) 返回最一般的解。

*证明*：

1. 统一约束使用 mgu 算法，保证完备
2. Trait 约束通过穷举 impl 匹配求解
3. 使用回溯处理不确定性
