### 内部可变性：运行时借用检查

```text
RefCell<T> 的核心机制:

编译时: &T 允许访问
运行时: 检查借用规则

借用状态机:
┌─────────┐    borrow()    ┌───────────┐
│ Unused  │ ─────────────→ │ Reading(1)│
└────┬────┘                └─────┬─────┘
     │                          │
     │ borrow_mut()            │ borrow()
     ▼                          ▼
┌─────────┐                ┌───────────┐
│ Writing │                │ Reading(n)│
└─────────┘                └───────────┘

冲突检测:
• 当前状态 = Writing，调用 borrow() → panic
• 当前状态 = Reading，调用 borrow_mut() → panic
• 当前状态 ≠ Unused，调用 borrow_mut() → panic
```

## 理论完善

### 可判定性论证

```text
定理: Rust借用检查（不包括泛型约束）是可判定的。

但: 以下问题编译时不可判定:
1. 引用计数何时变为0？
   → 需要运行时计数器

2. 运行时条件决定的借用模式？
   → 需要RefCell运行时检查

3. 锁的获取顺序是否死锁？
   → 需要运行时调度

证明概要:
根据停机问题，不存在算法可以在编译时判定任意程序
的运行时资源使用模式。
```

### 形式化语义扩展

```text
RefCell<T> 的分离逻辑表示:

RefCell(x) ≡ x ↦ (state, value)

where state ∈ { Unused, Reading(n), Writing }

运行时检查作为公理:
{RefCell(x) * x.state = Writing} borrow() {panic}

这与编译时借用检查的对比:
编译时: Γ ⊢ &T 要求 T: Freeze
运行时: RefCell 检查状态标志
```
