## 1. 背景介绍

### 1.1 什么是分离逻辑

分离逻辑(Separation Logic)是一种用于推理**共享可变状态**的程序逻辑。由John Reynolds (2002) 和 Peter O'Hearn 等人发展，最初用于验证指针操作程序，现在广泛应用于：

- **Rust**: 所有权和借用验证 (RustBelt, Iris)
- **C/C++**: 内存安全验证
- **并发程序**: 资源竞争检测

### 1.2 为什么需要分离逻辑

传统Hoare逻辑在堆推理方面存在局限：

```
传统问题:
{heap = h} x := alloc() {heap = h'?}

无法表达:
- h' 与 h 的关系
- 新分配内存的位置
- 哪些内存未改变
```

分离逻辑通过**分离合取**(*, separating conjunction)解决这些问题。

### 1.3 核心思想

```
分离合取: P * Q

含义:
- 堆可以分成两个不相交的部分
- 一部分满足 P
- 另一部分满足 Q

┌─────────────────────────────────────┐
│              Heap                   │
│  ┌─────────────┐ ┌─────────────┐   │
│  │  满足 P     │ │  满足 Q     │   │
│  │             │ │             │   │
│  └─────────────┘ └─────────────┘   │
│         ↑               ↑           │
│    不相交(分离)         │           │
└─────────────────────────────────────┘
```

## 2. 基础定义

### 2.1 堆(Heap)的数学定义

**定义 2.1** (堆):

```
h ∈ Heap = Addr ⇀fin Value
```

堆是**有限偏函数**，从地址映射到值。使用偏函数因为：

- 不是所有地址都已分配
- 访问未分配地址是未定义行为

**定义 2.2** (堆的域):

```
dom(h) = {a ∈ Addr | h(a) 有定义}
```

**定义 2.3** (堆的分离/不相交):

```
h₁ ⊥ h₂  ⟺  dom(h₁) ∩ dom(h₂) = ∅
```

**定义 2.4** (堆的并):

```
(h₁ • h₂)(a) = h₁(a)  如果 a ∈ dom(h₁)
             = h₂(a)  如果 a ∈ dom(h₂)
             = 未定义 其他
```

仅当 h₁ ⊥ h₂ 时定义。

### 2.2 存储(Store)与状态

**定义 2.5** (存储):

```
s ∈ Store = Var → Value
```

存储将变量映射到值（通常是地址）。

**定义 2.6** (状态):

```
σ ∈ State = Store × Heap
```

完整程序状态由存储和堆组成。

## 3. 断言语言

### 3.1 语法

```
P, Q ::= emp                    (空堆)
       | e₁ ↦ e₂               (单点)
       | P * Q                 (分离合取)
       | P ∧ Q                 (经典合取)
       | P ∨ Q                 (析取)
       | ¬P                    (否定)
       | ∃x.P                  (存在)
       | ∀x.P                  (全称)
       | P -* Q                (分离蕴含)
       | e₁ = e₂               (相等)
       | e₁ < e₂               (不等)
```

### 3.2 语义

**定义 3.1** (满足关系):

```
(s, h) ⊨ P  表示状态(s, h)满足断言P
```

**空堆**:

```
(s, h) ⊨ emp  ⟺  h = ∅
```

**单点**:

```
(s, h) ⊨ e₁ ↦ e₂  ⟺  dom(h) = {[e₁]s} ∧ h([e₁]s) = [e₂]s
```

其中 [e]s 是表达式 e 在存储 s 下的求值。

**分离合取**:

```
(s, h) ⊨ P * Q  ⟺  ∃h₁, h₂. h₁ ⊥ h₂ ∧ h = h₁ • h₂
                    ∧ (s, h₁) ⊨ P ∧ (s, h₂) ⊨ Q
```

**分离蕴含**:

```
(s, h) ⊨ P -* Q  ⟺  ∀h'. h' ⊥ h ∧ (s, h') ⊨ P
                     ⇒ (s, h • h') ⊨ Q
```

### 3.3 常见模式

**列表段**:

```
ls(x, y) ::= x = y ∧ emp          (空列表)
           | ∃z. x ↦ z * ls(z, y)  (非空列表)
```

**链表**:

```
list(x) ::= x = null ∧ emp
          | ∃y. x ↦ (v, y) * list(y)
```

**树**:

```
tree(x) ::= x = null ∧ emp
          | ∃l,r. x ↦ (v, l, r) * tree(l) * tree(r)
```

## 4. 可靠性定理

### 4.1 Hoare三元组

**定义 4.1** (部分正确性Hoare三元组):

```
⊢ {P} C {Q}
```

含义：如果程序C从满足P的状态开始执行，

- 如果终止，则最终状态满足Q
- 可能发散（不停机）

**定义 4.2** (完全正确性):

```
⊢ [P] C [Q]
```

含义：C一定终止，且最终状态满足Q。

### 4.2 可靠性定理

**定理 4.3** (分离逻辑可靠性):
> 如果 ⊢ {P} C {Q} 可证，则对于所有满足(s, h) ⊨ P的状态，
> 执行C后要么发散，要么终止于满足Q的状态。

形式化：

```
⊢ {P} C {Q}  ∧  (s, h) ⊨ P  ∧  ⟨C, (s, h)⟩ ⇓ (s', h')
────────────────────────────────────────────────────────
(s', h') ⊨ Q
```

其中 `⟨C, σ⟩ ⇓ σ'` 表示大步骤操作语义。

### 4.3 完备性说明

分离逻辑对于**指针操作**是完备的吗？

**答案**: 否。分离逻辑是可靠的但不是完备的，因为：

1. 存在有效但不可证的程序
2. 某些复杂数据结构需要归纳断言
3. 图结构可能无法精确描述

但分离逻辑在实践中非常有效。

## 5. 完整证明

### 5.1 证明策略

使用**结构归纳法**，对命令C的结构进行归纳。

### 5.2 基本命令的证明

#### 5.2.1 空命令 (Skip)

**规则**:

```
──────────
{P} skip {P}
```

**证明**:

- skip不改变任何状态
- 如果 (s, h) ⊨ P，执行skip后仍是(s, h)
- 所以 (s, h) ⊨ P ✓

#### 5.2.2 赋值 (Assignment)

**规则**:

```
─────────────────
{P[e/x]} x := e {P}
```

**证明**:
设初始状态(s, h)满足 P[e/x]。

执行x := e后，新存储 s' = s[x ↦ [e]s]。

需要证明: (s', h) ⊨ P

由替换引理：

```
(s, h) ⊨ P[e/x]  ⟺  (s[x ↦ [e]s], h) ⊨ P
```

这正是 (s', h) ⊨ P。✓

#### 5.2.3 内存分配 (Alloc)

**规则**:

```
─────────────────────────────
{emp} x := alloc() {x ↦ _}
```

**证明**:

- 初始状态(s, h)，其中h = ∅（因为前置条件是emp）
- alloc() 分配新地址a ∉ dom(h)
- 新状态: s' = s[x ↦ a], h' = h[a ↦ v]（v为初始值）

需要证明: (s', h') ⊨ x ↦ _

- [x]s' = a
- dom(h') = {a}
- h'(a) = v

所以 (s', h') ⊨ x ↦ v，蕴含 x ↦ _。✓

#### 5.2.4 内存释放 (Free)

**规则**:

```
──────────────────────
{x ↦ _} free(x) {emp}
```

**证明**:

- 初始状态(s, h)，其中h = [a ↦ v]且[x]s = a
- free(x) 释放地址a
- 新状态: s' = s, h' = h \ {a} = ∅

需要证明: (s', h') ⊨ emp

h' = ∅，所以满足emp。✓

#### 5.2.5 内存读取 (Lookup)

**规则**:

```
{∃v. x ↦ v ∧ P[v/y]} y := [x] {∃v. x ↦ v ∧ P[v/y]}
```

**证明**:

- 初始状态(s, h)满足 ∃v. x ↦ v ∧ P[v/y]
- 即存在v使得 h = [a ↦ v]且[x]s = a，且(s, h) ⊨ P[v/y]
- 执行y := [x]，新存储 s' = s[y ↦ v]
- 堆不变: h' = h

需要证明: (s', h') ⊨ ∃v. x ↦ v ∧ P[v/y]

- h' = h = [a ↦ v]
- 所以 x ↦ v 仍成立
- P[v/y] 在(s', h')中也成立，因为P[v/y]中y是自由的，被替换为v

✓

#### 5.2.6 内存写入 (Mutation)

**规则**:

```
{x ↦ _} [x] := e {x ↦ e}
```

**证明**:

- 初始状态(s, h)，其中[x]s = a，h = [a ↦ _]
- 执行[x] := e，新堆 h' = [a ↦ [e]s]
- 存储不变: s' = s

需要证明: (s', h') ⊨ x ↦ e

- [x]s' = a
- h'(a) = [e]s = [e]s'
- 所以 (s', h') ⊨ x ↦ e ✓

### 5.3 复合命令的证明

#### 5.3.1 顺序组合

**规则**:

```
{P} C₁ {Q}    {Q} C₂ {R}
──────────────────────────
{P} C₁; C₂ {R}
```

**证明**:

- 从满足P的状态开始
- 由归纳假设，执行C₁后得到Q（或发散）
- 如果终止，从Q开始执行C₂
- 由归纳假设，执行C₂后得到R（或发散）
- 所以整体得到R（或发散）✓

#### 5.3.2 条件语句

**规则**:

```
{P ∧ e} C₁ {Q}    {P ∧ ¬e} C₂ {Q}
──────────────────────────────────
{P} if e then C₁ else C₂ {Q}
```

**证明**:

- 如果e为真：
  - 状态满足P ∧ e
  - 由归纳假设，执行C₁后得到Q
- 如果e为假：
  - 状态满足P ∧ ¬e
  - 由归纳假设，执行C₂后得到Q
- 两种情况下都得到Q ✓

#### 5.3.3 循环语句

**规则**:

```
{P ∧ e} C {P}
───────────────────────────
{P} while e do C {P ∧ ¬e}
```

**证明**:
使用**循环不变量**P。

- 初始状态满足P
- 每次迭代：
  - 进入时满足P ∧ e
  - 由归纳假设，执行C后仍满足P
- 循环终止时：
  - 条件e为假
  - 所以满足P ∧ ¬e ✓

## 6. 关键引理

### 6.1 框架规则 (Frame Rule)

**定理 6.1** (框架规则):

```
{P} C {Q}    fv(R) ∩ mod(C) = ∅
────────────────────────────────
{P * R} C {Q * R}
```

**含义**: 如果C只修改特定资源，其他资源R保持不变。

**证明**:

- 初始状态(s, h₁ • h₂)，其中(s, h₁) ⊨ P，(s, h₂) ⊨ R
- 执行C：由于fv(R) ∩ mod(C) = ∅，C不修改h₂
- 由{P} C {Q}，执行后(s, h₁') ⊨ Q
- h₂不变
- 所以(s, h₁' • h₂) ⊨ Q * R ✓

### 6.2 局部性规则 (Rule of Consequence)

**定理 6.2**:

```
P' ⇒ P    {P} C {Q}    Q ⇒ Q'
───────────────────────────────
{P'} C {Q'}
```

**含义**: 可以强化前置条件，弱化后置条件。

### 6.3 分配规则 (Rule of Disjunction)

**定理 6.3**:

```
{P₁} C {Q₁}    {P₂} C {Q₂}
────────────────────────────
{P₁ ∨ P₂} C {Q₁ ∨ Q₂}
```

### 6.4 存在规则 (Rule of Existential)

**定理 6.4**:

```
对于所有x, {P(x)} C {Q(x)}
────────────────────────────
{∃x.P(x)} C {∃x.Q(x)}
```

## 7. Rust的扩展

### 7.1 标准分离逻辑的局限

标准分离逻辑假设：

- 所有资源都是独占的
- 没有共享只读访问

但Rust允许：

```rust
let x = 5;
let r1 = &x;  // 共享引用
let r2 = &x;  // 另一个共享引用
```

### 7.2 分数权限 (Fractional Permissions)

**Boyland (2003)** 引入分数权限：

```
π ∈ (0, 1]  权限分数

π = 1:  独占访问（读写）
π < 1:  共享访问（只读）

权限组合: π₁ + π₂ = 1 表示可组合
```

**Rust映射**:

```
&mut T  →  T^1      (独占，π=1)
&T      →  T^π      (共享，0<π<1)
```

### 7.3 生命周期作为时序逻辑

**扩展**: 将分离逻辑与时序逻辑结合

```
□P  - 总是P（在整个生命周期内）
◇P  - 最终P
P U Q - P直到Q
```

**Rust生命周期**:

```rust
fn borrow<'a>(x: &'a T) -> &'a U
```

对应：返回的引用与输入引用有相同的生命周期约束。

### 7.4 RustBelt的Iris框架

**Iris** 是一个高阶分离逻辑框架，用于验证Rust程序。

**核心概念**:

```
Own(T, v)  - 拥有类型T的值v
Shr(T, v)  - 共享引用
Frac(π, P) - 分数权限
```

**示例验证**:

```rust
fn swap(x: &mut i32, y: &mut i32) {
    let t = *x;
    *x = *y;
    *y = t;
}
```

Iris规范：

```
{Own(&mut i32, x) * Own(&mut i32, y)}
swap(x, y)
{Own(&mut i32, x) * Own(&mut i32, y) ∧ *x = old(*y) ∧ *y = old(*x)}
```

## 8. 实际应用

### 8.1 验证链表操作

**插入操作**:

```rust
fn insert(head: &mut Box<Node>, val: i32) {
    let new_node = Box::new(Node { val, next: None });
    new_node.next = Some(std::mem::replace(head, new_node));
}
```

**分离逻辑规范**:

```
{list(head) ∧ head ↦ node}
insert(head, v)
{∃new. head ↦ new * new ↦ (v, old_head) * list(old_head)}
```

### 8.2 验证无数据竞争

**定理**: 如果程序在分离逻辑中可证，则无数据竞争。

**证明思路**:

- 数据竞争需要同时读写同一地址
- 分离逻辑确保独占访问(π=1)时无其他访问
- 共享访问(π<1)时只有读访问

## 9. 结论

### 9.1 核心贡献

分离逻辑为Rust提供了：

1. **形式化语义**: 精确的数学定义
2. **可靠性保证**: 证明 implies 正确
3. **模块化验证**: 框架规则支持局部推理
4. **工具基础**: Creusot, Prusti等验证工具

### 9.2 与Rust的关系

```
Rust程序 ──► 分离逻辑规范 ──► 形式化证明
                │
                ▼
            内存安全保证
            数据竞争自由
            正确性验证
```

### 9.3 未来方向

1. **自动化**: 减少手动证明负担
2. **并发**: 更强大的并发程序验证
3. **Unsafe Rust**: 验证unsafe代码块
4. **标准库验证**: 验证core/alloc/std
