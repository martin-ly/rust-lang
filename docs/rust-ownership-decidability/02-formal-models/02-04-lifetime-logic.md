## 1. 引言：生命周期作为区域

### 1.1 为什么需要生命周期

**问题：悬挂引用**:

```rust
fn dangling_reference() -> &String {  // 错误!
    let s = String::from("hello");
    &s  // s在函数结束时被drop
}  // 返回的引用指向无效内存
```

编译器错误：

```text
error[E0106]: missing lifetime specifier
 --> src/main.rs:1:16
  |
1 | fn dangling() -> &String {
  |                ^ expected named lifetime parameter
```

**解决方案：显式生命周期**:

```rust
// 正确：生命周期保证引用有效
fn valid_reference(s: &String) -> &str {
    &s[0..5]  // 返回的引用与s有相同生命周期
}
```

## 2. 区域代数基础

### 2.1 区域的形式化定义

**区域作为控制流图的子集**:

```text
区域 (Region):

设 G = (N, E, n₀, n∞) 为控制流图，其中:
- N: 程序点集合
- E ⊆ N × N: 控制流边
- n₀: 入口节点
- n∞: 出口节点

区域 α 是 N 的子集，满足:
- α 是连通的 (在控制流意义上)
- α 有唯一的入口和出口

区域表示引用/资源有效的程序点集合。
```

**示例**:

```rust
fn example() {
    let x = 1;           // n₁: 'a 开始
    {                    // n₂
        let r = &x;      // n₃: 'b 开始
        println!("{}", r);
    }                    // n₄: 'b 结束
}                        // n₅: 'a 结束

区域:
'a = {n₁, n₂, n₃, n₄, n₅}
'b = {n₃, n₄}
```

### 2.2 区域操作

**区域代数**:

```text
设 Regions 为所有区域的集合，定义:

包含 (Subregion):    α ⊆ β
  区域α中的所有点都在β中
  "α 的生命周期短于 β"

连接 (Join):         α ⊔ β
  包含α和β的最小区域
  "α或β有效的区域"

交汇 (Meet):         α ⊓ β
  同时属于α和β的最大区域
  "α和β同时有效的区域"

空区域:              ⊥ = ∅
  从不有效的区域

全局区域:            ⊤ = N
  始终有效的区域 (对应 'static)
```

**区域格**:

```text
区域形成完全格 (Complete Lattice):

        ⊤ ('static)
       /   |   \
      /    |    \
   ...    ...    ...
      \    |    /
       \   |   /
        ⊥ (empty)

性质:
- 偏序: ⊆
- 最小上界: ⊔
- 最大下界: ⊓
```

### 2.3 区域约束

**生命周期约束**:

```text
生命周期约束形式:

'a: 'b   (读作 "'a outlives 'b")
含义: 'a ⊇ 'b  (区域'a包含'b)

示例:
fn foo<'a, 'b>(x: &'a i32, y: &'b i32) -> &'a i32
约束: 返回类型需要 'a: 'b 确保返回值有效
```

**约束集合**:

```text
约束集合 C ::= · | C, 'a: 'b

约束求解:
给定约束集合C，找到区域赋值 σ: Lifetime → Region 使得:
∀('a: 'b) ∈ C. σ('a) ⊇ σ('b)
```

## 3. 生命周期逻辑

### 3.1 生命周期命题

**基于RustBelt的生命周期逻辑**:

```text
生命周期命题:

[α]₁    - 生命周期α存活的token (恰好一个)
[↑α]    - 生命周期α结束的权限
&^α P   - 在生命周期α期间借用P

生命周期上下文:
[[L]] ≡ ★_{α∈L} ([α]₁ * ([α]₁ ⇒ ◇[↑α]))

其中:
- ★: 分离合取的大聚合
- *: 分离合取
- ⇒: 分离蕴含
- ◇: update modality
```

### 3.2 生命周期上下文

**上下文操作**:

```text
创建生命周期:
new_lifetime() =
  分配新的生命周期变量α
  创建 [α]₁

结束生命周期:
end_lifetime(α) =
  消耗 [α]₁
  获得 [↑α]
  释放借用资源

包含关系:
'a: 'b 意味着当 'a 结束时 'b 也结束
形式化: [↑a] ⊢ [↑b]
```

### 3.3 借用命题

**借用作为生命周期限定的资源**:

```text
借用命题 (Borrow Propositions):

&^α P ≡ 在生命周期α期间暂时持有P

性质:
1. 当α结束时，P返回原所有者
2. 借用期间原所有者被"冻结"
3. 借用可以被重新借用 (创建更短生命周期的借用)

重新借用:
&^α P 可以通过 &^β 重新借用，其中 β ⊆ α
```

**借用的创建与销毁**:

```text
借用创建:
Γ ⊢ P    α fresh
───────────────────
Γ * [α]₁ ⊢ &^α P * (P ⇒_α ◇[↑α])

借用结束:
Γ ⊢ &^α P    Γ, [↑α] ⊢ Q
──────────────────────────
Γ * [α]₁ ⊢ Q
```

## 4. 子类型与生命周期

### 4.1 生命周期方差

**方差定义**:

```text
类型构造器F对方位参数T的方差:

协变 (Covariant):    T₁ <: T₂  ⊢  F<T₁> <: F<T₂>
逆变 (Contravariant): T₁ <: T₂  ⊢  F<T₂> <: F<T₁>
不变 (Invariant):    T₁ <: T₂  ⊢  F<T₁> ≰ F<T₂> ∧ F<T₂> ≰ F<T₁>
```

**Rust中的方差**:

| 类型构造器 | 生命周期参数 | 类型参数 |
|-----------|------------|---------|
| &T | 协变 | 协变 |
| &mut T | 协变 | 不变 |
| `Box<T>` | - | 协变 |
| `Vec<T>` | - | 协变 |
| `fn(T) -> U` | - | T逆变, U协变 |
| *const T | - | 协变 |
| *mut T | - | 不变 |
| `Cell<T>` | - | 不变 |

### 4.2 子类型规则

**生命周期子类型**:

```text
生命周期包含蕴含子类型:

'a: 'b  ⊢  &'a T <: &'b T

证明:
如果 'a ⊇ 'b，则任何在'a有效的引用也在'b有效
因此 &'a T 可以安全地用作 &'b T
```

**复合类型子类型**:

```text
元组:
τ₁ <: τ₁'    τ₂ <: τ₂'
─────────────────────────
(τ₁, τ₂) <: (τ₁', τ₂')

函数:
τ₁' <: τ₁    τ₂ <: τ₂'
─────────────────────────
fn(τ₁) -> τ₂ <: fn(τ₁') -> τ₂'

引用:
'a: 'b    τ <: τ'
───────────────────
&'a τ <: &'b τ'
```

## 5. 生命周期推断

### 5.1 约束生成

**类型检查中的约束生成**:

```text
类型检查判断:
Γ ⊢ e : τ ~> C

其中C是生成的约束集合

约束生成规则:

变量:
────────────
Γ, x:τ ⊢ x : τ ~> ·

函数应用:
Γ ⊢ e₁ : fn(τ₁) -> τ₂ ~> C₁    Γ ⊢ e₂ : τ₁' ~> C₂
───────────────────────────────────────────────────
Γ ⊢ e₁ e₂ : τ₂ ~> C₁ ∪ C₂ ∪ {τ₁' <: τ₁}

借用:
Γ ⊢ e : τ ~> C
─────────────────────────────
Γ ⊢ &e : &'a τ ~> C ∪ {scope(e) ⊆ 'a}
```

### 5.2 约束求解

**约束求解算法**:

```text
约束求解:

solve(C) =
  1. 构建约束图 G = (V, E)
     V = 所有生命周期变量
     E = {('a, 'b) | 'a: 'b ∈ C}

  2. 检测循环 (对于严格约束)

  3. 计算传递闭包

  4. 为每个变量分配最小区域
     σ('a) = ⊔{σ('b) | 'a: 'b ∈ C}

  5. 验证所有约束满足
```

**示例**:

```rust
fn choose<'a, 'b>(x: &'a i32, y: &'b i32) -> &'a i32 {
    x
}

约束:
'a: 'return
'b: (无约束)

解:
'return = 'a
```

### 5.3 最小程序点

**最小生命周期赋值**:

```text
最小程序点 (Minimal Program Points):

对于每个生命周期变量'a，找到最小的区域使得:
1. 所有约束被满足
2. 借用检查通过

算法:
pp('a) = ⋂{ pp('b) | 'a: 'b ∈ C }

如果无约束，则分配包含所需语句的最小区域
```

## 6. 非词法生命周期 (NLL)

### 6.1 从词法到非词法

**词法生命周期 (Rust 1.0)**:

```rust
fn lexical_lifetime() {
    let mut data = vec![1, 2, 3];
    let slice = &data[0];  // 借用开始

    // 即使这里不再使用slice，借用仍持续到作用域结束
    // data.push(4);  // 错误! slice仍然"活"着

    println!("{}", slice);  // 最后使用
}  // 借用结束
```

**非词法生命周期 (Rust 2018+)**:

```rust
fn non_lexical_lifetime() {
    let mut data = vec![1, 2, 3];
    let slice = &data[0];

    println!("{}", slice);  // 最后一次使用

    // slice的借用在这里结束！
    data.push(4);  // OK!
}
```

### 6.2 基于数据流的分析

**数据流分析框架**:

```text
借用活跃性分析:

定义:
- Gen[n]: 在节点n生成的借用
- Kill[n]: 在节点n杀死的借用
- LiveIn[n]: 进入n时活跃的借用
- LiveOut[n]: 离开n时活跃的借用

方程:
LiveOut[n] = ⋃_{s∈succ(n)} LiveIn[s]
LiveIn[n] = Gen[n] ∪ (LiveOut[n] - Kill[n])

求解:
使用数据流分析的标准工作列表算法
```

**NLL算法**:

```text
NLL检查算法:

1. 构建控制流图 (CFG)
2. 识别所有借用点
3. 对每个借用，计算其活跃区域
4. 检查借用冲突:
   - 计算每对借用的活跃区域交集
   - 如果交集非空且借用冲突，报错
5. 检查悬挂引用:
   - 确保引用不活得比被引用数据长
```

## 8. 实践意义

### 编译时内存安全

```text
生命周期系统保证:

1. 无悬挂引用
   fn dangle() -> &String { ... }  // 编译错误

2. 无使用后释放
   let r = &x; drop(x); *r;  // 编译错误

3. 无迭代器失效
   let mut v = vec![1, 2, 3];
   for x in &v { v.push(*x); }  // 编译错误

4. 无数据竞争
   let mut x = 5;
   let r1 = &mut x;
   let r2 = &x;  // 编译错误
```

### 零成本抽象

```text
生命周期检查在编译时完成:

- 无运行时开销
- 生成的代码与手动管理内存一样高效
- 不依赖垃圾回收
```

## 生命周期规则总结

```text
┌─────────────────────────────────────────────────────────────────┐
│                      生命周期规则总结                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  基本规则:                                                      │
│  - 引用不能活得比被引用数据长                                    │
│  - 生命周期是区域的子集                                         │
│  - 生命周期约束 'a: 'b 表示 'a ⊇ 'b                             │
│                                                                 │
│  省略规则 (Elision):                                            │
│  - 每个引用类型有隐式生命周期参数                                │
│  - 单个生命周期时，输出=输入                                     │
│  - &self时，输出=self的生命周期                                  │
│                                                                 │
│  子类型:                                                        │
│  - &'a T <: &'b T  if 'a: 'b                                   │
│  - fn(A) -> B <: fn(C) -> D if C <: A and B <: D               │
│                                                                 │
│  NLL:                                                           │
│  - 生命周期基于使用位置而非作用域范围                             │
│  - 数据流分析确定借用活跃性                                      │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```
