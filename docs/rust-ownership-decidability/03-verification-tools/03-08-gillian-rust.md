## 1. 概述

Gillian-Rust 是一个基于 **Gillian 符号执行框架** 的 Rust 验证工具，采用**混合方法**验证 unsafe Rust 代码。
它结合了符号执行的自动化能力和分离逻辑的表达能力，为验证包含 unsafe 代码的 Rust 程序提供了一种新的途径。

**核心特点**:

- ✅ 支持 safe 和 unsafe Rust 代码
- ✅ 全自动验证流程
- ✅ 详细的内存模型
- ✅ 基于 Gillian 的多语言验证基础设施

### 2.1 Gillian 核心特性

## 3. Gillian-Rust 方法

### 3.1 混合验证架构

Gillian-Rust 采用**双引擎**架构:

```text
┌─────────────────────────────────────────────────────────────┐
│                    Gillian-Rust 架构                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │              符号执行引擎                            │   │
│  │  • 自动状态空间探索                                  │   │
│  │  • 路径约束求解                                      │   │
│  │  • 处理控制流分支                                    │   │
│  └─────────────────────────────────────────────────────┘   │
│                         ↕ 交互                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │              分离逻辑引擎                            │   │
│  │  • 内存状态建模                                      │   │
│  │  • 不变量推理                                        │   │
│  │  • 资源管理                                          │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**优势**:

- 符号执行处理控制流和路径探索
- 分离逻辑处理复杂内存推理
- 两者协同，提高验证能力

### 3.2 符号执行

**Gillian 的符号执行**特点:

1. **符号状态**: 使用符号值表示未知输入
2. **路径约束**: 跟踪每条路径的条件约束
3. **约束求解**: 使用 SMT 求解器验证属性

```rust
// 符号执行示例
fn symbolic_example(x: i32) -> i32 {
    if x > 0 {
        x + 1          // 路径 1: x > 0
    } else {
        x - 1          // 路径 2: x ≤ 0
    }
}

// Gillian-Rust 会探索两条路径:
// Path 1: x > 0, result = x + 1
// Path 2: x ≤ 0, result = x - 1
```

### 3.3 分离逻辑推理

**分离逻辑组件**:

| 断言 | 含义 |
|------|------|
| `emp` | 空堆 |
| `l ↦ v` | 位置 l 存储值 v |
| `P * Q` | P 和 Q 分离合取 |
| `P -* Q` | 分离蕴涵 (魔法棒) |

**在 Rust 中的应用**:

```rust
// 所有权建模
let x = Box::new(42);
// 分离逻辑: x.ptr ↦ 42 * x 拥有该内存

let r = &x;
// 共享引用: x.ptr ↦ 42 (共享权限)
```

## 4. 核心特性

### 4.1 Unsafe 代码建模

Gillian-Rust 的关键优势是**原生支持 unsafe Rust**:

| Unsafe 特性 | 建模方法 |
|-------------|----------|
| 原始指针 | 分离逻辑中的内存位置 |
| 解引用 | 内存访问断言 |
| 指针运算 | 地址计算和边界检查 |
| 类型转换 | 内存 reinterpretation |

**示例: 原始指针操作**:

```rust
unsafe fn swap_via_ptr(a: *mut i32, b: *mut i32) {
    let temp = *a;
    *a = *b;
    *b = temp;
}

// Gillian-Rust 验证目标:
// 前置: a ↦ v1 * b ↦ v2 (a, b 有效且分离)
// 后置: a ↦ v2 * b ↦ v1 (值已交换)
```

### 4.2 内存模型

Gillian-Rust 实现了**详细的内存模型**:

```text
内存模型层次:
┌─────────────────────────────────────────┐
│  抽象内存模型                           │
│  • 分配块 (allocation blocks)           │
│  • 权限追踪 (permission tracking)       │
├─────────────────────────────────────────┤
│  低级内存操作                           │
│  • 原始指针读写                         │
│  • 内存布局计算                         │
│  • 对齐检查                             │
├─────────────────────────────────────────┤
│  不安全模式                             │
│  • 未定义行为检测                       │
│  • 别名规则检查                         │
└─────────────────────────────────────────┘
```

### 4.3 自动化程度

Gillian-Rust 提供**全自动验证**:

- 无需手动证明脚本
- 自动发现内存安全违规
- 自动生成反例

## 6. 与相关工具对比

| 特性 | Gillian-Rust | RefinedRust | Kani | Prusti/Creusot |
|------|--------------|-------------|------|----------------|
| **方法** | 符号执行+分离逻辑 | 精细化类型 | 模型检测 | 演绎验证 |
| **Safe Rust** | ✅ | ✅ | ✅ | ✅ |
| **Unsafe Rust** | ✅ | ✅ | ✅ | ❌ |
| **自动化** | 全自动 | 半自动 | 全自动 | 半自动 |
| **基础性证明** | ❌ | ✅ | ❌ | ❌ |
| **内存模型** | 详细 | 详细 | 详细 | 抽象 |
| **状态空间** | 有界/启发式 | 无界 | 有界 | 无界 |

**选择建议**:

- **Gillian-Rust**: 需要全自动验证 unsafe 代码，非基础性证明可接受
- **RefinedRust**: 需要基础性证明，可接受半自动化
- **Kani**: 需要验证特定属性，可接受有界模型检测
- **Prusti/Creusot**: 纯 safe Rust，需要高度自动化

## 7. 应用场景

### 7.1 适用场景

1. **Unsafe 代码审查**
   - 验证 FFI 边界代码
   - 检查原始指针使用

2. **标准库验证**
   - 验证 Vec, String 等核心数据结构的 unsafe 实现

3. **安全关键代码**
   - 密码学原语实现
   - 系统编程代码

### 7.2 案例研究

**Vec 实现验证** (概念性):

```rust
impl<T> Vec<T> {
    unsafe fn get_unchecked_mut(&mut self, index: usize) -> &mut T {
        // Gillian-Rust 验证:
        // 1. index < self.len (前置条件)
        // 2. 返回的引用有效 (后置条件)
        // 3. 不违反别名规则
        &mut *self.as_mut_ptr().add(index)
    }
}
```

## 8. 限制与挑战

| 限制 | 说明 | 潜在解决方案 |
|------|------|--------------|
| **路径爆炸** | 符号执行的状态空间爆炸 | 使用启发式剪枝 |
| **循环处理** | 需要循环不变量 | 自动不变量推断 |
| **外部函数** | 对 FFI 调用建模困难 | 函数摘要 |
| **并发** | 目前不支持并发代码 | 扩展内存模型 |
| **泛型/Traits** | 支持有限 | 持续开发 |
