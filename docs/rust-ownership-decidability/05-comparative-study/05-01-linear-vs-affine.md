## 2. 形式化定义

### 2.1 线性类型系统

```text
线性λ演算语法:

t ::= x                     变量
    | λx.t                  线性抽象 (x必须恰好使用一次)
    | t₁ t₂                 应用
    | let x = t₁ in t₂      let绑定

类型规则 (线性):

Γ, x: A ⊢ x : A               (Var - 变量使用)

Γ, x: A ⊢ t : B               x在t中恰好使用一次
───────────────────────
Γ ⊢ λx.t : A ⊸ B

Γ ⊢ t₁ : A ⊸ B    Δ ⊢ t₂ : A   Γ和Δ不相交
────────────────────────────────────────
Γ, Δ ⊢ t₁ t₂ : B

关键约束: 上下文中每个变量必须恰好使用一次
```

### 2.2 仿射类型系统

```text
仿射λ演算语法 (与线性相同):

t ::= x | λx.t | t₁ t₂ | let x = t₁ in t₂ | drop x

类型规则 (仿射):

Γ ⊢ x : A                      (Var - 使用变量)

Γ ⊢ x : A                      (Drop - 丢弃变量)
───────────────────────
Γ \ {x} ⊢ drop x : Unit

Γ, x: A ⊢ t : B                x在t中使用0或1次
───────────────────────
Γ ⊢ λx.t : A ⊸ B

关键区别: 变量可以使用0次 (weakening) 或1次
```

## 3. 代码对比

### 3.1 资源使用模式

```rust
// Rust (仿射类型) - 允许忽略资源
fn affine_example() {
    let file = File::open("data.txt").unwrap();  // 资源创建
    // 可能不使用file - 允许！
    // Drop trait确保最终关闭
}  // file在这里自动drop
```

```clean
// Clean (线性类型) - 必须使用资源
linearExample :: File *World -> World
linearExample file world
    // 必须显式使用file
    # (ok, file`, world`) = readLine file world
    | not ok = closeFile file` world`  // 必须处理
    // ...
```

### 3.2 复制与共享

```rust
// Rust - 通过Clone显式复制
fn affine_share() {
    let s1 = String::from("hello");
    let s2 = s1.clone();  // 显式深拷贝
    // s1和s2都可用
}
```

```clean
// Clean - 线性类型下复制需要特殊处理
linearShare :: String -> (String, String)
linearShare s
    // 需要显式复制操作
    = (s, copy s)
```

## 4. 可判定性对比

### 4.1 形式化结果

| 属性 | 线性逻辑 | 仿射逻辑 |
|------|---------|---------|
| 命题片段 (MALL/MALLW) | PSPACE-Complete | PSPACE-Complete |
| 含指数 (!A, ?A) | 不可判定 | **可判定** |
| 完整一阶 | 不可判定 | 不可判定 |
| 完整高阶 | 不可判定 | 不可判定 |

### 4.2 Kopylov定理

```text
定理 (Kopylov, 2001):
完整命题仿射逻辑 (包含所有乘法、加法、指数和常量) 是可判定的。

证明技术:
1. 归约到正规形式
2. 向量游戏语义
3. 受控的弱化规则应用

与线性的关键区别:
- 弱化规则允许"忽略"公式
- 这限制了证明搜索空间
- 使得决策过程可以终止
```

## 6. Rust的设计选择

### 6.1 为什么是仿射而非线性

```text
Rust选择仿射类型的原因:

1. 实用性
   - 允许临时变量
   - 不需要显式drop每个值
   - 更符合直觉

2. 与现有代码兼容
   - 可以忽略不需要的错误码
   - 简化API设计

3. 错误恢复
   - ?操作符可以短路返回
   - 不需要处理每个中间结果

4. 性能权衡
   - Drop可能在作用域结束时才执行
   - 但通常可以通过显式drop控制
```

### 6.2 向线性的接近

```rust
// Rust可以通过lint接近线性行为

#![deny(unused_variables)]  // 禁止未使用变量

fn near_linear() {
    let x = String::from("hello");
    // 不使用x会导致编译错误
    drop(x);  // 显式drop
}

// 或使用must_use属性
#[must_use]
fn important_result() -> Result<(), Error> { ... }
```

## 7. 理论基础对比

### 7.1 范畴论语义

```text
线性类型:
- 笛卡尔封闭范畴的变体
- 张量积 (⊗) 作为核心结构
- 对象代表资源

仿射类型:
- 仿射范畴
- 弱化态射: A → 1
- 张量积 + 弱化

关系:
仿射范畴是线性范畴的弱化版本
添加弱化操作破坏了严格线性，但保持了仿射性
```

### 7.2 证明论对比

```text
sequent演算对比:

线性逻辑:
Γ, A, A ⊢ B     (不能收缩)
────────────
Γ, A ⊢ B

仿射逻辑:
Γ ⊢ B           (可以弱化)
────────────
Γ, A ⊢ B

Γ, A, A ⊢ B     (仍然不能收缩)
────────────
Γ, A ⊢ B
```
