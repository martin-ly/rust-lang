## 3. 流抽象

### 定理 3.1 (独立流)

> 单个流的丢失不影响其他流。

**vs TCP**:

- TCP: 头部阻塞
- QUIC: 流独立

∎

### 定理 3.2 (流类型)

| 类型 | 方向 | ID特征 |
|------|------|--------|
| Bidirectional | 双向 | 客户端: 0,4,8... 服务端: 1,5,9... |
| Send-only | 单向发送 | 客户端: 2,6,10... |
| Recv-only | 单向接收 | 服务端: 3,7,11... |

∎

## 4. 0-RTT安全

### 定理 4.1 (重放保护)

> 0-RTT数据有重放风险，应用层需处理。

```rust
// 0-RTT数据应在幂等操作中使用
connection.send_datagram(data)?;  // 可能被重放

// 非幂等操作应等待1-RTT
let (mut send, mut recv) = connection.open_bi().await?;
```

∎

```rust
// 忘记关闭流可能导致资源泄漏
let (mut send, _recv) = conn.open_bi().await?;

```
