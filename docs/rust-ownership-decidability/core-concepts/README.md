## 形式化定义索引

本文档集中的形式化定义遵循以下符号约定：

| 符号 | 含义 |
|------|------|
| `Γ ⊢ e : T` | 在上下文 Γ 中，表达式 e 具有类型 T |
| `Γ ⊨ own(x)` | 在上下文 Γ 中，变量 x 拥有其值的所有权 |
| `Γ ⊨ borrow(x, mut)` | 变量 x 被可变借用 |
| `Γ ⊨ borrow(x, imm)` | 变量 x 被不可变借用 |
| `'a <: 'b` | 生命周期 'a 是 'b 的子类型（'a 至少和 'b 一样长）|
| `T: Trait` | 类型 T 实现了 Trait |

## 性能影响分析汇总

### 零成本抽象

Rust 的所有权系统绝大多数检查在编译期完成，运行时开销极小：

```rust
// 这段代码
let x = vec![1, 2, 3];
let y = x;  // 所有权转移

// 编译后等价于 C 语言：
// void* x = malloc(...);
// void* y = x;  // 只是指针复制
```

### 运行时开销场景

| 特性 | 运行时开销 | 使用场景 |
|------|------------|----------|
| `RefCell<T>` | 借用标志位检查 | 单线程内部可变性 |
| `Mutex<T>` | 系统调用 + 上下文切换 | 多线程共享状态 |
| `Rc<T>` / `Arc<T>` | 引用计数增减 | 共享所有权 |
| `Box<T>` | 堆分配 | 动态大小类型 |

## 常见陷阱与解决方案索引

### 陷阱 1: 移动后使用

```rust
let s = String::from("hello");
let t = s;  // s 的所有权移动到 t
println!("{}", s);  // ❌ 编译错误！
```

**解决方案**: 使用 `clone()` 或重新设计数据结构

### 陷阱 2: 悬垂引用

```rust
let r: &i32;
{
    let x = 5;
    r = &x;  // ❌ x 在此处被丢弃
}
```

**解决方案**: 确保引用不超出被引用值的生命周期

### 陷阱 3: 迭代器失效

```rust
let mut v = vec![1, 2, 3];
for x in &v {
    v.push(*x);  // ❌ 编译错误！
}
```

**解决方案**: 使用 `clone()` 或改变迭代策略
