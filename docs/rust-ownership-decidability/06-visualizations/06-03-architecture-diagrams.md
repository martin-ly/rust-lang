## 1. Rust所有权系统架构

### 1.1 所有权层次结构

```text
Rust所有权系统架构
│
├── 1. 所有权核心 (Ownership Core)
│   ├── 1.1 三大规则
│   │   ├── 每个值有所有者
│   │   ├── 所有者唯一
│   │   └── 离开作用域时drop
│   │
│   ├── 1.2 所有权操作
│   │   ├── Move (转移)
│   │   ├── Copy (复制)
│   │   └── Drop (释放)
│   │
│   └── 1.3 所有权标记
│       ├── Copy trait
│       └── Drop trait
│
├── 2. 借用系统 (Borrowing System)
│   ├── 2.1 借用类型
│   │   ├── 共享借用 (&T)
│   │   └── 可变借用 (&mut T)
│   │
│   ├── 2.2 借用规则
│   │   ├── 多个&或一个&mut
│   │   └── 引用必须有效
│   │
│   └── 2.3 重新借用
│       ├── 从&mut重新借用
│       └── 冻结机制
│
├── 3. 生命周期系统 (Lifetime System)
│   ├── 3.1 生命周期参数
│   │   ├── 显式生命周期
│   │   └── 省略规则 (Elision)
│   │
│   ├── 3.2 生命周期推断
│   │   ├── 基于约束求解
│   │   └── NLL (非词法生命周期)
│   │
│   └── 3.3 生命周期约束
│       ├── 'a: 'b (包含)
│       └── 'static
│
└── 4. 并发安全 (Concurrency Safety)
    ├── 4.1 Send trait
    │   └── 跨线程转移所有权
    │
    ├── 4.2 Sync trait
    │   └── 跨线程共享引用
    │
    └── 4.3 内部可变性
        ├── Cell (无同步)
        ├── RefCell (运行时检查)
        ├── Mutex (互斥)
        └── RwLock (读写锁)
```

### 1.2 类型系统层次

```text
Rust类型系统层次
│
├── 基础类型层
│   ├── 标量类型
│   │   ├── 整数 (i8, i16, i32, i64, i128, isize)
│   │   ├── 无符号整数 (u8, u16, u32, u64, u128, usize)

│   └── 复合类型

│       └── 切片 &[T]

├── 所有权类型层
│   ├── 拥有类型

│   ├── 引用类型

├── 抽象类型层

│   │
│   ├── Trait
│   │   ├── 抽象接口
│   │   ├── 泛型约束 (T: Trait)

│   │
│   └── 关联类型
│       └── type Output;
│
└── 高级类型层
    ├── 函数类型
    │   └── fn(T) -> U
    ├── 闭包类型
    │   ├── Fn
    │   ├── FnMut
    │   └── FnOnce
    ├── 永不类型 (!)
    └── 存在类型 (impl Trait)
```

## 2. Rust编译器架构

### 2.1 编译流程架构

```text
Rust编译器 (rustc) 架构
│
├── 前端 (Frontend)
│   ├── 词法分析 (Lexer)
│   │   └── 字符流 → Token流
│   │
│   ├── 语法分析 (Parser)
│   │   └── Token流 → AST
│   │
│   ├── 名称解析 (Name Resolution)
│   │   ├── 解析模块路径
│   │   └── 解析名称到DefId
│   │
│   └── 早期Lint
│       └── 语法级别警告
│
├── 中端 (Middle)
│   ├── 类型推断与检查 (Typeck)
│   │   ├── Hindley-Milner扩展
│   │   ├── Trait求解
│   │   └── 类型约束求解
│   │
│   ├── 降级到HIR
│   │   └── AST → HIR (简化)
│   │
│   ├── 借用检查 (Borrowck)
│   │   ├── NLL (MIR级别)
│   │   ├── 区域推断
│   │   └── 冲突检测
│   │
│   ├── 降级到MIR
│   │   └── HIR → MIR (中级IR)
│   │
│   └── 优化
│       ├── 常量传播
│       ├── 死代码消除
│       └── 内联
│
└── 后端 (Backend)
    ├── LLVM IR生成
    │   └── MIR → LLVM IR
    │
    ├── LLVM优化
    │   └── 目标无关优化
    │
    └── 代码生成
        ├── 目标代码生成
        └── 链接
```

```text
借用检查器 (Borrow Checker) 内部架构

│   │   └── 变量使用点
│   │
│   ├── 子类型约束 (Subtyping)
│   │   ├── 赋值约束
│   │   ├── 调用约束
│   │   └── 返回值约束
│   │
│   └── 重新借用约束 (Reborrow)
│       └── 借用链追踪
│
├── 约束求解阶段
│   ├── 区域推断
│   │   ├── 固定点迭代
│   │   └── 最小上界计算
│   │
│   └── 约束验证
│       └── 检查一致性
│
├── 借用检查阶段

│   ├── 访问分类

│   │   └── 写访问 (Shallow/Deep)

        ├── 移动后使用
        ├── 冲突借用
        └── 生命周期错误
```

## 使用说明

这些架构图用于:

1. **系统理解**: 把握Rust所有权和验证工具的组件结构
2. **开发指导**: 理解编译器和验证工具的内部组织
3. **研究定位**: 识别各组件在整体中的位置和关系
4. **教学设计**: 层次化地呈现复杂系统
