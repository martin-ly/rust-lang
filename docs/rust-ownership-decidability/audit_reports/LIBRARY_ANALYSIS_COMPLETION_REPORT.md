### 2. HashMap<K, V> 哈希表

**覆盖内容**:

- ✅ 罗宾汉哈希 (Robin Hood Hashing) 形式化
- ✅ 探测距离方差分析
- ✅ 控制字节 (control byte) 内存模型
- ✅ 操作语义与复杂度
- ✅ 无锁读取的内存安全

**关键定理**:

- **定理 3.1**: 罗宾汉哈希方差 $O(\log n)$ (vs. 标准 $O(\log^2 n)$)
- **定理 4.1**: 插入期望 $O(\frac{1}{1-\alpha})$
- **定理 5.1**: 无未初始化读取 (控制字节检查)

**反例分析**:

- 不正确的Hash实现 (哈希与相等不一致)
- 迭代器失效场景

### 3. 同步原语 (Arc/Mutex/RwLock)

**覆盖内容**:

- ✅ Arc原子引用计数的内存序
- ✅ Mutex互斥性与死锁自由
- ✅ RwLock读者-写者问题形式化
- ✅ 条件变量等待-通知协议
- ✅ C11内存模型映射

**关键定理**:

- **定理 2.3**: Arc使用Release-Acquire建立happens-before
- **定理 3.1**: Mutex保证互斥性
- **定理 3.2**: 单Mutex无死锁
- **定理 4.2**: 写者优先防止读者饥饿

**思维表征方式**:

```text
Happens-Before 图:

Thread A (创建Arc数据)
  │
  ▼ write(data)
  │
  ▼ Release
  │
  ▼ fetch_add(1, Release) ────────┐
  │                               │
  ▼ clone()                       │ Synchronizes-With
  │                               │
  ▼ drop()                        │
  │                               │
  ▼ fetch_sub(1, Release) ────────┤
                                  │
Thread B (clone Arc)              │
  │                               │
  ▼ Acquire ◄─────────────────────┘
  │
  ▼ read(data)  // 安全!
```

### 5. Crossbeam 并发原语

**覆盖内容**:

- ✅ Epoch-Based Reclamation (EBR) 形式化
- ✅ Michael-Scott无锁队列
- ✅ Treiber栈
- ✅ Chase-Lev工作窃取双端队列
- ✅ ABA问题解决
- ✅ 线性化点分析

**关键定理**:

- **定理 2.1**: EBR安全性 (无UAF)
- **定理 2.2**: 内存回收延迟 $O(n)$ 个epoch
- **定理 3.1**: Michael-Scott队列线性化
- **定理 6.1**: Lock-Free进度保证

**思维表征方式**:

```text
Epoch-Based Reclamation:

Global Epoch:     5 ──────► 6 ──────► 7
                  │          │          │
Thread 1:         5          6          6
                  │          │          │
Thread 2:         5          5          6
                  │          │          │
Thread 3:         5          5          5

垃圾回收条件: 节点epoch < min(所有线程的本地epoch)

Epoch 5 的垃圾可以在全局epoch 7时回收
(因为所有线程都已进入epoch 6或更高)
```

### 6. parking_lot 同步原语

**覆盖内容**:

- ✅ Word-size Mutex (1字节实现)
- ✅ 状态机分析
- ✅ 读者计数压缩 (RwLock)
- ✅ 公平性与饥饿自由
- ✅ 优先级继承

**关键定理**:

- **定理 2.1**: 解锁传递正确性
- **定理 3.1**: 读者计数压缩正确性
- **定理 5.1**: 空间复杂度 $O(1)$
- **定理 5.2**: 无竞争lock为 $O(1)$

**对比分析**:

```text
空间对比:
┌─────────────────┬──────────────┬─────────────┐
│ 类型            │ parking_lot  │ std::sync   │
├─────────────────┼──────────────┼─────────────┤
│ Mutex           │ 1 byte       │ 40 bytes    │
│ RwLock          │ 8 bytes      │ 40 bytes    │
│ Condvar         │ 8 bytes      │ 48 bytes    │
└─────────────────┴──────────────┴─────────────┘

性能对比 (纳秒):
┌─────────────────┬──────────────┬─────────────┐
│ 操作            │ parking_lot  │ std::sync   │
├─────────────────┼──────────────┼─────────────┤
│ lock/unlock     │ 15 ns        │ 80 ns       │
│ 有竞争lock      │ 100 ns       │ 300 ns      │
└─────────────────┴──────────────┴─────────────┘
```
