## 3.2 核心概念

### 3.2.1 类型安全与静态检查

Rust是一种静态类型、强类型语言，编译器在编译时进行严格的类型检查，确保类型一致性和安全性。类型安全意味着程序不会在运行时遇到未定义的类型操作，静态检查则确保这些保证在编译时得到验证。

### 3.2.2 参数多态性

参数多态性通过泛型实现，允许函数和数据结构对多种类型进行操作，而不需要为每种类型编写专门的代码。Rust的泛型系统支持类型参数、生命周期参数和常量参数。

### 3.2.3 类型约束与边界

类型约束通过特征边界（trait bounds）实现，限定了泛型参数必须满足的性质。这对应于类型论中的有界量化（bounded quantification）。Rust支持多重约束、where子句和关联类型。

### 3.2.4 型变规则

型变描述了复合类型的子类型关系如何依赖于其组成类型的子类型关系。Rust中存在三种型变关系：

- 协变（Covariant）：如果A是B的子类型，则`F<A>`是`F<B>`的子类型
- 逆变（Contravariant）：如果A是B的子类型，则`F<B>`是`F<A>`的子类型
- 不变（Invariant）：即使A是B的子类型，`F<A>`和`F<B>`之间也没有子类型关系

### 3.2.5 子类型关系

Rust通过特征对象实现了有界的子类型多态。特征对象允许处理实现相同特征的不同类型，实现了受限的子类型多态。Rust的子类型关系主要通过特征和生命周期约束表达。
