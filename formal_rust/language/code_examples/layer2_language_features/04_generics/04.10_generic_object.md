# 04.10 泛型与trait对象（Generic & Trait Object）

---

## 理论简述

泛型和trait对象是Rust多态性的两种主要实现方式。
泛型在编译期实现零成本抽象，trait对象通过类型擦除实现运行时多态。
两者可结合用于灵活的接口设计。
相关理论详见[泛型与trait对象理论](../../04_generics/05_generic_object.md)与[trait对象](../../12_traits/06_trait_object.md)。

---

## 核心代码示例

```rust
trait Draw {
    fn draw(&self);
}

// 泛型参数实现静态分派
fn draw_static<T: Draw>(item: &T) {
    item.draw();
}

// trait对象实现动态分派
fn draw_dynamic(item: &dyn Draw) {
    item.draw();
}

struct Circle;
impl Draw for Circle {
    fn draw(&self) { println!("Circle"); }
}

fn main() {
    let c = Circle;
    draw_static(&c);   // 静态分派
    draw_dynamic(&c);  // 动态分派
}
```

---

## 详细注释

- `fn draw_static<T: Draw>(item: &T)` 泛型参数实现静态分派，编译期确定类型。
- `fn draw_dynamic(item: &dyn Draw)` trait对象实现动态分派，运行时确定类型。
- 泛型和trait对象可结合用于灵活接口设计。

## 边界情况与常见错误

- **边界情况**：
  - trait对象需对象安全。
  - 泛型和trait对象混用时的生命周期和类型约束。
- **常见错误**：
  - trait未对象安全导致编译错误。
  - 泛型参数未满足trait bound。
  - trait对象生命周期不明确。

---

## FAQ

- **Q: 泛型和trait对象有何区别？**
  - A: 泛型为编译期多态，trait对象为运行时多态。
- **Q: trait对象的对象安全如何判断？**
  - A: 不能有泛型方法、Self返回值等。
- **Q: 泛型与trait对象常见应用场景？**
  - A: 插件系统、容器、接口抽象等。
