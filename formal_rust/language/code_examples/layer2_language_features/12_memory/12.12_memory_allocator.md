# 12.12 内存分配器（Memory Allocator）

---

## 理论简述

内存分配器负责管理程序的堆分配与释放。Rust支持自定义全局分配器，适用于嵌入式、性能优化等场景。相关理论详见[内存管理机制](../../11_memory_management/01_memory_management_theory.md)与[分配器实现](../../11_memory_management/05_allocator.md)。

---

## 核心代码示例

```rust
use std::alloc::{GlobalAlloc, Layout, System};

struct MyAllocator;

unsafe impl GlobalAlloc for MyAllocator {
    unsafe fn alloc(&self, layout: Layout) -> *mut u8 {
        System.alloc(layout)
    }
    unsafe fn dealloc(&self, ptr: *mut u8, layout: Layout) {
        System.dealloc(ptr, layout)
    }
}

#[global_allocator]
static GLOBAL: MyAllocator = MyAllocator;

fn main() {
    let v = Box::new(42);
    println!("v = {}", v);
}
```

---

## 详细注释

- 实现GlobalAlloc trait可自定义分配器逻辑。
- 通过#[global_allocator]指定全局分配器。
- 可用于内存池、跟踪分配、嵌入式优化等。

## 边界情况与常见错误

- **边界情况**：
  - 嵌入式、实时系统需定制分配器。
  - 多线程分配器需保证线程安全。
- **常见错误**：
  - 分配器实现不安全导致内存泄漏。
  - 忘记释放内存或重复释放。
  - 分配器与平台ABI不兼容。

---

## FAQ

- **Q: Rust如何自定义全局分配器？**
  - A: 实现GlobalAlloc trait并用#[global_allocator]标注。
- **Q: 分配器常见应用场景？**
  - A: 嵌入式、内存池、性能优化、内存跟踪等。
- **Q: 分配器相关编译错误如何排查？**
  - A: 检查trait实现、内存释放和线程安全。
