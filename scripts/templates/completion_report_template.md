# 完成报告模板（强制规范，P4-1）

> 用途：所有 `reports/*COMPLETION*.md` / `reports/*完成*.md` 必须遵循本模板。
> 目的：杜绝"全部完成 / 全过"式自夸叙事，使每份完成报告**可复核、含反例、暴露未达标项**。
> 执行：脚本/审计可在生成报告时 `diff` 本模板的强制段，缺段即视为不合规。

---

## 强制段（缺一不可）

### 1. 元信息

- 日期、范围、工具链基线（如 Rust 1.97.0）、报告类型（完成/审计/迁移）、**验收状态**（必须含未达标项数量，禁止只写"全部通过"）。

### 2. 执行摘要（必须含"未达标项"）

- 完成的 1–5 条要点；**未达标/遗留项 ≥1 条**（若没有，须说明"本次无遗留"并给出可复核证据）。
- 一句话结论（含量化结果，禁止纯形容词）。

### 3. 反例与失败项（Counter-examples & Failures）

- 至少列出本轮**未通过 / 误判 / 回退**的案例，或显式声明"本次未观察到反例，验证方法：<命令>"。
- 每个反例：现象 → 根因 → 处置 → 验证命令。

### 4. 可复核命令（Reproducible）

- 报告中**每一个量化结论**必须附一条可执行命令与期望输出，例如：

  ```bash
  python scripts/check_metadata_consistency.py        # 期望：D1..D6 计数
  python scripts/detect_content_overlap_v2.py         # 期望：hits=N
  python scripts/check_kg_shapes.py                   # 期望：K1..K6
  python scripts/semantic_health.py                   # 期望：total_score / grade
  ```

- 禁止"无法复现的断言"。

### 5. 基线数字（Baseline）

- 给出可机器跟踪的关键指标当前值与目标值（表格）。

### 6. 变更文件清单（Manifest）

- 新增/修改/删除的文件路径列表，每条注明变更类型与行数变化。

### 7. 风险与下一步（Risks & Next）

- 已知风险、依赖、回退方案；下一步任务与优先级。

---

## 反例（不合规写法，禁止）

```markdown
## 结论
本项目已完成全部对齐，10 大质量门全部通过，达到可持续维护基线。
```

> 不合规原因：无未达标项、无反例、无可复核命令、无基线数字、无变更清单。

## 合规写法（示例骨架）

```markdown
## 验收状态
通过 12 / 13 项；未达标 1 项：D1 Bloom 互斥 64 例（目标 0）。

## 执行摘要
- 完成：……
- 未达标/遗留：D1 64 例、T2 关系塌缩 99.2%、KG 缺 bloomLevel 55 实体（记录不阻断）。

## 反例与失败项
- D6 误报：早期规则把 "Explains how Cargo resolves…" 判为套话（已收紧模板，复跑 108→108 均为真套话）。验证：`<命令>`。

## 可复核命令
python scripts/semantic_health.py   # 期望：total=54.2 grade=FAIL（基线）

## 基线数字
| 指标 | 当前 | 目标 |
| metadata_score | 36.3 | ≥60 |
```

- 在 CI 增加 lint：报告 md 必须含 `## 反例与失败项`、`## 可复核命令`、`## 基线数字` 三个段标题，否则 warning。
- 可由 `scripts/check_report_compliance.py`（待建）实现。
