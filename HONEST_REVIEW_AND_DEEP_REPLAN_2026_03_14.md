✅ 好:

```markdown
## Rust 1.94 array_windows 语义

### 形式化定义
`array_windows<const N: usize>()` 将切片 `&[T]` 转换为 `&[T; N]` 的迭代器

### 语义优势
1. **类型安全**: 编译期确定窗口大小，避免运行时边界检查
2. **模式匹配**: 支持 `|[a, b, c]|` 解构
3. **零成本抽象**: 与 `windows()` 相比无运行时开销

### 实际示例
```rust
// 检测ABBA模式
s.as_bytes().array_windows::<4>()
    .any(|[a1, b1, b2, a2]| a1 == a2 && b1 == b2)
```

### 与1.93的对比

| 特性 | 1.93 | 1.94 | 优势 |
|------|------|------|------|
| 窗口类型 | `&[T]` (动态) | `&[T; N]` (静态) | 类型安全 |

```

## 错误处理

### 使用 ControlFlow 提前终止 (Rust 1.94+)

在 Rust 1.94 之前，异步流的提前返回通常使用 `Option` 或 `Result`：
```rust
// 1.93 方式 - 语义不够明确
items.iter().try_for_each(|n| {
    if n < 0 { None }  // 语义：找不到？还是错误？
    else { Some(()) }
})
```

Rust 1.94 的 `ControlFlow` 提供了更精确的控制流语义：

```rust
// 1.94 方式 - 语义明确
items.iter().try_for_each(|&n| {
    if n < 0 { ControlFlow::Break(n) }  // 语义：找到目标，终止
    else { ControlFlow::Continue(()) }   // 语义：继续搜索
})
```

**语义对比**:

- `Option`: 关注"有无"
- `Result`: 关注"成败"
- `ControlFlow`: 关注"继续/终止" - 更适合迭代控制

```
