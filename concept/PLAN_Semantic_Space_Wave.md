## 一、核心问题定义

用户反馈当前梳理缺少对 Rust **表征空间（Representational Space）** / **语义空间（Semantic Space）** 的系统分析：

> Rust 的内部机制是"内部完备但封闭"的——其程序设计、系统设计、架构设计的机制组合方面，语义方面必然存在"能表达"与"不能表达"的边界，以及等价表达的组合关系。

这需要从**元语言**层面回答：

1. **Rust 的表征空间是什么？**（类型系统 + 所有权 + Trait + 宏 + unsafe 构成的"设计空间"）
2. **安全 Rust 的语义封闭性**（safe 子集是否是"封闭世界"？unsafe 如何打破封闭？）
3. **能表达 vs 不能表达的边界**（哪些计算/结构可以表达？哪些被刻意排除？哪些需要 unsafe？）
4. **等价表达的语义保持**（同一概念在 Rust 中的多种表达方式的语义等价性分析）
5. **机制组合的语义空间**（所有权×生命周期×Trait×泛型×宏×async×unsafe 的组合爆炸与约束）

## 二、理论框架

基于网络权威来源，采用以下理论框架：

### 2.1 Felleisen 表达力理论（1989）

> "一个特性是否增加语言表达力，取决于它是否需要全局变换而非局部变换来实现。"

应用于 Rust：

- **局部变换可实现的**（不增加表达力）：语法糖（? 运算符）、模式匹配简化
- **需要全局变换的**（增加表达力）：异常 → Result（控制流全局重写）、继承 → Trait（设计模式全局改变）

### 2.2 观察等价性（Observational Equivalence）

> "若两个表达式在所有上下文中行为不可区分，则它们观察等价。"

应用于 Rust：

- `Box<T>` vs `Unique<T>`（unsafe 内部）——观察等价但实现不同
- `dyn Trait` vs `impl Trait`——语义等价但分发方式不同
- `Rc<T>` vs `Arc<T>`（单线程场景）——观察等价但线程安全保证不同

### 2.3 Rust 类型系统的图灵完备性

> Rust 类型系统（Trait + 关联类型）是图灵完备的（Leffler 2017），但受递归深度限制。这意味着类型检查本身可能不终止，但编译器通过递归限制保证了实际可用性。

### 2.4 语义保持与完备性

- **安全 Rust 是图灵完备的**：可以表达所有可计算函数
- **但不是 total 的**：存在无限循环（停机问题不可判定）
- **错误在定义时捕获**：编译通过 ⟹ 使用时无类型错误（强静态类型的语义封闭性）

### 3.1 新增：`00_meta/semantic_space.md` — Rust 表征空间总论（~800-1000 行）

这是 Wave 11 的核心交付物，系统论述 Rust 的表征空间。

**章节结构**：

#### §1 表征空间的定义

- Rust 表征空间 = {类型系统, 所有权系统, Trait系统, 生命周期系统, 宏系统, unsafe系统, async系统}
- 每个子系统的表征能力范围
- 子系统之间的交互约束（如：所有权 × 生命周期 = 借用规则）

#### §2 安全 Rust 的语义封闭性

- **封闭世界假设**：safe Rust 是一个封闭的形式系统
  - 公理：所有权唯一性、借用规则、生命周期约束
  - 推理规则：类型检查、借用检查
  - 封闭性：safe 代码不能突破这些规则（除非通过 unsafe）
- **逃逸舱口**：unsafe 作为封闭系统的"门"
  - unsafe 不改变类型系统的规则，但允许程序员手动保证不变量
  - 类比：Haskell 的 `unsafePerformIO`、ML 的 `unsafe_cast`

#### §3 能表达 vs 不能表达的边界

**能且高效表达**（Rust 的 Sweet Spot）：

| 概念 | Rust 表达 | 语义保持 | 成本 |
|:---|:---|:---|:---|
| 系统编程 | 所有权 + unsafe | 完全 | 零运行时 |
| 零成本抽象 | 泛型 + 单态化 | 完全 | 编译时间 |
| fearless并发 | Send/Sync + 借用 | 完全 | 零运行时 |
| 确定性资源管理 | RAII + Drop | 完全 | 零运行时 |
| 编译期计算 | const generics + const fn | 部分 | 编译时间 |

**能但低效/痛苦表达**：

| 概念 | Rust 表达 | 痛点 | 替代方案 |
|:---|:---|:---|:---|
| GUI 开发 | 生命周期与回调冲突 | 自引用、事件循环 | `Rc<RefCell>`、`Pin` |
| 动态类型 | enum 模拟 | 样板代码 | `dyn Any`（有限）|
| 运行时反射 | `Any::downcast` | 类型信息丢失 | 宏生成代码 |
| 复杂元编程 | 过程宏 | 调试困难、无类型信息 | `macro_rules!` + 约定 |
| 快速原型 | 编译时间 + 学习曲线 | 迭代慢 | `cargo script` |

**不能表达（或故意排除）**：

| 概念 | 排除原因 | Rust 替代 | 历史证据 |
|:---|:---|:---|:---|
| 绿色线程 | FFI 成本、运行时依赖 | async/await + OS 线程 | RFC 230 |
| OOP 继承 | Orphan Rule、组合优于继承 | Trait + 组合 | 设计哲学 |
| 隐式转换 | 类型安全、意外行为 | 显式 `From`/`Into` | 设计哲学 |
| 异常控制流 | 隐藏控制流、非局部跳转 | `Result` + `?` | 设计哲学 |
| GC 自动回收 | 运行时开销、非确定性 | 所有权 + `Rc`/`Arc` | 设计哲学 |
| 运行时反射 | 编译期信息擦除 | 宏 + `Any` | 设计哲学 |
| 可变长度数组 (VLA) | 栈安全、类型系统简化 | `Vec<T>` | 设计哲学 |
| 联合体自动析构 | 不知道哪个变体活跃 | `enum` + `match` | 设计哲学 |

#### §4 等价表达的语义保持

**等价表达谱系**：

```text
同一语义的不同 Rust 表达方式:

继承层次 ──→ Trait + 默认方法 + 组合
   │              │
   │              └─ 语义保持：多态替换
   │              └─ 成本：无运行时差异
   │
异常控制流 ──→ Result<T, E> + ? 运算符
   │              │
   │              └─ 语义保持：错误路径显式化
   │              └─ 成本：零运行时（与异常表对比）
   │
虚函数调用 ──→ enum + match  或  dyn Trait
   │              │
   │              └─ enum: 静态分发、零成本、封闭变体
   │              └─ dyn: 动态分发、vtable、开放扩展
   │
GC 自动回收 ──→ 所有权 + Rc/Arc + Weak
   │              │
   │              └─ 语义保持：内存最终释放
   │              └─ 成本：循环引用需手动打破
   │
模板元编程 ──→ const generics + 过程宏
   │              │
   │              └─ 语义保持：编译期计算
   │              └─ 成本：宏无类型信息
```

**等价性判定标准**：

- **观察等价**：所有上下文中行为不可区分
- **语义等价**：最终计算结果相同（允许实现差异）
- **性能等价**：运行时开销相同（零成本抽象）

#### §5 机制组合的语义空间

Rust 的核心机制可以看作一个代数组合系统：

**基础算子**：

- `Own(T)`：所有权（线性资源）
- `Borrow(T, mode)`：借用（共享/独占）
- `Lifetime('a)`：生命周期约束
- `Trait(T)`：行为抽象
- `Generic<T>`：参数多态

**组合规则**：

```text
Own(T) × Lifetime('a) → Box<'a, T>  // 有生命周期的所有权
Borrow(T, shared) × Lifetime('a) → &'a T  // 共享引用
Borrow(T, exclusive) × Lifetime('a) → &'a mut T  // 独占引用
Trait(T) × Generic<T> → impl Trait / dyn Trait  // 抽象返回
Own(T) × Trait(Send) → 跨线程转移安全
Own(T) × Trait(Sync) → 跨线程共享安全
```

**非法组合**（编译器拒绝）：

```text
Borrow(T, shared) × Borrow(T, exclusive) → ❌ 编译错误 E0502
Own(T) × Own(T) → ❌ 所有权已转移 E0382
Lifetime('a) × Lifetime('b) where 'a > 'b → ❌ 生命周期不足 E0597
```

**组合爆炸与约束**：

- 合法组合 ≈ 类型系统可判定的程序空间
- 非法组合 ≈ 类型系统排除的程序空间（UB 空间）
- unsafe = 手动证明非法组合实际上是安全的

#### §6 Rust 表征空间与其他语言的对比

| 维度 | Rust | C++ | Haskell | Go | Java |
|:---|:---|:---|:---|:---|:---|
| 表征空间边界 | 编译器强制 | 程序员自律 | 类型系统 | GC 简化 | VM 抽象 |
| 封闭性 | safe 封闭，unsafe 逃逸 | 完全开放 | 纯函数封闭，IO 逃逸 | 运行时封闭 | 类型封闭 |
| 表达力等价 | 与 C++ 等价（图灵完备） | 基准 | 与 Rust 等价 | 与 Rust 等价 | 与 Rust 等价 |
| 有效表达差异 | 零成本安全 | 零成本但不安全 | 安全但有 GC | 简单但不零成本 | 安全但有 VM |
| 不能表达 | GC、继承、异常 | — | 无 GC 的系统编程 | 零成本抽象 | 无 VM 的系统编程 |
