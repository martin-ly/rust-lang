## 五、内容规范

### 5.1 文件命名规范

- 使用 `NN_english_name.md` 格式，确保国际化和排序
- 例如：`01_ownership.md`、`02_borrowing.md`

### 5.2 内容结构模板

每个概念文件应遵循以下结构：

```markdown
# 概念名称

## 一、权威定义（Wikipedia / 官方文档对齐）
## 二、概念属性矩阵
## 三、形式化理论根基
## 四、知识来源关系
## 五、思维导图
## 六、决策/边界判定树
## 七、示例与反例分析
## 八、相关概念关联
## 九、待补充与演进方向
```

### 5.3 来源标注规范

所有关键论断必须标注来源，格式：

> **[来源类型: 具体来源]** 论断内容
>
> 例如：**[Wikipedia: Rust (programming language)]** Rust is a multi-paradigm, general-purpose programming language.
