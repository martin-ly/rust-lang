# Rust 知识体系内容分级体系（Grading System）

> **受众**: [初学者] · [进阶] · [专家] · [研究者]
> **定位**: 为 `concept/` 知识体系建立统一的**内容深度分级标准**，解决"认知悬崖"问题，让学习者自主决定阅读深度。
> **决策依据**: 1-B（全栈均衡）+ 3-B（标记分级）+ 4-BC（形式化直觉+工程形式化）
> **对应 Rust 版本**: 1.96.0 (Edition 2024)
> **状态**: v3.0 分级体系 — 强制执行

---

## 四级分级标签

每个 `concept/` 文件必须在头部元数据区域明确标注以下内容分级：

```markdown
> **内容分级**: [综述级] / [实验级] / [专家级] / [研究者级]
```

### 综述级 [Survey]

| 属性 | 说明 |
|:---|:---|
| **定义** | 提供某主题的概览性介绍，帮助学习者建立宏观认知地图 |
| **代码要求** | ≥0 个可编译示例；允许伪代码或概念性描述 |
| **数学形式化** | 无；仅用自然语言描述 |
| **前置知识** | 无特殊要求 |
| **典型读者** | 初学者、技术管理者、跨领域迁移者 |
| **阅读策略** | 快速浏览，建立印象即可 |
| **典型内容** | 多语言对比矩阵、生态全景图、应用场景概述 |

**示例文件**: `05_comparative/03_paradigm_matrix.md` · `06_ecosystem/04_application_domains.md`

---

### 实验级 [Experimental]

| 属性 | 说明 |
|:---|:---|
| **定义** | 介绍 nightly/不稳定特性、实验性工具或前沿研究方向 |
| **代码要求** | ≥1 个可编译示例（需标注 `#![feature(...)]`）；允许 `compile_fail` 展示限制 |
| **数学形式化** | 可无；如有，必须标注 `[教学类比 — 非严格证明]` |
| **前置知识** | 需掌握对应稳定特性的基础概念 |
| **典型读者** | 进阶开发者、早期采用者、贡献者 |
| **阅读策略** | 按需阅读；关注状态标签（ nightly / MCP / RFC 阶段）|
| **典型内容** | Effects System 预览、Pin Ergonomics 实验、Gen Blocks |

**示例文件**: `07_future/03_evolution.md` · `07_future/rust_1_97_preview.md`

---

### 专家级 [Expert]

| 属性 | 说明 |
|:---|:---|
| **定义** | 深入讲解生产环境必需的高级模式、性能优化、安全边界 |
| **代码要求** | ≥2 个可编译示例；必须包含反例（`compile_fail` 或错误示范）|
| **数学形式化** | 可无；如有，必须标注 `[教学类比 — 非严格证明]` |
| **前置知识** | 需完整掌握 L1-L3 核心概念 |
| **典型读者** | 中高级 Rust 开发者、库作者、系统工程师 |
| **阅读策略** | 精读；必须配合代码实践 |
| **典型内容** | Unsafe 内存模型、并发无锁算法、宏卫生性、FFI 边界 |

**示例文件**: `03_advanced/03_unsafe.md` · `03_advanced/01_concurrency.md`

---

### 研究者级 [Research]

| 属性 | 说明 |
|:---|:---|
| **定义** | 面向程序语言研究者的形式化内容；或与最新学术进展直接相关 |
| **代码要求** | ≥1 个可验证示例（Kani/Verus/Miri）；允许纯概念性描述 |
| **数学形式化** | 允许；但必须明确区分**教学类比**与**严格定理** |
| **前置知识** | 需具备 PL 理论基础或愿意接受高认知负荷 |
| **典型读者** | 研究人员、形式化验证工程师、PL 爱好者 |
| **阅读策略** | 选读；非 Rust 开发的必要路径 |
| **典型内容** | Tree Borrows 别名模型、分离逻辑应用、类型论基础 |

**示例文件**: `04_formal/03_ownership_formal.md` · `04_formal/22_modern_verification_tools.md`


| 受众 \ 分级 | 综述级 | 实验级 | 专家级 | 研究者级 |
|:---|:---:|:---:|:---:|:---:|
| **初学者** | ✅ 首选 | ⚠️ 慎读 | ❌ 不推荐 | ❌ 不推荐 |
| **进阶** | ✅ 快速浏览 | ✅ 首选 | ⚠️ 按需 | ❌ 不推荐 |
| **专家** | ✅ 查缺补漏 | ✅ 跟踪前沿 | ✅ 首选 | ⚠️ 按需 |
| **研究者** | ✅ 了解背景 | ✅ 了解趋势 | ✅ 工程参考 | ✅ 首选 |

## L4 形式化层特殊规范（决策 4-BC）

L4 (`04_formal/`) 所有文件必须遵守以下附加规范：

### 1. 教学类比标注

任何使用数学符号（⊗, ⊸, !, ∀, ∃ 等）的段落，若**未经机器验证**，必须标注：

```markdown
> **[教学类比]** 以下符号推理用于建立直觉，非严格形式化证明。
> 如需严格证明，请参考 [RustBelt (POPL 2018)](...) 或 [Tree Borrows (PLDI 2025)](...)。
```

### 2. 纯数学内容删除标准

以下类型的纯数学内容**应当删除或移至外部链接**：

- 与 Rust 语义无直接关联的线性逻辑公理系统完整推导
- 与 Rust 类型系统无直接关联的纯范畴论构造
- 未引用具体 Rust 语言特性的一般性类型论定理

### 3. 工程形式化强化

以下工具/进展**必须覆盖**（至少概念 + 可验证示例）：

| 工具/进展 | 最小覆盖要求 | 状态 |
|:---|:---|:---:|
| **Tree Borrows** | 概念 + Miri 验证示例 | 待补全 |
| **BorrowSanitizer** | 概念 + 与 Miri 的对比 | 待补全 |
| **Kani** | 概念 + Autoharness 示例 | 待补全 |
| **Safety Tags (RFC #3842)** | 概念 + 标准库 API 标注示例 | 待补全 |
| **AutoVerus** | 概念 + LLM 辅助证明示例 | 待补全 |

## 认知悬崖缓冲导航

在 L3（高级层）与 L4（形式化层）之间，建立缓冲导航：

```text
L3 高级 ──→ [03_advanced/00_before_formal.md] ──→ L4 形式化
              "是否需要读 L4？"
              - 我是系统开发者？→ 读 L4 的 [工程形式化] 部分
              - 我是 PL 研究者？→ 读 L4 全部
              - 我只想写更好的 Rust 代码？→ 跳过 L4，继续 L5-L6
```

## CI 强制检查

`kb_auditor.py` 已增强以下检查：

1. **双标签存在性**: 每个 `concept/*.md` 必须包含 `**受众**:` 和 `**内容分级**:`
2. **标签有效性**: 标签值必须在预定义集合中
3. **代码示例匹配**: `[专家级]` 文件必须包含 ≥2 个 ` ```rust ` 块；`[研究者级]` 必须包含 ≥1 个可验证示例
4. **L4 特殊检查**: `04_formal/` 文件必须包含 `[教学类比]` 或 `[严格形式化]` 标注
