# Agent 工作指南

## 项目概述

Rust 分层概念知识体系（`concept/`），v1.2 封顶状态。

- **45 个 Markdown 文件**，覆盖 L0-L7 八层架构
- **~35,000 行内容**，从入门直觉到形式化验证
- **Bloom 层级标注 100% 覆盖**（45/45）
- **来源标注率平均 13.6%**（核心概念文件 ≥15%）
- **0 死链接**，代码块问题 0

## 目录结构

```
concept/
├── 00_meta/          # 元信息：索引、学习指南、语义空间、审计清单
├── 01_foundation/    # L1 基础：所有权、借用、生命周期、类型系统
├── 02_intermediate/  # L2 进阶：Trait、泛型、内存管理、错误处理
├── 03_advanced/      # L3 高级：并发、异步、unsafe、宏
├── 04_formal/        # L4 形式化：线性逻辑、类型论、所有权形式化、RustBelt
├── 05_comparative/   # L5 对比：Rust vs C++/Go/Haskell，范式矩阵
├── 06_ecosystem/     # L6 生态：工具链、设计模式、区块链、ECS、WASI
├── 07_future/        # L7 前沿：AI 集成、形式化方法、语言演进、版本跟踪
```

## 权威来源分级

|:---|:---|:---|
| 一级 | Rust Reference / RFCs / POPL·PLDI 论文 | 定义、定理、边界 |

| 三级 | TRPL / Rustonomicon | 工程直觉、示例 |

## 维护规范

1. **版本跟踪**: 每 6 周扫描 Rust stable release，更新 `07_future/rust_version_tracking.md`
2. **来源更新**: 新增内容必须附 `[来源: ...]` 标注
3. **交叉一致性**: 修改核心概念定义时，检查所有引用该概念的文件
4. **审计运行**: 重大修改后执行 `python scripts/concept_audit.py`
