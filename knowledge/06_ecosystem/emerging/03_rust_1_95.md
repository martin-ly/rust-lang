# Rust 1.95 新特性

> **Bloom 层级**: 理解
> **📌 简介**: Rust 1.95.0 于 2026 年 4 月 16 日发布，是 Rust 2026 年上半年的重要里程碑版本。
> 核心亮点包括 `if let` guards in match、`cfg_select!` 编译时条件宏、PowerPC 内联汇编稳定化、标准库大量新 API（`assert_matches!`、`push_mut`、`update` 等），以及 7 个 Apple 平台晋升 Tier 2。
>
> **发布日期**: 2026-04-16
> **版本状态**: ✅ Stable
> **权威来源**: [Rust 1.95 Release Notes](https://releases.rs/docs/1.95.0/)

---

## 🎯 版本概览

> **[来源: Rust Official Docs]**

Rust 1.95 主要聚焦于：

- **语言特性**: `if let` guards 进入 match 表达式，`cfg_select!` 编译时条件选择，路径段关键字导入重命名
- **标准库扩展**: `assert_matches!`、`push_mut`/`insert_mut`、`Atomic*::update`、`MaybeUninit` 数组转换、`Layout` 分配器辅助、`core::range`
- **平台扩展**: PowerPC/PowerPC64 内联汇编稳定化，7 个 Apple 平台 + powerpc64 musl 晋升 Tier 2
- **编译器**: `--remap-path-scope` 稳定化，`str::contains` aarch64 neon 加速
- **工具改进**: rustdoc 搜索排序优化、"hide deprecated items" 设置

## 🚀 主要新特性
>
> **[来源: Rust Official Docs]**

### 1. `if let` Guards in Match Arms
>
> **[来源: Rust Official Docs]**

Rust 1.88 稳定了 let chains。1.95 将此能力带入 match 表达式，允许基于模式匹配的条件守卫：

```rust,ignore
match value {
    Some(x) if let Ok(y) = compute(x) => {
        // `x` 和 `y` 在此处均可用
        println!("{}, {}", x, y);
    }
    _ => {}
}
```

> **注意**: 编译器目前不会将 `if let` guards 中的模式纳入 match 穷尽性检查，与常规 `if` guards 行为一致。

### 2. `cfg_select!` 编译时条件选择宏
>
> **[来源: Rust Official Docs]**

`cfg_select!` 提供类似编译期 `match` 的 `cfg` 条件选择能力，可替代流行的 `cfg-if` crate：

```rust,ignore
// 条件代码块选择
cfg_select! {
    unix => {
        fn foo() { /* Unix 特定实现 */ }
    }
    target_pointer_width = "32" => {
        fn foo() { /* 非 Unix 32 位实现 */ }
    }
    _ => {
        fn foo() { /* 回退实现 */ }
    }
}

// 表达式级使用
let os_str = cfg_select! {
    windows => "windows",
    target_os = "macos" => "macos",
    _ => "other",
};
```

> **跟踪 Issue**: [rust-lang/rust#136098](https://github.com/rust-lang/rust/issues/136098)

### 3. `assert_matches!` / `debug_assert_matches!`
>
> **[来源: Rust Official Docs]**

模式匹配断言宏，在测试中验证值是否符合预期模式：

```rust,ignore
use std::assert_matches;

let result: Result<i32, &str> = Ok(42);
assert_matches!(result, Ok(n) if n > 0);

let opt: Option<i32> = Some(10);
assert_matches!(opt, Some(_));

// debug 版本（release 模式下无开销）
debug_assert_matches!(result, Ok(42));
```

### 4. 集合 `push_mut` / `insert_mut`
>
> **[来源: [Rust Reference](https://doc.rust-lang.org/reference/)]**

`Vec`、`VecDeque`、`LinkedList` 新增返回可变引用的插入方法：

```rust,ignore
use std::collections::{VecDeque, LinkedList};

// Vec: push 并获取新元素的可变引用
let mut vec = vec![1, 2, 3];
let last: &mut i32 = vec.push_mut(4);
*last += 10;
assert_eq!(vec, [1, 2, 3, 14]);

// Vec: 在指定位置插入并获取可变引用
let inserted: &mut i32 = vec.insert_mut(1, 100);
*inserted *= 2;
assert_eq!(vec, [1, 200, 2, 3, 14]);

// VecDeque: 双端 push 返回可变引用
let mut deque = VecDeque::new();
let front = deque.push_front_mut(1);
*front += 1;
let back = deque.push_back_mut(10);
*back -= 1;
assert_eq!(deque, [2, 9]);

// LinkedList
let mut list = LinkedList::new();
*list.push_front_mut(1) += 1;
*list.push_back_mut(3) -= 1;
assert_eq!(list.into_iter().collect::<Vec<_>>(), [2, 2]);
```

### 5. `MaybeUninit` 数组双向转换
>
> **[来源: [The Rust Programming Language](https://doc.rust-lang.org/book/)]**

```rust,ignore
use std::mem::MaybeUninit;

// [MaybeUninit<T>; N] ↔ MaybeUninit<[T; N]>
let arr: [MaybeUninit<i32>; 4] = [
    MaybeUninit::new(1),
    MaybeUninit::new(2),
    MaybeUninit::new(3),
    MaybeUninit::new(4),
];
let uninit_array = MaybeUninit::from(arr);

// 反向转换
let arr_back: [MaybeUninit<i32>; 4] = uninit_array.into();

// AsRef/AsMut 获取视图
let view: &[MaybeUninit<i32>] = uninit_array.as_ref();
```

### 6. 原子类型 `update` / `try_update`
>
> **[来源: [Rust Standard Library](https://doc.rust-lang.org/std/)]**

```rust,ignore
use std::sync::atomic::{AtomicU32, Ordering};

let counter = AtomicU32::new(5);

// update: 读取 → 计算 → CAS 循环封装，返回旧值
let prev = counter.update(Ordering::Relaxed, |x| x * 2);
assert_eq!(prev, 5);
assert_eq!(counter.load(Ordering::Relaxed), 10);

// try_update: 条件更新，仅在闭包返回 Some 时执行
let did_update = counter.try_update(Ordering::Relaxed, |x| {
    if x > 8 { Some(x - 3) } else { None }
});
assert!(did_update.is_some());
assert_eq!(counter.load(Ordering::Relaxed), 7);
```

### 7. 原始指针安全转换
>
> **[来源: [Rustonomicon](https://doc.rust-lang.org/nomicon/)]**

```rust,ignore
let ptr: *const u8 = &42u8;

// SAFETY: 指针已知有效
let ref_u8: &u8 = unsafe { ptr.as_ref_unchecked() };

let mut_ptr: *mut String = &mut String::from("hello");
let ref_mut: &mut String = unsafe { mut_ptr.as_mut_unchecked() };
```

### 8. `bool: TryFrom<{integer}>`
>
> **[来源: [Rust By Example](https://doc.rust-lang.org/rust-by-example/)]**

```rust,ignore
// 显式转换整数到 bool
let flag: bool = 1i32.try_into()?; // Ok(true)
let flag: bool = 0i32.try_into()?; // Ok(false)
let flag: bool = 2i32.try_into()?; // Err(BoolError)
```

### 9. `core::range` 模块与 `RangeInclusive` 类型
>
> **[来源: [Rust Reference](https://doc.rust-lang.org/reference/)]**

```rust,ignore
use core::range::RangeInclusive;

let r = RangeInclusive::new(1, 10);
assert_eq!(r.start(), &1);
assert_eq!(r.end(), &10);
assert!(r.contains(&5));
```

### 10. `Cell<[T]>` 与 `Cell<[T; N]>` 的数组视图
>
> **[来源: [The Rust Programming Language](https://doc.rust-lang.org/book/)]**

```rust,ignore
use std::cell::Cell;

let cell_array: Cell<[i32; 3]> = Cell::new([1, 2, 3]);
let cell_slice: &[Cell<i32>] = cell_array.as_ref();

cell_slice[0].set(100);
assert_eq!(cell_array.get()[0], 100);
```

### 11. `Layout` 分配器辅助方法
>
> **[来源: [Rust Standard Library](https://doc.rust-lang.org/std/)]**

```rust,ignore
use std::alloc::Layout;

// dangling_ptr: 获取对齐的悬空指针（用于占位）
let layout = Layout::new::<i32>();
let ptr = layout.dangling_ptr();

// repeat: 计算 N 个副本的布局
let repeated = layout.repeat(10).unwrap();

// repeat_packed: 紧凑排列（无填充）
let packed = layout.repeat_packed(10);

// extend_packed: 与其他类型紧凑扩展
let combined = layout.extend_packed(Layout::new::<u8>()).unwrap();
```

### 12. `core::hint::cold_path`
>
> **[来源: [Rustonomicon](https://doc.rust-lang.org/nomicon/)]**

标记极少执行的分支路径，优化编译器的分支预测和代码布局：

```rust,ignore
use std::hint::cold_path;

fn parse_input(input: &str) -> Result<i32, ParseError> {
    if input.is_empty() {
        cold_path();
        return Err(ParseError::Empty);
    }
    input.parse().map_err(|_| {
        cold_path();
        ParseError::Invalid
    })
}
```

### 13. Const 上下文 API 稳定化
>
> **[来源: [Rust By Example](https://doc.rust-lang.org/rust-by-example/)]**

以下 API 现在可在 `const fn` 中使用：

```rust,ignore
use std::ops::ControlFlow;

const fn check_flow() -> bool {
    let cf = ControlFlow::Break(42);
    cf.is_break() // ✅ Rust 1.95 前不可用
}

// fmt::from_fn 也可在 const 中使用
```

### 14. 路径段关键字导入重命名
>
> **[来源: [Rust Reference](https://doc.rust-lang.org/reference/)]**

现在可以导入路径段关键字（如 `crate`、`super`、`self`）并进行重命名：

```rust
// ✅ Rust 1.95: 允许重命名路径段关键字
use std::io::{self as io_mod, Read};
```

### 15. `irrefutable_let_patterns` Lint 不再作用于 Let Chains
>
> **[来源: [The Rust Programming Language](https://doc.rust-lang.org/book/)]**

```rust,ignore
// 之前: 触发 irrefutable_let_patterns lint
if let Some(x) = opt && let Some(y) = opt2 {
    // ...
}

// Rust 1.95: let chains 不再触发此 lint
```

### 16. PowerPC/PowerPC64 内联汇编稳定化
>
> **[来源: [Rust Standard Library](https://doc.rust-lang.org/std/)]**

```rust
use std::arch::asm;

#[cfg(any(target_arch = "powerpc", target_arch = "powerpc64"))]
fn read_timebase() -> u64 {
    let mut tb: u64;
    unsafe {
        asm!(
            "mftb {0}",
            out(reg) tb,
            options(nomem, nostack, preserves_flags),
        );
    }
    tb
}
```

## ⚠️ 兼容性注意
>
> **[来源: [Rust By Example](https://doc.rust-lang.org/rust-by-example/)]**

### 1. JSON Target Specs 已去稳定化
>
> **[来源: [Rust Reference](https://doc.rust-lang.org/reference/)]**

自定义目标规范现在需要 `-Z unstable-options`：

```bash
# ❌ 之前: 可直接使用
rustc --target my-target.json

# ✅ 之后: 需要 nightly + 不稳定选项
rustc +nightly -Z unstable-options --target my-target.json
```

> Cargo 自动处理：使用 `-Z json-target-spec` flag。

### 2. `$crate::{self}` 导入不再允许
>
> **[来源: [The Rust Programming Language](https://doc.rust-lang.org/book/)]**

```rust,ignore
// ❌ 编译错误
use $crate::{self};

// ✅ 正确写法
use $crate;
```

### 3. `Eq::assert_receiver_is_total_eq` 已废弃
>
> **[来源: [Rust Standard Library](https://doc.rust-lang.org/std/)]**

手动实现 `Eq` 时会触发未来兼容性警告：

```rust,ignore
// ❌ 触发警告
impl Eq for MyType {
    fn assert_receiver_is_total_eq(&self) {}
}

// ✅ 无需手动实现此方法
impl Eq for MyType {}
```

### 4. 数组强制转换推断变化
>
> **[来源: [Rustonomicon](https://doc.rust-lang.org/nomicon/)]**

数组强制转换可能减少类型推断约束，某些代码需要显式类型标注：

```rust,ignore
// 可能需要显式标注
let arr: [i32; 3] = [1, 2, 3];
let boxed: Box<[i32]> = arr.into(); // 若推断失败需显式标注
```

### 5. `mut ref` 模式正确 Feature-Gate
>
> **[来源: [Rust By Example](https://doc.rust-lang.org/rust-by-example/)]**

```rust,ignore
// ❌ 之前（误稳定）: struct 字段简写中允许
let Struct { mut ref field } = s;

// ✅ 之后: 需要 `#![feature(match_ergonomics_2024)]`
```

### 6. 其他兼容性变更
>
> **[来源: [Rust Reference](https://doc.rust-lang.org/reference/)]**

- **`ambiguous_glob_imported_traits` FCW**: glob 导入的歧义 trait 现在触发未来兼容性警告
- **生命周期边界检查增强**: 仅包含类型参数的类型现在检查生命周期边界
- **可见性相关歧义导入错误**: 报告更多可见性相关的歧义导入错误
- **`non_exhaustive` 枚举判别式读取**: 匹配 `#[non_exhaustive]` 枚举现在读取判别式，可能影响闭包捕获
- **derive helper attributes 冲突 FCW**: 与内置属性冲突的 derive helper attributes 触发警告
- **const-eval padding 一致性**: 极少数情况下可能导致编译错误（指针字节进入 padding）
- **`use $crate` 严格化**: 无重命名的 `$crate` 自导入不再允许

## 🔄 迁移指南
>
> **[来源: [The Rust Programming Language](https://doc.rust-lang.org/book/)]**

### 从 `cfg-if` 迁移到 `cfg_select!`
>
> **[来源: [Rust Standard Library](https://doc.rust-lang.org/std/)]**

```rust,ignore
// 之前 (cfg-if crate)
use cfg_if::cfg_if;
cfg_if! {
    if #[cfg(unix)] {
        fn os_specific() {}
    } else {
        fn os_specific() {}
    }
}

// 之后 (Rust 1.95 内置)
cfg_select! {
    unix => { fn os_specific() {} }
    _ => { fn os_specific() {} }
}
```

### 原子操作简化
>
> **[来源: [Rustonomicon](https://doc.rust-lang.org/nomicon/)]**

```rust,ignore
// 之前: 手动 CAS 循环
loop {
    let current = atomic.load(Ordering::Relaxed);
    let new = current * 2;
    if atomic.compare_exchange_weak(
        current, new, Ordering::Relaxed, Ordering::Relaxed
    ).is_ok() { break; }
}

// 之后: update 方法
atomic.update(Ordering::Relaxed, |x| x * 2);
```

### 断言迁移
>
> **[来源: [Rust By Example](https://doc.rust-lang.org/rust-by-example/)]**

```rust,ignore
// 之前
assert!(matches!(result, Ok(_)));

// 之后
assert_matches!(result, Ok(_));
```

## ⚖️ 设计权衡
>
> **[来源: [Rust Standard Library](https://doc.rust-lang.org/std/)]**

| API/特性 | 优点 | 代价 | 适用场景 |
|---------|------|------|---------|
| `cfg_select!` | 零依赖、内置 | 语法与 cfg-if 不同 | 条件编译 |
| `assert_matches!` | 语义清晰、错误信息友好 | 新增宏 | 测试中断言模式匹配 |
| `push_mut` | 避免二次查找 | API 新增 | 插入后立即修改 |
| `Atomic::update` | 消除 CAS 样板 | 内部循环 | 原子更新热路径 |
| `bool::TryFrom<int>` | 类型安全 | 需处理 Err | 整数到 bool 转换 |
| `Layout::dangling_ptr` | 安全的悬空指针 | 仅用于占位 | 自定义分配器 |
| `cold_path` | 分支预测优化 | 需判断冷热路径 | 错误处理路径 |
